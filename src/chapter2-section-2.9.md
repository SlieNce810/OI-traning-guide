# 2.9 数值方法简介

> **学习目标**：掌握二分法和三分法在连续函数极值/求根问题中的应用，理解单调性/单峰性/精度控制这三个核心要素。

## 理论基础

### 为什么需要学这个？

算法竞赛中的数值方法不像其他章节那样"理论丰富"，但它解决的是另一类问题：当解析解不存在或太难求时，如何用迭代逼近得到足够精确的数值答案。这类题的标志就是输出要求"保留X位小数"。很多同学觉得二分法 trivial，但一到实际题目就卡在终止条件、精度设置、函数性质判断这些细节上。本节的核心思想很简单：**在已知函数形状的前提下，用逐步缩小搜索区间的方法逼近答案**。二分法要求单调，三分法要求单峰——抓住这两个前提条件，剩下的都是机械操作。

### 核心概念

**二分法的单调性前提**：如果你知道函数 f(x) 在区间 [L,R] 上是单调的，且有 f(L) 和 f(R) 异号，那么二分法一定能找到根。取中点 mid = (L+R)/2，检查 f(mid)·f(L) 的符号，决定下一步搜索左半区还是右半区。核心口诀：**异号保中点，同号抛一半**。如果f不是单调的，二分法可能漏根。

**三分法的单峰函数前提**：如果要找凸函数的最小值（或凹函数的最大值），三分法是不二之选。在 [L,R] 内取两个三分点 m1 = L+(R-L)/3 和 m2 = R-(R-L)/3。比较 f(m1) 和 f(m2)：
- 对于凸函数（找最小值）：如果 f(m1) < f(m2)，极值点一定在 [L, m2] 内，丢掉右边；否则极值点在 [m1, R] 内，丢掉左边。
- 每轮丢弃1/3的区间。注意：三分法必须确认函数是单峰的，否则可能收敛到错误的极值点。

**误差控制与迭代终止条件**：数值方法的关键问题是"什么时候停下来"。常见的终止条件有：（1）区间宽度 R-L < EPS，其中 EPS 一般取 10^(-d-2)（d是输出要求的小数位数）；（2）固定迭代次数，例如二分100次可使区间缩小 2^100 ≈ 10^30 倍，对 double 精度已经足够。在实际竞赛中，**固定迭代次数比判断区间宽度更可靠**，因为避免了浮点比较的不可靠性。

**二分/三分终止条件的三种写法**：（1）**while(L+EPS < R)**——最直观的写法，当区间宽度小于EPS时停止。优势是语义清晰，劣势在于浮点精度问题：如果L和R量级很大（如10^9），L+EPS在浮点表示下等于L，导致死循环；反之量级很小（如10^{-9}），EPS可能不够精细。（2）**固定迭代次数**——`for(int i=0; i<100; i++)`，二分100次对double精度已足够。优势是完全避免浮点比较，无死循环风险；劣势是可能需要更多迭代（但100次通常可以忽略不计）。（3）**相对误差控制**——`while((R-L)/max(1.0, min(fabs(L), fabs(R))) > EPS)`，综合了绝对误差和相对误差。优势是对所有量级都健壮，劣势是实现较复杂。竞赛中最推荐的是**方法（2）**——简单、安全、可靠。对于三分法，同样建议固定迭代（通常80-100次足够），因为每次缩小的比例是2/3而非1/2，迭代效率略低于二分但有同样的健壮性。

**牛顿迭代法的几何直觉**：牛顿法求解 f(x)=0 的核心思想是**用切线不断逼近**。在猜测点x_k处画出函数f的切线，切线与x轴的交点就是下一个猜测x_{k+1}。切线方程是 y = f(x_k) + f'(x_k)(x-x_k)，令y=0得 x_{k+1} = x_k - f(x_k)/f'(x_k)。几何上看：每次迭代都是在当前点"沿着曲线方向直线走下去"，在曲线接近直线的地方收敛极快（二次收敛）。但牛顿法有陷阱：（1）如果f'(x_k)接近零（切线几乎水平），x_{k+1}会跳得很远；（2）如果初始猜测离根太远，可能不收敛。竞赛中牛顿法主要用于两类场景：求平方根（f(x)=x²-n, 迭代 x_{k+1}=(x_k+n/x_k)/2）和多项式求根（需配合导数）。对于一般竞赛场景，牛顿法相比二分法的优势在于：如果函数光滑且导数易于计算，牛顿法所需的迭代次数少一个数量级。

**常见陷阱**：
- 二分整数区间时 mid 用 (L+R)/2，但注意避免溢出：写成 L+(R-L)/2
- 浮点数的等值比较永远用 fabs(a-b) < EPS，不要用 ==
- 如果函数值量级很大或很小，EPS 需要相应调整
- 三分法在 O(10^3) 级别迭代即可满足 double 精度

**数值稳定性的基本概念**：数值稳定性是指在浮点运算的有限精度下，算法结果是否可靠。核心原则：（1）**避免相近数相减**——两个相近的浮点数相减会导致有效数字严重丢失（称为"抵消误差"）。例如解二次方程 x²-10^10x+1=0 时，用标准求根公式 x=(-b±√(b²-4ac))/(2a) 计算较小根，√(b²-4ac)≈|b|=10^10，分子做 10^10-10^10≈0 丢失所有精度。正确做法是用根与系数关系：若已知x₁≈10^10，则 x₂ = c/(a·x₁) ≈ 10^{-10}。（2）**避免大数吃小数**——大量级数与小量级数相加时，小数被完全吞没。例如累积 Σ 1/i² 用float时，从大加到小数值稳定，从小加到大则1/n²的贡献会逐渐丢失。（3）**避免递归计算的误差放大**——迭代公式的条件数（condition number）决定了误差放大倍数。牛顿法在重根附近的收敛从二次退化为线性，就是因为f'(x)趋近于0导致除法放大微小误差。在竞赛中，数值稳定性通常只需记住避免相近数相减和大数吃小数两种典型错误，其他情况double的53位尾数精度足够。

### 知识脉络

```
单调函数  ──→  二分法求根  ──→  整数二分 / 浮点二分
     │
     └── 应用: 最大化最小值 / 最小化最大值（二分答案）
     
单峰函数  ──→  三分法求极值  ──→  凸函数/凹函数
     │
     └── 注意: 必须先验证单峰性
```

二分法和三分法是数值方法的两大支柱。二分法不仅用于求根，更是"二分答案"这种竞赛套路的基础——当你需要"在满足条件的前提下最大化某个值"时，可以对答案值域二分然后判断可行性。三分法则专注于单峰极值问题，关键是确认函数的单峰性（常见于凸函数族的最大值函数）。本节中的**二分法的判定函数设计**与第1.2节二分答案中的check函数设计完全通用——两者都是将"求最优"转化为"判定可行性"；**三分法搜索单峰极值**则在第1.2节中贪心正确性的"单峰论证"（如突击战的调度最优性）中有理论上的呼应。

### 快速上手模板

```cpp
// 浮点二分法：求 f(x) = 0 在 [L,R] 上的根
const double EPS = 1e-9;
double bisection(double L, double R) {
    for (int iter = 0; iter < 100; iter++) {  // 固定迭代100次
        double mid = (L + R) / 2;
        if (f(L) * f(mid) <= 0)  // 根在左半区
            R = mid;
        else
            L = mid;
    }
    return (L + R) / 2;
}

// 三分法：求凸函数 f(x) 在 [L,R] 的最小值
double ternary_search(double L, double R) {
    for (int iter = 0; iter < 100; iter++) {
        double m1 = L + (R - L) / 3;
        double m2 = R - (R - L) / 3;
        if (f(m1) < f(m2))  // m1更靠近最小值（凸函数）
            R = m2;
        else
            L = m1;
    }
    return f((L + R) / 2);
}

// 整数二分答案：最大化满足 check(x) 的 x
int binary_answer(int lo, int hi) {
    while (lo < hi) {
        int mid = lo + (hi - lo + 1) / 2;  // 上取整，防死循环
        if (check(mid))
            lo = mid;  // mid可行，尝试更大的值
        else
            hi = mid - 1;
    }
    return lo;
}
```

## 例题37  误差曲线（Error Curves, Chengdu 2010, LA5009）

### 题目描述
有n条二次误差曲线，第i条为S_i(x) = a_i·x² + b_i·x + c_i（a_i ≥ 0）。定义总误差F(x) = max{S_1(x), S_2(x), …, S_n(x)}，即所有曲线的逐点最大值。求F(x)在区间[0, 1000]上的最小值。

**输入**：第一行T（T ≤ 100）。每组第一行n（1 ≤ n ≤ 10000）。接下来n行，每行三个整数a,b,c（0 ≤ a ≤ 100, |b| ≤ 5000, |c| ≤ 5000）。

**输出**：对于每组数据，输出F(x)的最小值，保留4位小数。

### 解题思路

**凸函数性质**：
每条曲线S_i(x)由于a_i ≥ 0，是凸函数（开口向上的抛物线）。多个凸函数的逐点最大值也是凸函数（U形）。

因此F(x)在[0, 1000]上是单峰的（先减后增），可以用三分搜索找到最小值。

**三分搜索**：
在区间[L, R]中，取两个三分点m1 = L + (R-L)/3和m2 = R - (R-L)/3。
- 如果F(m1) < F(m2)，最小值在[L, m2]中（收缩右边界R=m2）
- 否则，最小值在[m1, R]中（收缩左边界L=m1）

迭代约100次即可收敛到足够精度。

### 算法方法
- **数值方法/三分搜索**：利用凸函数的单峰性质
- **极值问题**：max of convex = convex

### 复杂度分析
- **时间复杂度**：O(T·n·iter)，n≤10000，iter≈100
- **空间复杂度**：O(n)，存储曲线系数

```cpp
// 例题37  误差曲线（Error Curves, Chengdu 2010, LA5009）
// 刘汝佳
#include <algorithm>
#include <cstdio>
using namespace std;
const int NN = 10000 + 10;
int T, n, a[NN], b[NN], c[NN];  // 曲线系数 a,b,c

// 计算F(x) = max a_i*x^2 + b_i*x + c_i（所有曲线的最大值）
double F(double x) {
  double ans = a[0] * x * x + b[0] * x + c[0];
  for (int i = 1; i < n; i++)
    ans = max(ans, a[i] * x * x + b[i] * x + c[i]);  // 取逐点最大值
  return ans;
}

int main() {
  scanf("%d", &T);
  while (T--) {
    scanf("%d", &n);
    for (int i = 0; i < n; i++) scanf("%d%d%d", &a[i], &b[i], &c[i]);
    double L = 0.0, R = 1000.0;  // 搜索区间[0,1000]
    for (int i = 0; i < 100; i++) {  // 迭代100次确保精度
      double m1 = L + (R - L) / 3;  // 左三分点
      double m2 = R - (R - L) / 3;  // 右三分点
      if (F(m1) < F(m2)) R = m2;   // 最小值在左侧，收缩右边界
      else L = m1;                  // 最小值在右侧，收缩左边界
    }
    printf("%.4lf\n", F(L));  // 输出最小误差
  }
  return 0;
}
```

## 例题36  解方程（Solve It!, UVa 10341）

### 题目描述
解方程：p·e^{-x} + q·sin(x) + r·cos(x) + s·tan(x) + t·x² + u = 0，在x∈[0, 1]范围内求解（最多一个解）。系数p,q,r,s,t,u都是整数（0 ≤ p,r ≤ 20，-20 ≤ q,s,t ≤ 0）。

**输入**：多组数据，每组一行6个整数p,q,r,s,t,u。

**输出**：如果[0,1]内有解，输出解保留4位小数；否则输出"No solution"。

### 解题思路

**单调性分析**：
定义F(x) = p·e^{-x} + q·sin(x) + r·cos(x) + s·tan(x) + t·x² + u

分析各分量的单调性：
- p·e^{-x}递减（导数为-p·e^{-x}≤0，p≥0）
- q·sin(x)递减（q≤0）
- r·cos(x)递减（导数为-r·sin(x)，r≥0，sin(x)≥0在[0,1]上）
- s·tan(x)递减（s≤0，tan在[0,1]上递增）
- t·x²递减（t≤0）

所以F(x)在[0,1]上是严格单调递减的。因此最多有一个根。

**二分法求解**：
检查F(0)和F(1)：
- 如果F(0) < -eps 或 F(1) > eps → 无解（根不在[0,1]内）
- 否则，在[0,1]上二分：如果F(mid) < 0，根在左侧(y=mid)；否则根在右侧(x=mid)
  迭代约100次即可收敛。

### 算法方法
- **数值方法/二分法**：利用函数的单调性
- **微积分**：分析各项导数的符号确认单调性

### 复杂度分析
- **时间复杂度**：O(iter·T)，iter≈100
- **空间复杂度**：O(1)

```cpp
// 例题36  解方程（Solve It!, UVa 10341）
// 刘汝佳
#include<cstdio>
#include<cmath>
#include <iostream>
#define F(x) (p*exp(-x)+q*sin(x)+r*cos(x)+s*tan(x)+t*(x)*(x)+u)
using namespace std;
const double eps = 1e-14;
int main() {
  for(int p, r, q, s, t, u; cin>>p>>q>>r>>s>>t>>u; ) {
    double f0 = F(0), f1 = F(1);
    if(f1 > eps || f0 < -eps) {
      puts("No solution");
      continue;
    }
    double x = 0, y = 1, m;
    for(int i = 0; i < 100; i++) {
      m = x + (y-x)/2;
      if(F(m) < 0) y = m; else x = m;
    }
    printf("%.4lf\n", m);
  }
  return 0;
}
```

## 例题38  桥上的绳索（Bridge, Hangzhou 2005, UVa1356）

### 题目描述
一座桥由若干段等宽的桥墩和桥面组成。给定总跨距D（桥墩宽度），总长度B，桥面下垂高度H（最大凹陷处），以及绳索总长度L。桥面形状是抛物线y = a·x²。需要计算桥面从最高点到最低点的竖直距离。桥面被n段等宽的桥墩支撑（n = ⌈B/D⌉），每段的绳索长度相等（L1 = L/n）。

**输入**：第一行T。每组一行D, H, B, L。

**输出**：对于每组数据，输出"Case X:\n"和桥面的竖直落差，保留2位小数。

### 解题思路

**抛物线弧长公式**：
对于抛物线y = a·x²，参数a满足在[-w/2, w/2]上最大高度为h，即a = 4h/w²。

抛物线弧长公式：
∫√(1 + (dy/dx)²) dx = ∫√(1 + (2a·x)²) dx

原函数：
F(a, x) = (x·√(a²+x²) + a²·ln|x+√(a²+x²)|)/2

其中这里的a参数是1/(2a)（抛物线的参数化）。

单段弧长 = 4a · (F(b, w/2) - F(b, 0))，其中b = 1/(2a) = w²/(8h)。

**二分法求解h**：
给定弧长L1和宽度D1，需要求高度h。弧长是h的单调递增函数，可以使用二分法求解h。

然后在已知h后计算最终答案H-h。

### 算法方法
- **数值方法/二分法**：利用弧长关于高度的单调性
- **微积分**：抛物线弧长的解析积分

### 复杂度分析
- **时间复杂度**：O(T·iter)，每次二分约50次迭代
- **空间复杂度**：O(1)

```cpp
// 例题38  桥上的绳索（Bridge, Hangzhou 2005, UVa1356）
// 刘汝佳
#include <cmath>
#include <cstdio>

// sqrt(a^2+x^2)的原函数
double F(double a, double x) {
  double a2 = a * a, x2 = x * x;
  return (x * sqrt(a2 + x2) + a2 * log(fabs(x + sqrt(a2 + x2)))) / 2;
}

// 宽度为w，高度为h的抛物线长度，也就是前文中的p(w,h)
double parabola_arc_length(double w, double h) {
  double a = 4.0 * h / (w * w), b = 1.0 / (2 * a);
  // 如果不用对称性，就是(F(b,w/2)-F(b,-w/2))*2*a
  return (F(b, w / 2) - F(b, 0)) * 4 * a;
}

int main() {
  int T;
  scanf("%d", &T);
  for (int kase = 1, D, H, B, L; kase <= T; kase++) {
    scanf("%d%d%d%d", &D, &H, &B, &L);
    int n = (B + D - 1) / D;  // 间隔数
    double D1 = (double)B / n, L1 = (double)L / n, x = 0, y = H;
    while (y - x > 1e-5) {  // 二分法求解高度
      double m = x + (y - x) / 2;
      if (parabola_arc_length(D1, m) < L1) x = m;
      else y = m;
    }
    if (kase > 1) puts("");
    printf("Case %d:\n%.2lf\n", kase, H - x);
  }
  return 0;
}
// 25877139 	1356 	Bridge 	Accepted 	C++ 	0.000 	2020-12-23 03:17:10
```
