# 06 - 位运算

> 用0和1直接操作数据，效率极高，面试/竞赛常考。

## 一、基础知识

### 1.1 核心定义

**位运算**直接操作整数的二进制表示中的每一位。计算机底层所有数据都是二进制存储的，位运算跳过高级抽象，直接在 bit 级别执行操作，是现代 CPU 上最快的指令。

六种基本位运算：AND (`&`)、OR (`|`)、XOR (`^`)、NOT (`!`)、左移 (`<<`)、右移 (`>>`)。

### 1.2 工作原理

位运算是按位独立的：两个整数的每一位独立运算，互不影响。

- **AND `&`**：两位都为 1 结果才为 1 → 用于"掩码"（提取特定位）、判断奇偶（`x & 1`）
- **OR `|`**：任一位为 1 结果即为 1 → 用于"设置某位为 1"
- **XOR `^`**：不同为 1，相同为 0 → 性质：`a ^ a = 0`, `a ^ 0 = a`。用于找唯一出现一次的数、交换两变量
- **左移 `<<`**：等价于乘 2 的幂次 → `1 << k` 创建只有第 k 位为 1 的掩码
- **右移 `>>`**：等价于除以 2 的幂次 → 遍历二进制位

核心技巧：`n & (n - 1)` 消除最低位的 1；`n & -n` 只保留最低位的 1（lowbit）。

### 1.3 适用场景与前提

| 场景 | 技术 | 例子 |
|------|------|------|
| 子集枚举 | `mask` 遍历 0..(1<<n) | 状压 DP 中枚举所有子集 |
| 状态压缩 | 用 int 的位表示集合 | 已访问城市、已选物品 |
| 唯一出现次数 | XOR 异或 | 数组中有唯一出现奇数次的数 |
| 快速判断奇偶 | `n & 1` | 比 `n % 2` 更快 |
| 2 的幂判断 | `n > 0 && (n & (n-1)) == 0` | 二进制只有一位为 1 |
| 位计数 | `n.count_ones()` | Rust 内置 |

**前提**：数据范围必须在整数类型的位数内（通常 ≤32 或 ≤64）。位运算不能直接用于浮点数或字符串。

### 1.4 优缺分析

**优点：**
- CPU 原生指令，速度极快（单时钟周期），是最高效的操作
- 状态压缩用 O(1) 空间表示集合，允许指数级枚举
- XOR 的对称性和自反性让许多数学题一步出解

**缺点：**
- 不直观，代码可读性差——`n & (n-1)` 不写注释别人看不懂
- 位数限制（64 位上限），无法处理超大数据
- 优先级坑：`&` 和 `|` 优先级低于 `==`，常常需要加括号
- 右移负数在 Rust 中是算术右移（保留符号），但不同语言行为不同

## 二、算法本质

**大白话：位运算就是在二进制层面操作数字。计算机底层就是0和1，位运算比加减乘除还快。**

数字在计算机中都是二进制的：
```
十进制 5  = 二进制 0101
十进制 3  = 二进制 0011
十进制 13 = 二进制 1101
```

## 三、六种基本位运算

| 运算 | 符号 | 规则 | 例子（5和3） |
|------|------|------|------------|
| 与 AND | `&` | 都是1才是1 | `0101 & 0011 = 0001` (1) |
| 或 OR | `\|` | 有1就是1 | `0101 \| 0011 = 0111` (7) |
| 异或 XOR | `^` | 不同为1 | `0101 ^ 0011 = 0110` (6) |
| 取反 NOT | `!` | 0变1，1变0 | `!0101 = ...1010` |
| 左移 | `<<` | 所有位左移 | `0101 << 1 = 1010` (10) |
| 右移 | `>>` | 所有位右移 | `0101 >> 1 = 0010` (2) |

### 直觉记忆：
- **AND `&`**：两个人都同意才通过 → "都1才1"
- **OR `|`**：一个人同意就通过 → "有1就1"
- **XOR `^`**：两个人意见不同才通过 → "不同为1"
- **左移 `<< n`**：相当于 × 2^n
- **右移 `>> n`**：相当于 ÷ 2^n

## 四、常用位运算技巧

### 3.1 判断奇偶

```rust
// 最后一位是1就是奇数，是0就是偶数
fn is_odd(n: i32) -> bool {
    n & 1 == 1  // 比 n % 2 != 0 更快
}
```

### 3.2 获取/设置/清除某一位

```rust
let n: i32 = 0b1101; // 13，二进制 1101
let i = 2; // 操作第2位（从右数，0开始）

// 获取第 i 位的值（0 或 1）
let bit_i = (n >> i) & 1;    // 13 >> 2 = 0b11, & 1 = 1

// 设置第 i 位为 1
let set = n | (1 << i);      // 13 | 4 = 0b1101 | 0b0100 = 0b1101 = 13

// 清除第 i 位（设为 0）
let clear = n & !(1 << i);   // 13 & !4 = 0b1101 & 0b...1011 = 0b1001 = 9

// 翻转第 i 位
let flip = n ^ (1 << i);     // 13 ^ 4 = 0b1101 ^ 0b0100 = 0b1001 = 9
```

### 3.3 最低位的1（lowbit）

```rust
// 获取最低位的1：n & (-n)  或  n & (!n + 1)
// 例如：n = 0b1100，lowbit = 0b0100
fn lowbit(n: i32) -> i32 {
    n & (-n)  // Rust 中 i32 负数用补码表示
    // ⚠️ 注意：当 n = i32::MIN 时，-n 在 debug 模式下会 panic（溢出）
    // 如果数据范围包含 i32::MIN，请使用 wrapping_neg() 或将类型改为 u32
}
```

> 💡 **为什么 `n & (-n)` 等于 lowbit？** 这依赖于**补码**（Two's Complement）的性质。
> 
> 补码规则：`-n = !n + 1`（按位取反再加一）。
> 
> 以 `n = 0b1100` 为例：
> - `n = 0b...1100`（最低位的 1 在 bit2）
> - `!n = 0b...0011`（全部取反）
> - `!n + 1 = 0b...0100`（加一后，bit2 变回 1，低位置零）
> - `n & (-n) = 0b1100 & 0b0100 = 0b0100`（只有最低位的 1 保留）
> 
> 关键：`!n` 把最低位 1 变 0，加 1 后进位又把它变回 1，而低位的 0 无法进位。所以只有最低位 1 及以下会"恢复"。

### 3.4 去掉最低位的1

```rust
// n & (n - 1)：去掉最低位的1
// 例如：n = 0b1100，结果 = 0b1000
let n = n & (n - 1);
```

**应用：数1的个数（LeetCode 191）**
```rust
fn hamming_weight(mut n: u32) -> i32 {
    let mut count = 0;
    while n != 0 {
        n &= n - 1; // 每次去掉一个1
        count += 1;
    }
    count
}
```

### 3.5 XOR 的三大性质

```
a ^ a = 0      （自己异或自己等于0）
a ^ 0 = a      （异或0不变）
a ^ b ^ a = b  （异或两次相当于抵消）
```

**经典应用：找只出现一次的数（LeetCode 136）**
```rust
fn single_number(nums: Vec<i32>) -> i32 {
    nums.iter().fold(0, |acc, &x| acc ^ x)
    // 出现两次的数异或后抵消，剩下出现一次的
}
```

## 五、位掩码（Bitmask）

**大白话：用一个整数的每一位表示一个"开关"状态。**

**你已经用过了！**（5/26 和 5/27 的位运算题目）

### 4.1 用 bitmask 表示集合

```rust
// 用一个 u32 表示 26 个字母的出现情况
let mut mask: u32 = 0;

// 添加字母 'c'（第2位）
mask |= 1 << (b'c' - b'a');

// 检查字母 'c' 是否存在
let has_c = mask & (1 << (b'c' - b'a')) != 0;

// 删除字母 'c'
mask &= !(1 << (b'c' - b'a'));

// 集合大小（1的个数）
let size = mask.count_ones();
```

### 4.2 枚举子集

```rust
// 枚举 mask 的所有子集（包括空集）
let mask: u32 = 0b1011; // {0, 1, 3}
let mut sub = mask;
loop {
    // 处理子集 sub
    println!("{:04b}", sub);
    if sub == 0 { break; }
    sub = (sub - 1) & mask;
}
// 输出: 1011, 1010, 1001, 1000, 0011, 0010, 0001, 0000
```

### 4.3 枚举所有大小为 k 的子集

```rust
// 从 n 个元素中选 k 个的所有组合
fn enumerate_combinations(n: usize, k: usize) {
    let mut mask: u32 = (1 << k) - 1; // 初始：最低 k 位全是1
    while mask < (1 << n) {
        // 处理 mask
        println!("{:0width$b}", mask, width = n);
        // Gosper's hack：生成下一个相同 popcount 的数
        let c = mask & (!mask + 1); // lowbit
        let r = mask + c;
        mask = (((r ^ mask) >> 2) / c) | r;
    }
}
```

## 六、经典例题

### 例题 1：LeetCode 137. 只出现一次的数字 II

> 给定数组 `nums`，除某个元素仅出现一次外，其余每个元素都恰出现三次。找出那个只出现一次的元素。要求线性时间、常数空间。
>
> **示例：** `nums = [2,2,3,2]` → `3`；`nums = [0,1,0,1,0,1,99]` → `99`
> **约束：** `1 <= nums.length <= 3*10^4`, `-2^31 <= nums[i] <= 2^31-1`

```rust
fn single_number(nums: Vec<i32>) -> i32 {
    let mut ans = 0i32;
    for bit in 0..32 {
        let sum: i32 = nums.iter().map(|&x| (x >> bit) & 1).sum();
        if sum % 3 != 0 {
            ans |= 1 << bit;
        }
    }
    ans
}
```

**思路：** 逐位统计。如果某一位上1的个数不是3的倍数，说明那个唯一的数在这一位上是1。

### 例题 2：LeetCode 78. 子集

> 给定不含重复元素的数组 `nums`，返回所有可能的子集（幂集），可按任意顺序。
>
> **示例：** `nums = [1,2,3]` → `[[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]`
> **约束：** `1 <= nums.length <= 10`, `-10 <= nums[i] <= 10`

```rust
fn subsets(nums: Vec<i32>) -> Vec<Vec<i32>> {
    let n = nums.len();
    let mut ans = vec![];
    for mask in 0..(1u32 << n) {
        let mut subset = vec![];
        for i in 0..n {
            if mask & (1 << i) != 0 {
                subset.push(nums[i]);
            }
        }
        ans.push(subset);
    }
    ans
}
```

**思路：** n 个元素，每个元素"选"或"不选"，正好 2^n 种情况，用 0 到 2^n-1 的二进制表示。

### 例题 3：判断是否是2的幂（LeetCode 231）

```rust
fn is_power_of_two(n: i32) -> bool {
    n > 0 && n & (n - 1) == 0
    // 2的幂的二进制只有一个1
    // n & (n-1) 去掉唯一的1后变成0
}
```

## 七、测试题

1. **问：** 为什么 `n & (n-1)` 能去掉最低位的1？

   **答：** `n-1` 会把最低位的1变成0，它右边的0全变成1。与 n 做 AND 后，最低位的1和它右边的位全部清零。例如 `1100 & 1011 = 1000`。

2. **问：** 如何快速判断两个数是否只有一位不同？

   **答：** `(a ^ b)` 只有不同位为1，然后判断 `(a^b) & ((a^b)-1) == 0`（只有一个1）。
