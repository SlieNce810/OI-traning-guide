# 16 - 高级数据结构

> 线段树、树状数组、并查集——竞赛和面试进阶的利器。

## 一、基础知识

### 1.1 核心定义

本文涵盖三种支撑高级算法竞赛的数据结构：

- **并查集**（Union-Find / DSU）：管理不相交集合的合并与查询，近乎 O(1) 的均摊时间
- **树状数组**（Fenwick Tree / BIT）：O(log n) 单点更新 + 前缀查询，代码极短
- **线段树**（Segment Tree）：O(log n) 区间查询 + 区间更新，功能最强大但也最复杂

### 1.2 工作原理

- **并查集**：每个集合用一棵树表示，`find(x)` 找根节点并路径压缩，`union(x,y)` 按秩合并。经过优化后均摊 O(α(n)) ≈ O(1)。
- **树状数组**：利用 lowbit 性质，`tree[i]` 存储区间 `[i-lowbit(i)+1, i]` 的信息。更新时沿 `i += lowbit(i)` 向上，查询时沿 `i -= lowbit(i)` 向下。
- **线段树**：二叉树每个节点代表一个区间，mid 分割左右子区间。懒标记（lazy propagation）支持区间修改。

### 1.3 适用场景与前提

| 场景 | 数据结构 | 时间复杂度 |
|------|---------|-----------|
| 动态连通性 | 并查集 | 近似 O(1) |
| 区间求和/前缀和 | 树状数组 | O(log n) |
| 区间最大/最小值 | 线段树 | O(log n) |
| 区间加 + 区间求和 | 线段树（懒标记） | O(log n) |
| 逆序对统计 | 树状数组 | O(n log n) |

**前提**：树状数组和线段树要求操作可合并（如 sum、max 满足结合律）。线段树空间 O(4n)，需警惕 n 较大时的内存。

### 1.4 优缺分析

**优点：**
- 并查集代码极短（5 行），功能强大，几乎无代价
- 树状数组常数小、代码少，是前缀问题的首选
- 线段树功能最全面，区间修改/查询一条龙

**缺点：**
- 线段树代码量较大（50+ 行），调试困难
- 树状数组只支持前缀操作和可逆运算（如 sum），不支持 max 的区间覆盖
- 并查集不支持删除操作（只有合并，没有拆分）
- Rust 中递归线段树有栈深度风险（需迭代或限制 n）

## 二、并查集 (Union-Find)

### 算法本质

**大白话：并查集管理一堆"朋友圈"——能快速回答"A和B是不是一个圈子的？"和"把A的圈子和B的圈子合并"。**

```rust
struct UnionFind {
    parent: Vec<usize>,
    rank: Vec<usize>,
    count: usize, // 连通分量数
}

impl UnionFind {
    fn new(n: usize) -> Self {
        UnionFind {
            parent: (0..n).collect(), // 初始时每个人是自己的老大
            rank: vec![0; n],
            count: n,
        }
    }

    /// 找到 x 的根（老大）—— 带路径压缩
    fn find(&mut self, x: usize) -> usize {
        if self.parent[x] != x {
            self.parent[x] = self.find(self.parent[x]); // 路径压缩
        }
        self.parent[x]
    }

    /// 合并 x 和 y 所在的集合 —— 按秩合并
    fn union(&mut self, x: usize, y: usize) -> bool {
        let (rx, ry) = (self.find(x), self.find(y));
        if rx == ry { return false; } // 已经在同一集合

        // 按秩合并：矮树挂在高树上
        match self.rank[rx].cmp(&self.rank[ry]) {
            std::cmp::Ordering::Less => self.parent[rx] = ry,
            std::cmp::Ordering::Greater => self.parent[ry] = rx,
            std::cmp::Ordering::Equal => {
                self.parent[ry] = rx;
                self.rank[rx] += 1;
            }
        }
        self.count -= 1;
        true
    }

    /// 判断是否连通
    fn connected(&mut self, x: usize, y: usize) -> bool {
        self.find(x) == self.find(y)
    }
}
```

**两个优化的直觉：**
- **路径压缩**：查找时把沿途节点都直接挂到根上，下次查找更快
- **按秩合并**：总是把小树挂到大树上，保持树的平衡

### 例题：省份数量（LeetCode 547）

```rust
fn find_circle_num(is_connected: Vec<Vec<i32>>) -> i32 {
    let n = is_connected.len();
    let mut uf = UnionFind::new(n);

    for i in 0..n {
        for j in i + 1..n {
            if is_connected[i][j] == 1 {
                uf.union(i, j);
            }
        }
    }
    uf.count as i32
}
```

### 例题：冗余连接（LeetCode 684）

> 找到使图成环的那条边。

```rust
fn find_redundant_connection(edges: Vec<Vec<i32>>) -> Vec<i32> {
    let n = edges.len();
    let mut uf = UnionFind::new(n + 1);

    for edge in &edges {
        let (u, v) = (edge[0] as usize, edge[1] as usize);
        if !uf.union(u, v) {
            return edge.clone(); // 已经连通了，这条边是冗余的
        }
    }
    vec![]
}
```

## 三、树状数组 (Binary Indexed Tree / Fenwick Tree)

### 算法本质

**大白话：高效支持"单点修改"和"前缀求和"的数据结构。**

| 操作 | 普通数组 | 前缀和数组 | 树状数组 |
|------|---------|-----------|---------|
| 单点修改 | O(1) | O(n) | **O(log n)** |
| 前缀查询 | O(n) | O(1) | **O(log n)** |

树状数组在两种操作之间取了平衡。

**lowbit(i) 是什么？** `i & i.wrapping_neg()` 提取 i 的二进制最低位的1。
例如 `lowbit(6) = lowbit(0b110) = 0b010 = 2`。树状数组用 lowbit 来确定每个位置负责管理的区间范围：
- tree[1] 管 [1,1]，tree[2] 管 [1,2]，tree[4] 管 [1,4]，tree[6] 管 [5,6]
- `wrapping_neg()` 是因为 i 是 usize（无符号），不能直接写 `-i`

```rust
struct BIT {
    tree: Vec<i64>,
    n: usize,
}

impl BIT {
    fn new(n: usize) -> Self {
        BIT { tree: vec![0; n + 1], n }
    }

    /// 单点更新：在位置 i 加上 val
    fn update(&mut self, mut i: usize, val: i64) {
        while i <= self.n {
            self.tree[i] += val;
            i += i & i.wrapping_neg(); // i += lowbit(i)
        }
    }

    /// 前缀查询：求 [1, i] 的和
    fn query(&self, mut i: usize) -> i64 {
        let mut sum = 0;
        while i > 0 {
            sum += self.tree[i];
            i -= i & i.wrapping_neg(); // i -= lowbit(i)
        }
        sum
    }

    /// 区间查询：求 [l, r] 的和
    fn range_query(&self, l: usize, r: usize) -> i64 {
        if l == 0 { self.query(r) } else { self.query(r) - self.query(l - 1) }
        // 注意：BIT 使用 1-indexed，l == 0 时不能做 l-1（usize 下溢 panic）
    }
}
```

### 经典应用：逆序对计数

```rust
fn count_inversions(nums: &[i32]) -> i64 {
    // 离散化
    let mut sorted = nums.to_vec();
    sorted.sort();
    sorted.dedup();

    let mut bit = BIT::new(sorted.len());
    let mut count = 0i64;

    for &num in nums {
        let rank = sorted.partition_point(|&x| x < num) + 1;
        // 在 rank 右边的已插入的数都构成逆序对
        count += bit.query(sorted.len()) - bit.query(rank);
        bit.update(rank, 1);
    }
    count
}
```

## 四、线段树 (Segment Tree)

### 算法本质

**大白话：线段树能在 O(log n) 时间内完成"区间查询"和"区间修改"。**

比树状数组功能更强，但代码更复杂。

```rust
struct SegTree {
    n: usize,
    tree: Vec<i64>,  // 区间和
    lazy: Vec<i64>,  // 懒标记
}

impl SegTree {
    fn new(n: usize) -> Self {
        SegTree {
            n,
            tree: vec![0; 4 * n],
            lazy: vec![0; 4 * n],
        }
    }

    fn push_down(&mut self, node: usize, left_size: usize, right_size: usize) {
        if self.lazy[node] != 0 {
            self.tree[node * 2] += self.lazy[node] * left_size as i64;
            self.tree[node * 2 + 1] += self.lazy[node] * right_size as i64;
            self.lazy[node * 2] += self.lazy[node];
            self.lazy[node * 2 + 1] += self.lazy[node];
            self.lazy[node] = 0;
        }
    }

    /// 区间修改：[l, r] 每个元素加 val
    fn update(&mut self, node: usize, start: usize, end: usize, l: usize, r: usize, val: i64) {
        if l <= start && end <= r {
            self.tree[node] += val * (end - start + 1) as i64;
            self.lazy[node] += val;
            return;
        }
        let mid = (start + end) / 2;
        self.push_down(node, mid - start + 1, end - mid);
        if l <= mid { self.update(node * 2, start, mid, l, r, val); }
        if r > mid { self.update(node * 2 + 1, mid + 1, end, l, r, val); }
        self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1];
    }

    /// 区间查询：[l, r] 的和
    fn query(&mut self, node: usize, start: usize, end: usize, l: usize, r: usize) -> i64 {
        if l <= start && end <= r {
            return self.tree[node];
        }
        let mid = (start + end) / 2;
        self.push_down(node, mid - start + 1, end - mid);
        let mut result = 0;
        if l <= mid { result += self.query(node * 2, start, mid, l, r); }
        if r > mid { result += self.query(node * 2 + 1, mid + 1, end, l, r); }
        result
    }
}
```

### 什么时候用什么？

| 需求 | 推荐数据结构 |
|------|------------|
| 单点修改 + 前缀和 | 树状数组 |
| 区间修改 + 区间查询 | 线段树 |
| 连通性查询/合并 | 并查集 |

> 💡 **线段树的懒标记（Lazy Propagation）直觉**：
> 
> 懒标记的核心思想是**延迟更新**——当修改一个区间时，如果当前节点完全被区间覆盖，我们只更新这个节点的值和懒标记，不往下递归更新子节点。真正的更新等到下次查询或修改需要访问子节点时才"下推"（push_down）。
> 
> **类比**：就像老板让你更新 100 份报告。如果你每份都立刻去改，要跑 100 趟。聪明的做法是贴一张便条（懒标记）说"报告 1-100 都要加 10"，等有人要看具体某份报告时再更新那一份。这样大部分报告可能永远不需要单独更新。
> 
> **线段树 vs 树状数组的选择**：树状数组只能处理**可逆**的前缀操作（如 sum：`sum(l,r) = sum(r) - sum(l-1)`），不能处理 **max/min**（因为 max(l,r) ≠ max(r) - max(l-1)）。如果要区间 max/min，必须用线段树。

## 五、测试题

1. **问：** 并查集的路径压缩和按秩合并一起用，时间复杂度是多少？

   **答：** 接近 O(1)（准确说是 O(α(n))，α 是反阿克曼函数，增长极慢，对所有实际输入都 ≤ 5）。

2. **问：** 树状数组和线段树的区别？

   **答：** 树状数组代码简单、常数小，但只支持"前缀操作"。线段树更通用，支持区间修改、区间查询，但代码复杂。
