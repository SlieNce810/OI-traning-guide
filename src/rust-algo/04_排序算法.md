# 04 - 排序算法

> 排序是算法的基石，很多高级算法的第一步就是排序。

## 一、基础知识

### 1.1 核心定义

**排序算法**将一组元素按特定顺序（升序或降序）重新排列。在算法竞赛和面试中，排序不仅是独立的题目，更是前缀和的预处理、二分的前提条件、贪心策略的基础步骤。

两个关键概念：
- **稳定排序**：相等元素的相对顺序在排序前后不变（如 Rust 的 `.sort()`）
- **原地排序**：不需要额外分配 O(n) 空间的排序（如快排、堆排）

### 1.2 工作原理

排序算法的本质是**比较与交换**。不同算法在"如何减少比较次数"上各有策略：

- **归并排序**：分治法——递归二分数组，合并两个有序序列
- **快速排序**：分治法——选基准 pivot，将数组分为"小于 pivot"和"大于 pivot"两部分
- **堆排序**：维护一个堆结构，每次弹出最值
- **计数排序**：不基于比较——统计每个值出现次数，直接构造有序结果

复杂度下界定理：任何基于比较的排序算法，最坏情况下**至少需要 Ω(n log n)** 次比较（下界）。归并排序和堆排序达到了上界 O(n log n)，因此比较排序的最优复杂度是 **Θ(n log n)**。

### 1.3 适用场景与前提

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| 通用排序（刷题默认） | `.sort_unstable()` | 最快的内置排序 |
| 要求稳定 | `.sort()` | 基于 Timsort（归并+插入） |
| 数据范围很小（≤10⁴） | 计数排序 | O(n+k)，极快 |
| 求 Top-K | 大小为 K 的小顶堆 | 时间复杂度 O(n log k) |
| 求逆序对 | 归并排序 | 合并过程天然可统计 |
| 自定义排序 | `.sort_by()` / `.sort_by_key()` | 区间合并、按多个关键字排序 |
| 链表排序 | 归并排序 | 链表不能随机访问，快排不合适 |

**前提条件**：排序要求元素之间可比较（实现了 `Ord` trait）。对于浮点数，需要用 `partial_cmp` 处理 NaN。

### 1.4 优缺分析

**优点：**
- 排序是解决一堆问题的万能第一步——"先排序，再用双指针/贪心/二分"是经典模式
- Rust 的标准排序经过高度优化（Timsort），大多数场景直接调库即可
- 稳定排序保留原始顺序信息，适合多关键字排序

**缺点：**
- 基于比较的排序 O(n log n) 是理论下界，无法突破
- 排序本身 O(n log n)，可能成为算法的瓶颈（如滑动窗口中每次收缩都重新排序不可行）
- 排序会破坏原数组的索引信息；需要保留原始位置时，必须额外存储索引后再排序
- 不稳定排序可能改变业务逻辑（如对 struct 按某一字段排序，其他字段顺序被破坏）

## 二、算法本质

**大白话：排序就是把一堆乱七八糟的数据按照某种规则排好队。**

你可能想说"Rust 不是有 `.sort()` 吗？为什么要学排序？"因为：
1. 面试/竞赛可能考排序的原理
2. 理解排序原理能帮你理解更高级的算法（如归并排序 → 逆序对计数）
3. **自定义排序**是刷题中超高频的技巧

## 三、常见排序算法对比

| 算法 | 时间复杂度（平均） | 空间 | 稳定性 | 特点 |
|------|-----------------|------|--------|------|
| 冒泡排序 | O(n²) | O(1) | 稳定 | 简单但慢，不实用 |
| 选择排序 | O(n²) | O(1) | 不稳定 | 简单但慢 |
| 插入排序 | O(n²) | O(1) | 稳定 | 小数据量快 |
| 归并排序 | O(n log n) | O(n) | 稳定 | 分治思想，可求逆序对 |
| 快速排序 | O(n log n) | O(log n) | 不稳定 | 实际最快，Rust默认用 |
| 堆排序 | O(n log n) | O(1) | 不稳定 | 可求 Top-K |
| 计数排序 | O(n+k) | O(k) | 稳定 | 只能排整数，范围小时极快 |

> ⚠️ **各算法最坏时间复杂度**：快速排序最坏 O(n²)（每次 pivot 都是最值，如已排序数组直接取末尾做 pivot）；堆排序、归并排序在任何情况下都是 O(n log n)。

**稳定性是什么意思？** 相同值的元素排序后相对顺序不变就是稳定的。

> 💡 **稳定性实例**：假设按分数排序学生记录 `[(张三,90), (李四,85), (王五,90)]`。
> - 稳定排序后：`[(李四,85), (张三,90), (王五,90)]` —— 张三仍在王五前面（保持原始顺序）
> - 不稳定排序后：可能是 `[(李四,85), (王五,90), (张三,90)]` —— 同名次者顺序变了
> 
> **什么时候必须用稳定排序？** 多关键字排序时——先按第二关键字排，再按第一关键字稳定排，就能得到正确结果。
> - 稳定排序：Rust 的 `.sort()`
> - 不稳定排序：Rust 的 `.sort_unstable()`（更快一点）

## 四、重点掌握的排序

### 3.1 归并排序（分治思想的代表）

**核心思想：把数组分成两半，分别排好序，再合并。**

```
[3, 1, 4, 1, 5, 9]
       ↙      ↘
[3, 1, 4]  [1, 5, 9]     ← 分
  ↙   ↘      ↙   ↘
[3] [1,4]  [1] [5,9]     ← 继续分
     ↓          ↓
[1,3,4]    [1,5,9]       ← 合并（merge）
       ↘  ↙
[1, 1, 3, 4, 5, 9]       ← 最终合并
```

```rust
fn merge_sort(arr: &mut [i32]) {
    let n = arr.len();
    if n <= 1 { return; }

    let mid = n / 2;
    merge_sort(&mut arr[..mid]);   // 排左半
    merge_sort(&mut arr[mid..]);   // 排右半

    // 合并两个有序数组
    let left = arr[..mid].to_vec();
    let right = arr[mid..].to_vec();
    let (mut i, mut j, mut k) = (0, 0, 0);
    while i < left.len() && j < right.len() {
        if left[i] <= right[j] {
            arr[k] = left[i];
            i += 1;
        } else {
            arr[k] = right[j];
            j += 1;
        }
        k += 1;
    }
    while i < left.len() { arr[k] = left[i]; i += 1; k += 1; }
    while j < right.len() { arr[k] = right[j]; j += 1; k += 1; }
}
```

**归并排序的延伸应用：求逆序对数量**（LeetCode 剑指 Offer 51）

### 3.2 快速排序

**核心思想：选一个"基准"，比它小的放左边，比它大的放右边，然后递归处理。**

```rust
fn quick_sort(arr: &mut [i32]) {
    if arr.len() <= 1 { return; }
    let pivot_idx = partition(arr);
    quick_sort(&mut arr[..pivot_idx]);
    quick_sort(&mut arr[pivot_idx + 1..]);
}

fn partition(arr: &mut [i32]) -> usize {
    let pivot = arr[arr.len() - 1]; // 选最后一个做基准
    let mut i = 0; // i 指向"小于区"的下一个位置
    for j in 0..arr.len() - 1 {
        if arr[j] <= pivot {
            arr.swap(i, j);
            i += 1;
        }
    }
    arr.swap(i, arr.len() - 1);
    i
}
```

## 五、Rust 中的排序（刷题必备）

### 4.1 基础排序

```rust
let mut v = vec![3, 1, 4, 1, 5, 9];

v.sort();                // 升序（稳定排序）
v.sort_unstable();       // 升序（不稳定但更快）
v.sort_by(|a, b| b.cmp(a)); // 降序
```

### 4.2 自定义排序（超高频！）

```rust
// 按绝对值排序
let mut v = vec![-3, 1, -4, 1, 5];
v.sort_by_key(|&x| x.abs());
// 结果: [1, 1, -3, -4, 5]

// 按第二个元素排序（二维数组/元组）
let mut intervals = vec![(3, 4), (1, 2), (2, 6)];
intervals.sort_by_key(|&(_, end)| end); // 按结束时间排序
// 结果: [(1,2), (3,4), (2,6)]

// 多关键字排序：先按 x 升序，x 相同按 y 降序
let mut points = vec![(1, 3), (2, 1), (1, 5)];
points.sort_by(|a, b| {
    a.0.cmp(&b.0).then(b.1.cmp(&a.1))
});
// 结果: [(1,5), (1,3), (2,1)]
```

### 4.3 排序的经典技巧

**技巧 1：排序 + 去重**

```rust
let mut v = vec![3, 1, 4, 1, 5, 3];
v.sort();
v.dedup(); // 去除相邻重复元素
// 结果: [1, 3, 4, 5]
```

**技巧 2：按索引排序（不移动原数组）**

```rust
let arr = vec![30, 10, 40, 20];
let mut indices: Vec<usize> = (0..arr.len()).collect();
indices.sort_by_key(|&i| arr[i]);
// indices = [1, 3, 0, 2]，即 arr[1]=10 最小
```

**技巧 3：浮点数排序**

```rust
let mut v = vec![3.0_f64, 1.0, 4.0, 1.5];
v.sort_by(|a, b| a.partial_cmp(b).unwrap());
```

## 六、堆（优先队列）

**大白话：堆就是一个能快速取出最大/最小值的数据结构。**

Rust 的 `BinaryHeap` 是**大顶堆**（默认取最大值）。

```rust
use std::collections::BinaryHeap;
use std::cmp::Reverse;

// 大顶堆（默认）
let mut max_heap = BinaryHeap::new();
max_heap.push(3);
max_heap.push(1);
max_heap.push(4);
assert_eq!(max_heap.peek(), Some(&4)); // 最大值
assert_eq!(max_heap.pop(), Some(4));   // 取出最大值

// 小顶堆（用 Reverse 包装）
let mut min_heap = BinaryHeap::new();
min_heap.push(Reverse(3));
min_heap.push(Reverse(1));
min_heap.push(Reverse(4));
assert_eq!(min_heap.pop(), Some(Reverse(1))); // 取出最小值
```

### 经典应用：求第 K 大的元素（LeetCode 215）

```rust
use std::collections::BinaryHeap;
use std::cmp::Reverse;

fn find_kth_largest(nums: Vec<i32>, k: i32) -> i32 {
    // 维护一个大小为 k 的小顶堆
    let mut heap = BinaryHeap::new();
    for num in nums {
        heap.push(Reverse(num));
        if heap.len() > k as usize {
            heap.pop(); // 弹出最小的，保留 k 个最大的
        }
    }
    heap.peek().unwrap().0
}
```

## 七、例题详解

### 例题：LeetCode 56. 合并区间

> 以数组 `intervals` 表示区间集合，`intervals[i] = [start_i, end_i]`。合并所有重叠区间，返回不重叠的区间数组。
>
> **示例：** `intervals = [[1,3],[2,6],[8,10],[15,18]]` → `[[1,6],[8,10],[15,18]]`
> **约束：** `1 <= intervals.length <= 10^4`, `intervals[i].length == 2`

```rust
fn merge(mut intervals: Vec<Vec<i32>>) -> Vec<Vec<i32>> {
    intervals.sort_by_key(|v| v[0]); // 按起点排序
    let mut ans: Vec<Vec<i32>> = vec![];

    for interval in intervals {
        if let Some(last) = ans.last_mut() {
            if last[1] >= interval[0] {
                // 有重叠，合并
                last[1] = last[1].max(interval[1]);
                continue;
            }
        }
        ans.push(interval);
    }
    ans
}
```

**思路：** 排序后，如果当前区间的起点 <= 上一个区间的终点，就有重叠，合并。

## 八、测试题

1. **问：** `sort()` 和 `sort_unstable()` 的区别？

   **答：** `sort()` 是稳定排序（相同元素的相对顺序不变），`sort_unstable()` 不保证稳定但更快。刷题中如果不关心稳定性，优先用 `sort_unstable()`。

2. **问：** 如何对 `Vec<(i32, i32)>` 先按第一个元素升序，再按第二个元素降序？

   **答：** `v.sort_by(|a, b| a.0.cmp(&b.0).then(b.1.cmp(&a.1)));`
