# 09 - 链表

> Rust 的所有权让链表变得有趣（和痛苦），但掌握后你会更理解 Rust。

## 一、基础知识

### 1.1 核心定义

**链表**（Linked List）是一种线性数据结构，每个节点包含数据和一个指向下一个节点的指针（或引用）。与数组不同，链表的节点分散在内存中，通过指针串联。

Rust 中链表的标准定义为 `Option<Box<ListNode>>`：`Option` 表示"可能有下一个节点"，`Box` 将节点分配在堆上（递归类型必须用 Box 避免无限大小）。

### 1.2 工作原理

链表的基本操作围绕"改变指针指向"展开：

- **插入**：创建新节点，将它的 next 指向前驱的 next，再将前驱的 next 指向它 → O(1)
- **删除**：将前驱的 next 指向 `node.next.take()`，删除节点被自动释放 → O(1)
- **遍历**：从 head 开始，沿 next 链依次访问 → O(n)
- **随机访问**：需要从 head 开始数 → O(n)，这是链表相比数组的主要劣势

Rust 特有问题：所有权的移动。`.take()` 是 Rust 链表中最高频的方法——它从 Option 中"偷走"值，留下 None，避免了所有权冲突。

> 🔑 **`.take()` 详解**：`Option::take(&mut self) -> Option<T>` 将 `self` 中的值移出，并将 `self` 设为 `None`。等价于 `std::mem::replace(self, None)`。
> 
> ```rust
> // 为什么需要 take()？
> let mut node: Option<Box<ListNode>> = Some(Box::new(ListNode::new(1)));
> // 错误：不能从 &mut Option 中移出值
> // let next = node.unwrap().next;  // ❌ 编译错误
> 
> // 正确：先 take() 偷出所有权，原位置变成 None
> let next = node.as_mut().unwrap().next.take();  // ✅
> ```
> 
> **直觉**：就像你把钱包从口袋里拿出来——`take()` 掏出了钱，口袋空了（None）。你不能同时拥有一块钱的两个"拥有者"，Rust 的所有权规则不允许。

### 1.3 适用场景与前提

| 场景 | 原因 |
|------|------|
| 频繁头部插入/删除 | O(1)，数组需要 O(n) 移动元素 |
| 不确定数据量 | 不需要预分配，动态增长 |
| 两数相加（逆序存储） | 天然从低位向高位进位 |
| 合并有序链表 | 递归或迭代，指针操作简洁 |
| LRU 缓存 | 快速移动节点到头部 |

**前提**：需要承受 O(n) 的随机访问代价。任何需要 `list[i]` 的场景都不适合链表。

### 1.4 优缺分析

**优点：**
- 插入/删除 O(1)（给定节点位置），无需移动其他元素
- 动态大小，不需要预分配容量
- 节点分散存储，不受连续内存碎片影响

**缺点：**
- 随机访问 O(n)，无法二分查找
- 内存开销大——每个节点额外存储一个指针
- 缓存不友好——节点分散导致 CPU 缓存频繁失效
- Rust 中操作链表比 C++/Java 繁琐（`Box`、`Option`、`.take()`、`as_mut()`）

## 二、算法本质

**大白话：链表就是一串通过"指针"连起来的节点，像火车车厢一样一个接一个。**

和数组的区别：
- **数组**：连续内存，随机访问 O(1)，插入删除 O(n)
- **链表**：分散内存，随机访问 O(n)，插入删除 O(1)

## 三、LeetCode 中的链表定义

```rust
// LeetCode 标准链表定义
#[derive(PartialEq, Eq, Clone, Debug)]
pub struct ListNode {
    pub val: i32,
    pub next: Option<Box<ListNode>>,
}

impl ListNode {
    pub fn new(val: i32) -> Self {
        ListNode { val, next: None }
    }
}
```

**Option<Box<ListNode>>** 的含义：
- `Option`：可能有下一个节点，也可能没有（None = 链表结束）
- `Box`：把节点放在堆上（因为链表大小编译期不确定）

## 四、链表基本操作

### 3.1 创建链表

```rust
fn create_list(vals: &[i32]) -> Option<Box<ListNode>> {
    let mut head = None;
    for &val in vals.iter().rev() {
        let mut node = ListNode::new(val);
        node.next = head;
        head = Some(Box::new(node));
    }
    head
}
```

### 3.2 遍历链表

```rust
fn traverse(head: &Option<Box<ListNode>>) {
    let mut current = head;
    while let Some(node) = current {
        println!("{}", node.val);
        current = &node.next;
    }
}
```

## 五、经典链表题型

### 题型 1：反转链表（LeetCode 206）

**最经典的链表题，必须会！**

```rust
fn reverse_list(head: Option<Box<ListNode>>) -> Option<Box<ListNode>> {
    let mut prev = None;
    let mut current = head;

    while let Some(mut node) = current {
        current = node.next.take(); // 取出下一个节点
        node.next = prev;            // 当前节点指向前一个
        prev = Some(node);           // 前进
    }
    prev
}
```

**过程模拟（1 → 2 → 3）：**
```
初始:  prev=None, current=1→2→3
步骤1: prev=1→None, current=2→3
步骤2: prev=2→1→None, current=3
步骤3: prev=3→2→1→None, current=None
结果:  3→2→1
```

### 题型 2：合并两个有序链表（LeetCode 21）

```rust
fn merge_two_lists(
    list1: Option<Box<ListNode>>,
    list2: Option<Box<ListNode>>,
) -> Option<Box<ListNode>> {
    match (list1, list2) {
        (None, None) => None,
        (Some(n), None) | (None, Some(n)) => Some(n),
        (Some(mut l1), Some(mut l2)) => {
            if l1.val <= l2.val {
                l1.next = merge_two_lists(l1.next, Some(l2));
                Some(l1)
            } else {
                l2.next = merge_two_lists(Some(l1), l2.next);
                Some(l2)
            }
        }
    }
}
```

### 题型 3：检测环（LeetCode 141）

**快慢指针法（Floyd 判环法）**

算法思路：slow 每次走1步，fast 每次走2步。如果有环，两者一定会相遇；如果无环，fast 先到 None。

由于 Rust 所有权的限制，标准链表结构无法直接用快慢指针。LeetCode 上这题可以用 HashSet 存已访问节点来检测环：

```rust

// 思路：遍历链表，把每个节点的地址存入 HashSet

// Rust 中推荐用 HashSet 方案或将数据转为 Vec 处理
```

### 题型 4：找链表中间节点（LeetCode 876）

```rust
fn middle_node(head: Option<Box<ListNode>>) -> Option<Box<ListNode>> {
    let mut slow = &head;
    let mut fast = &head;

    while fast.is_some() && fast.as_ref().unwrap().next.is_some() {
        slow = &slow.as_ref().unwrap().next;
        fast = &fast.as_ref().unwrap().next.as_ref().unwrap().next;
    }
    slow.clone()
}
```

### 题型 5：删除链表倒数第N个节点（LeetCode 19）

**思路：先计算链表总长度 len，然后找到第 `len - n` 个节点（即要删除节点的前一个），执行删除。**

> 另一种经典方法是"快慢指针"：先让快指针走 n 步，然后快慢一起走，快到头时慢在要删除节点的前面。这里为了代码清晰，用两次遍历法。

```rust
fn remove_nth_from_end(head: Option<Box<ListNode>>, n: i32) -> Option<Box<ListNode>> {
    let mut dummy = Box::new(ListNode { val: 0, next: head });

    // 先计算链表长度
    let mut len = 0;
    let mut p = &dummy.next;
    while let Some(node) = p {
        len += 1;
        p = &node.next;
    }

    // 找到要删除节点的前一个
    let target = len - n;
    let mut p = &mut dummy;
    for _ in 0..target {
        p = p.next.as_mut().unwrap();
    }

    // 删除节点
    let next = p.next.as_mut().unwrap().next.take();
    p.next = next;

    dummy.next
}
```

### 哨兵节点（dummy node）技巧

**当链表头可能被删除或改变时，用一个虚拟头节点简化代码。**

```rust
let mut dummy = Box::new(ListNode { val: 0, next: head });
// 操作完后返回 dummy.next
```

| 技巧 | 用途 |
|------|------|
| 哨兵节点 (dummy) | 头节点可能变化时 |
| 快慢指针 | 找中间、检测环 |

## 七、测试题

1. **问：** 为什么 Rust 链表用 `Option<Box<ListNode>>` 而不是普通指针？

   **答：** Rust 没有 null 指针，用 Option 表示"有或没有"。Box 把节点放在堆上，因为链表的大小在编译期不确定（递归类型必须用 Box）。

2. **问：** `.take()` 方法做了什么？

   **答：** `option.take()` 取出 Option 中的值，原来的 Option 变成 None。这是 Rust 中转移链表节点所有权的关键方法。
