# 15 - 字符串算法

> 处理字符串的高效算法：模式匹配、字符串哈希。

## 一、基础知识

### 1.1 核心定义

字符串算法处理**文本匹配、比较、转换**等场景。本文涵盖三大核心算法：

- **KMP**（Knuth-Morris-Pratt）：O(n+m) 的模式匹配，利用失配时已匹配部分的信息避免回溯
- **Z 函数**：计算每个后缀与原字符串的最长公共前缀（LCP），是 KMP 的替代品
- **字符串哈希**（Rabin-Karp）：将字符串映射为整数哈希值，O(1) 比较子串是否相等

### 1.2 工作原理

- **KMP**：构建 next/pi 数组——`pi[i]` = 模式串前缀 `[0..i]` 的最长相等前后缀长度。匹配失败时 j 跳转到 `pi[j-1]`，而非复位到 0。
- **Z 函数**：`z[i]` = s[0..] 与 s[i..] 的 LCP。利用已计算的 z-box 加速（类似于 KMP 的"跳过已匹配"思想）。
- **字符串哈希**：`hash(s) = Σ(s[i] × base^i) mod P`。两个子串比较转化为哈希值比较。

### 1.3 适用场景与前提

| 场景 | 算法 | 复杂度 |
|------|------|--------|
| 单模式匹配 | KMP | O(n+m) |
| 多模式匹配 | Aho-Corasick（AC 自动机） | O(n + 匹配总数) |
| 最长公共前缀 | Z 函数 | O(n) |
| 判断子串相等 | 字符串哈希 | O(1) 每次 |
| 最长回文子串 | Manacher | O(n) |

**前提**：对于 LeetCode 常用场景，KMP / Z 函数对任意大小的字符集都适用（O(n+m) 与字符集大小无关）。字符串哈希依赖好的 base 和模数选择（常用 base=131, mod=10⁹+7），需注意哈希碰撞。实际使用中建议用双哈希（两个不同模数）降低碰撞概率。

### 1.4 优缺分析

**优点：**
- KMP 线性时间，永不回溯文本指针
- 字符串哈希编码简单，O(1) 比较，用途极广
- 这些算法是竞赛/面试中的硬通货，常考

**缺点：**
- KMP 的 next 数组推导抽象，初学者容易放弃
- 字符串哈希存在碰撞风险（需双哈希或大模数规避）
- 对于简单场景（如小字符串），暴力匹配反而更快（常数小）

## 二、KMP 算法

### 算法本质

**大白话：在文本串中查找模式串的位置，利用"已匹配部分"的信息避免重复比较。**

暴力匹配：每次失配就从头开始 → O(n×m)
KMP：利用 next 数组跳过不必要的比较 → O(n+m)

### next 数组（也叫 failure function / pi 数组）

`next[i]` = 模式串 `pattern[0..=i]` 的**最长相等前后缀的长度**。

> 📖 **什么是"最长相等前后缀"？** 
> - **前缀**：从开头到某个位置（不含最后一个字符）的子串
> - **后缀**：从某个位置到结尾（不含第一个字符）的子串
> - **相等**：前缀和后缀的字符完全相同
> - **最长**：在所有相等的前后缀中选最长的那个
> 
> 例如 `pattern = "ababa"` 的前缀有 `"a"`, `"ab"`, `"aba"`, `"abab"`；后缀有 `"a"`, `"ba"`, `"aba"`, `"baba"`。最长相等的是 `"aba"`（长度 3），所以 `next[4] = 3`。

```
pattern = "aabaaab"
next    = [0,1,0,1,2,2,3]

解释：
"a"       → 无前后缀相等 → 0
"aa"      → "a" == "a" → 1
"aab"     → 无 → 0
"aaba"    → "a" == "a" → 1
"aabaa"   → "aa" == "aa" → 2
"aabaaa"  → "aa" == "aa" → 2
"aabaaab" → "aab" == "aab" → 3
```

> 🔑 **next 数组如何加速匹配？**
> 
> 假设 `text = "ababcabc"，pattern = "abc"`，next = [0,0,0]。
> ```
> text:    a b a b c a b c
> pattern: a b c
>            ↑ 失配！j=1（'b' vs 'a'）
> ```
> 暴力法会退回 i=1 重来。但 KMP 利用 next[1]=0，知道"pattern 的前 2 个字符 `ab` 中没有相等前后缀"，所以 j 跳到 0，而不回退 i。
> 
> **核心思想**：已匹配的部分 `[0..j-1]` 中，如果它的某个后缀等于某个前缀，那么失配后可以跳过这个前缀的长度继续匹配——因为后缀已经和文本匹配了，对应的前缀也必然匹配。

### KMP 实现

```rust
fn kmp_search(text: &str, pattern: &str) -> Vec<usize> {
    let (s, p) = (text.as_bytes(), pattern.as_bytes());
    let n = s.len();
    let m = p.len();
    if m == 0 { return vec![]; }

    // 构建 next 数组
    let mut next = vec![0usize; m];
    let mut j = 0;
    for i in 1..m {
        while j > 0 && p[i] != p[j] {
            j = next[j - 1];
        }
        if p[i] == p[j] {
            j += 1;
        }
        next[i] = j;
    }

    // 匹配
    let mut result = vec![];
    let mut j = 0;
    for i in 0..n {
        while j > 0 && s[i] != p[j] {
            j = next[j - 1]; // 利用 next 数组跳转
        }
        if s[i] == p[j] {
            j += 1;
        }
        if j == m {
            result.push(i + 1 - m); // 找到匹配
            j = next[j - 1];        // 继续查找下一个匹配
        }
    }
    result
}
```

### 例题：实现 strStr()（LeetCode 28）

```rust
fn str_str(haystack: String, needle: String) -> i32 {
    match kmp_search(&haystack, &needle).first() {
        Some(&pos) => pos as i32,
        None => -1,
    }
}
```

## 三、Z 函数

### 算法本质

`z[i]` = 字符串 `s` 从位置 i 开始的子串与 `s` 本身的最长公共前缀长度。

```
s = "aabxaab"
z = [7, 1, 0, 0, 3, 1, 0]
     ^              ^
     整个字符串      "aab" == "aab"
```

```rust
fn z_function(s: &[u8]) -> Vec<usize> {
    let n = s.len();
    let mut z = vec![0usize; n];
    z[0] = n;
    let (mut l, mut r) = (0, 0);

    for i in 1..n {
        if i < r {
            z[i] = z[i - l].min(r - i);
        }
        while i + z[i] < n && s[z[i]] == s[i + z[i]] {
            z[i] += 1;
        }
        if i + z[i] > r {
            l = i;
            r = i + z[i];
        }
    }
    z
}
```

### Z 函数用于字符串匹配

把 pattern + "#" + text 拼接起来求 Z 函数，`z[i] == m` 的位置就是匹配位置。

## 四、字符串哈希（Rabin-Karp）

### 算法本质

**大白话：把字符串变成一个数字（哈希值），比较字符串变成比较数字，O(1) 时间。**

```rust
struct StringHash {
    hash: Vec<u64>,
    pow: Vec<u64>,
    modp: u64,
}

impl StringHash {
    fn new(s: &[u8]) -> Self {
        let base = 131u64;
        let modp = 1_000_000_007u64;
        let n = s.len();
        let mut hash = vec![0u64; n + 1];
        let mut pow = vec![1u64; n + 1];

        for i in 0..n {
            hash[i + 1] = (hash[i] * base + s[i] as u64) % modp;
            pow[i + 1] = pow[i] * base % modp;
        }
        StringHash { hash, pow, modp }
    }

    /// 获取子串 [l, r] 的哈希值（0-indexed）
    fn get(&self, l: usize, r: usize) -> u64 {
        (self.hash[r + 1] + self.modp - self.hash[l] * self.pow[r - l + 1] % self.modp) % self.modp
    }
}
```

### 使用场景
- 快速判断两个子串是否相等
- 字符串匹配
- 最长公共子串（二分 + 哈希）

## 五、Manacher 算法（最长回文子串）

### 暴力 vs Manacher

- **暴力：** 枚举每个中心，向两边扩展 → O(n²)
- **Manacher：** 利用回文的对称性 → O(n)

### 例题：最长回文子串（LeetCode 5）

简单做法（中心扩展，面试够用）：

```rust
fn longest_palindrome(s: String) -> String {
    let s = s.as_bytes();
    let n = s.len();
    let mut start = 0;
    let mut max_len = 1;

    // 从中心向两边扩展
    let expand = |mut l: i32, mut r: i32| -> (usize, usize) {
        while l >= 0 && (r as usize) < n && s[l as usize] == s[r as usize] {
            l -= 1;
            r += 1;
        }
        ((l + 1) as usize, (r - l - 1) as usize)
    };

    for i in 0..n {
        // 奇数长度回文
        let (s1, l1) = expand(i as i32, i as i32);
        if l1 > max_len { start = s1; max_len = l1; }

        // 偶数长度回文
        let (s2, l2) = expand(i as i32, i as i32 + 1);
        if l2 > max_len { start = s2; max_len = l2; }
    }

    String::from_utf8(s[start..start + max_len].to_vec()).unwrap()
}
```

## 七、测试题

1. **问：** KMP 的 next 数组有什么用？

   **答：** 当匹配失败时，next 数组告诉我们模式串应该回退到哪个位置继续匹配，而不是从头开始。

2. **问：** 字符串哈希有什么缺点？

   **答：** 有哈希冲突的风险（不同字符串可能有相同哈希值）。可以用双模数降低冲突概率。
