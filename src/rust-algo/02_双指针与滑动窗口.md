# 02 - 双指针与滑动窗口

> 真正的算法能力不是背模板，是理解"为什么指针这样移动不会漏掉答案"。

## 一、基础知识

### 1.1 核心定义

**双指针**：用两个索引变量协同遍历，将 O(n²) 暴力枚举优化到 O(n)。核心不是"用两个变量"，而是**每次移动都能排除一批不可能的解**。

**滑动窗口**：双指针的高级形态，维护一个满足条件的连续区间。右指针不断扩张，左指针在条件被破坏时收缩。

### 1.2 工作原理

双指针将暴力 O(n²) 优化到 O(n) 的核心机制：**排除法 + 单调性**。

| 模式 | 排除逻辑 | 为什么会漏掉答案 |
|------|---------|----------------|
| 相向双指针 | `a[l]+a[r] < target` → 排除 `a[l]`（因为 `a[r]` 已是最大） | 如果不排序就用，`a[r]` 未必是最大值 |
| 快慢指针 | fast 扫描，slow 收集，跳过不需保留的元素 | slow 前面不会再有需要保留的元素 |
| 滑动窗口 | 窗口内某个量随扩张单调不减（如字符计数、和），一旦超标就收缩 | 如果问题不满足这个单调性，滑动窗口会出错 |

**滑动窗口的均摊 O(n) 原理**：虽然代码有 `for + while` 两层，但 `left` 在整个生命周期中最多从 0 走到 n-1（每个位置最多进一次、出一次），所以 while 总执行次数 ≤ n。

> **🔑 均摊分析详解**：
> ```
> for right in 0..n:        ← 执行 n 次，每次 O(1)
>     while 窗口不合法:       ← 看起来是 O(n)，但全局只执行 ≤ n 次
>         left += 1          ← left 从不超过 right，right 到 n 时 left ≤ n
> ```
> 外层循环 n 次，while 总收缩次数 ≤ n（因为 left 只增不减），总操作 ≤ 2n = **O(n)**。这就像去银行排队——你排一次队，工作人员服务你一次；虽然看起来有两层（排队 + 服务），但每个人只被服务一次。
>
> **⚠️ 初学者常见误区**：不要把 `for` 里的 `while` 自动理解为 O(n²)。只要内层循环的迭代次数独立于外层（如 left 只进不退），就是用**均摊分析**的时候。

### 1.3 适用场景与前提

| 模式 | 识别信号 | 前提 |
|------|---------|------|
| 相向双指针 | "有序数组"、"两/三数之和"、"接雨水" | 数组有序（或能证明排除法成立） |
| 快慢指针 | "原地删除"、"去重"、"移动零" | 结果不要求保持所有元素 |
| 滑动窗口（变长） | "满足条件的**最长/最短**子串/子数组" | 窗口状态随扩张单调不减 |
| 固定窗口 | "异位词"、"长度为K的子串" | 窗口大小固定 |

**滑动窗口不适用的情况**：如果窗口内条件不单调。例如"子数组乘积最大"——缩小窗口后乘积可能变大也可能变小，不能用滑动窗口。

### 1.4 优缺分析

**优点**：O(n) 极致效率，空间通常 O(1)，代码模式固定易复用。

**缺点**：必须理解"为什么能排除"才能用对；窗口条件不单调时失效；变长和固定窗口模板容易混淆。

## 二、从暴力到优化（核心推导）

学算法最重要的不是记住结论，而是理解**思考过程**。

### 以两数之和 II 为例

**暴力法（O(n²)）**：枚举所有 `(i, j)` 组合，检查 `a[i] + a[j] == target`。

```rust
fn brute_two_sum(nums: &[i32], target: i32) -> Vec<i32> {
    for i in 0..nums.len() {
        for j in i+1..nums.len() {
            if nums[i] + nums[j] == target {
                return vec![i as i32 + 1, j as i32 + 1];
            }
        }
    }
    vec![]
}
```

**为什么慢**：n=10⁴ 时，需要 ~5×10⁷ 次比较，超时。

**关键观察**：数组有序！如果 `a[l] + a[r] < target`，那 `a[l]` 和 `a[l+1..r-1]` 中任何一个的和都 `< target`（因为 `a[r]` 已经是最大的了）。所以可以**一步排除掉 a[l] 参与的所有组合**。

```rust
// 优化后：每次比较后至少排除一个元素，总比较次数 ≤ n
fn two_sum(numbers: Vec<i32>, target: i32) -> Vec<i32> {
    // 指针从两端开始，利用"有序"这个关键性质
    let (mut l, mut r) = (0, numbers.len() - 1);
    while l < r {
        let sum = numbers[l] + numbers[r];
        if sum == target {
            return vec![l as i32 + 1, r as i32 + 1];
        }
        // 核心：sum < target → 左指针当前值太小，和谁都凑不够
        // 安全排除 a[l]，因为 a[r] 是剩余中最大的
        // sum > target → 右指针当前值太大，和谁都超了
        // 安全排除 a[r]，因为 a[l] 是剩余中最小的
        if sum < target { l += 1; } else { r -= 1; }
    }
    vec![]
}
```

### 思维模式总结

1. **先写暴力**：理清"要做哪些比较"
2. **找可排除的规律**：什么情况下能安全跳过一批比较？（需要有序性/单调性）
3. **设计指针移动规则**：每次移动至少排除一个候选，保证不漏最优解
4. **验证正确性**：手动模拟小例子，确认边界情况

## 三、双指针的三种模式

### 模式一：相向双指针

```
左指针 →           ← 右指针
[1, 2, 3, 4, 5, 6, 7]
 ↑                    ↑
 L                    R
```

**模板**：

```rust
fn opposite_two_pointer(arr: &[i32], target: i32) -> bool {
    // 两个指针从两端出发，向中间靠拢
    // 每次比较后，根据结果排除一侧（不会漏掉答案）
    let (mut l, mut r) = (0, arr.len() - 1);
    while l < r {
        match (arr[l] + arr[r]).cmp(&target) {
            std::cmp::Ordering::Less    => l += 1, // 左值太小，即使加上最大的右值也不够
            std::cmp::Ordering::Greater => r -= 1, // 右值太大，即使加上最小的左值也超标
            std::cmp::Ordering::Equal   => return true,
        }
    }
    false
}
```

### 例题：LeetCode 11. 盛最多水的容器

```rust
fn max_area(height: Vec<i32>) -> i32 {
    let (mut l, mut r) = (0, height.len() - 1);
    let mut ans = 0;
    while l < r {
        // 面积 = 宽度 × 矮边高度（短板效应：水位由较矮的板决定）
        let area = (r - l) as i32 * height[l].min(height[r]);
        ans = ans.max(area);
        // 关键决策：移动较短的那条边
        // 为什么？因为移动较长边，宽度减少，高度受短板限制不会增加 → 面积一定变小
        // 只有移动短边，新边有可能比原短边高 → 面积才有可能变大
        if height[l] < height[r] {
            l += 1;
        } else {
            r -= 1;
        }
    }
    ans
}
```

**常见错误**：很多人会想"移动两边一起算，再选面积大的"。这不是双指针——这样退化为暴力枚举。

---

### 模式二：快慢指针（同向双指针）

```
慢指针 → 快指针 →
[1, 2, 3, 4, 5, 6, 7]
 ↑        ↑
slow     fast
```

slow 是"已处理区间"的右边界，fast 是"扫描器"。fast 遇到需要保留的元素就交给 slow，slow 右移；遇到不需保留的就跳过。

### 例题：LeetCode 283. 移动零

```rust
fn move_zeroes(nums: &mut Vec<i32>) {
    // slow 指向"下一个非零元素应该放的位置"
    // 不变量：slow 左侧全是非零元素（且保持相对顺序）
    let mut slow = 0;
    for fast in 0..nums.len() {
        if nums[fast] != 0 {
            // 非零元素交换到前面，slow 前进
            // 用 swap 而不是覆盖：因为被交换过来的元素可能是 0
            nums.swap(slow, fast);
            slow += 1;
        }
    }
    // 循环结束后 slow 指向第一个 0 应该在的位置
}
```

**常见错误**：用覆盖 `nums[slow] = nums[fast]` 而不是 swap。这会把后面的非零原值覆盖掉（不过本题覆盖的是 0，不影响。但在需要保留所有元素的变体中会出错）。

### 模式三：滑动窗口

```
     窗口
  [------]
[1, 2, 3, 4, 5, 6, 7]
  ↑        ↑
  L        R
```

**识别口诀**："满足条件的**最长/最短**子数组/子串" → 滑动窗口。

## 四、滑动窗口万能模板

### 模板一：求最长

```rust
/// 求和不超过 limit 的最长子数组长度
fn longest_window(arr: &[i32], limit: i32) -> usize {
    // left 是窗口左边界，right 是右边界
    // 不变量：窗口 [left, right] 始终满足条件
    let mut left = 0;
    let mut ans = 0;
    let mut window_sum = 0;

    for right in 0..arr.len() {
        // 步骤 1：新元素进窗口
        window_sum += arr[right];

        // 步骤 2：窗口不满足条件 → 收缩左端直到恢复
        // 为什么用 while 不是 if？因为可能需要收缩多次
        while window_sum > limit {
            window_sum -= arr[left];
            left += 1;
        }

        // 步骤 3：窗口已恢复合法状态，更新答案
        ans = ans.max(right - left + 1);
    }
    ans
}
```

### 模板二：求最短

```rust
/// 求和 >= target 的最短子数组长度
fn shortest_window(arr: &[i32], target: i32) -> usize {
    let mut left = 0;
    let mut ans = usize::MAX;
    let mut window_sum = 0;

    for right in 0..arr.len() {
        window_sum += arr[right];

        // 与"求最长"的区别：
        // 这里是"满足条件时更新答案并收缩"（因为要尽量短）
        while window_sum >= target {
            ans = ans.min(right - left + 1);
            window_sum -= arr[left];
            left += 1;
        }
        // "求最长"做法：不满足时才收缩，满足时更新答案
    }
    if ans == usize::MAX { 0 } else { ans }
}
```

### 模板选择决策树

```
问题问"最长" → 不满足条件时收缩，满足时更新 → 模板一
问题问"最短" → 满足条件时收缩+更新 → 模板二
窗口大小固定  → 固定窗口，维护频率数组 → 模板三
```

## 五、经典例题详解

### LeetCode 3. 无重复字符的最长子串

```rust
fn length_of_longest_substring(s: String) -> i32 {
    let s = s.as_bytes();
    // 用定长数组 [0; 128] 代替 HashMap，因字符集为 ASCII，O(1) 访问
    // 这是工程中常见的"空间换时间"：已知字符集小时直接用数组
    let mut cnt = [0i32; 128];
    let mut left = 0;
    let mut ans = 0;

    for right in 0..s.len() {
        let ch = s[right] as usize;
        cnt[ch] += 1;

        // 有重复字符：收缩左端直到重复消除
        // 注意判断条件：cnt[ch] > 1 表示刚加入的字符出现了两次
        // 收缩后必定会把这个字符的 count 降回 1
        while cnt[ch] > 1 {
            cnt[s[left] as usize] -= 1;
            left += 1;
        }

        ans = ans.max(right - left + 1);
    }
    ans as i32
}
```

**手动模拟** `s = "abcabcbb"`：

```
right=0: 'a' 进窗口 → [a]       → ans=1
right=1: 'b' 进窗口 → [ab]      → ans=2
right=2: 'c' 进窗口 → [abc]     → ans=3
right=3: 'a' 进窗口 → [abca]    → 重复! 收缩到 [bca] → ans=3
right=4: 'b' 进窗口 → [bcab]    → 重复! 收缩到 [cab] → ans=3
right=5: 'c' 进窗口 → [cabc]    → 重复! 收缩到 [abc] → ans=3
right=6: 'b' 进窗口 → [abcb]    → 重复! 收缩到 [cb]  → ans=3
right=7: 'b' 进窗口 → [cbb]     → 重复! 收缩到 [b]   → ans=3
结果: 3 ✓
```

### LeetCode 1004. 最大连续 1 的个数 III

**转化思维**："最多翻转 k 个 0" → "窗口内 0 的个数不超过 k"。

```rust
fn longest_ones(nums: Vec<i32>, k: i32) -> i32 {
    let mut left = 0;
    // zero_cnt: 窗口内 0 的数量，是判断"是否需要收缩"的唯一标准
    let mut zero_cnt = 0;
    let mut ans = 0;

    for right in 0..nums.len() {
        if nums[right] == 0 {
            zero_cnt += 1;
        }
        // 修复窗口：0 太多就收缩
        while zero_cnt > k {
            if nums[left] == 0 {
                zero_cnt -= 1;
            }
            left += 1;
        }
        ans = ans.max(right - left + 1);
    }
    ans as i32
}
```

**常见错误**：忘记 `while` 用 `if`。如果窗口内一下子多了 >1 个 0（虽然本题一次进一个元素不会，但含其他计数的问题可能），`if` 只收缩一次，窗口可能仍不合法。

### LeetCode 438. 字母异位词（固定窗口）

```rust
fn find_anagrams(s: String, p: String) -> Vec<i32> {
    let (s, p) = (s.as_bytes(), p.as_bytes());
    if s.len() < p.len() { return vec![]; }

    // 模式串 p 的字符频率是"目标"
    let mut target = [0i32; 26];
    for &b in p { target[(b - b'a') as usize] += 1; }

    // window 是当前固定大小窗口的频率
    let mut window = [0i32; 26];
    let mut ans = vec![];

    for i in 0..s.len() {
        // 右端进窗口
        window[(s[i] - b'a') as usize] += 1;

        // 窗口满了，左端出窗口
        if i >= p.len() {
            let left_char = (s[i - p.len()] - b'a') as usize;
            window[left_char] -= 1;
        }

        // 频率匹配就是异位词
        if window == target {
            ans.push((i + 1 - p.len()) as i32);
        }
    }
    ans
}
```

## 六、常见错误集锦

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 相向双指针未排序就用 | 排除法不成立，漏解 | 先排序 |
| 滑动窗口用 `if` 代替 `while` | 窗口可能仍不合法 | 必须 `while` 循环收缩 |
| 滑动窗口中维护的状态不随收缩更新 | 状态和窗口不同步 | 每次收缩都要同步更新状态 |
| 求最短窗口模板和求最长混用 | 返回错误答案 | 求最长：不满足时收缩；求最短：满足时收缩 |
| 求最短窗口时，收缩前忘记先更新答案 | 漏掉合法解 | 先更新 `ans = ans.min(...)` 再 `left++` |
| 求最长窗口时在收缩循环内更新答案 | 答案偏小 | 求最长：while 收缩完成后（窗口合法）再更新 |
| 求最长窗口时，收缩循环内更新答案 | 答案偏小 | 求最长：在 while 之外（窗口合法后）更新答案 |
| 移动零用覆盖代替 swap | 原位置的值丢失 | 用 swap 而非覆盖 |

## 八、测试题

1. **问：** 为什么滑动窗口是 O(n) 不是 O(n²)？
   **答：** left 指针只会前进不会后退，整个生命周期最多移动 n 次。两层循环的总执行次数 = for 的 n 次 + while 的 ≤n 次 = O(n)。

2. **问：** 什么时候滑动窗口不能用？
   **答：** 窗口内条件不单调时。例如"子数组乘积最大"——缩小窗口乘积可能变大也可能变小。

3. **问：** 三数之和如何用双指针？
   **答：** 排序后固定第一个数，剩余部分用相向双指针找两数之和。
