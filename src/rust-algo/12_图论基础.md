# 12 - 图论基础

> 节点和边组成的网络，用来解决"连通性"和"最短路径"问题。

## 一、基础知识

### 1.1 核心定义

**图**（Graph）由一组**节点**（Vertex）和连接它们的**边**（Edge）组成。根据边是否有方向分为有向图和无向图，根据边是否有权重分为无权图和带权图。

核心术语：度（连接的边数）、入度（指向该节点的边数，有向图）、出度（从该节点出发的边数，有向图）、路径（从 u 到 v 经过的边序列）、连通分量（互相可达的节点集合）、环（起点=终点的非平凡路径）。

### 1.2 工作原理

图的两种存储方式：
- **邻接表**（`Vec<Vec<usize>>`）：每个节点维护一个邻居列表。空间 O(V+E)，适合稀疏图，LeetCode 首选
- **邻接矩阵**（`Vec<Vec<bool>>`）：二维矩阵标记边。空间 O(V²)，适合稠密图

两种基本遍历：
- **DFS**（递归/栈）：沿一条路径深入到底再回溯，适合找连通分量、拓扑排序、检测环
- **BFS**（队列）：逐层扩展，适合无权图的最短路径、层次遍历

### 1.3 适用场景与前提

| 问题 | 算法 | 复杂度 |
|------|------|--------|
| 无权图最短路径 | BFS | O(V+E) |
| 带权图最短路径 | Dijkstra（非负权） | O((V+E) log V) |
| 拓扑排序 | DFS / Kahn 算法 | O(V+E) |
| 连通分量 | Union-Find 或 DFS | O(V+E) |
| 二分图检测 | BFS/DFS 染色 | O(V+E) |
| 最小生成树 | Prim / Kruskal | O(E log V) |

**前提**：Dijkstra 要求边权非负（负权需 Bellman-Ford 或 SPFA）。拓扑排序前提是 DAG（有向无环图）。图的规模（V 和 E）决定用邻接表还是矩阵。

> ⚠️ **Dijkstra 为什么不能处理负权边？** Dijkstra 基于贪心——每次从优先队列中取出当前距离最小的节点，并断言它的最短距离已确定。但如果存在负权边，后续节点可能通过负权边"抄近路"，之前取出的节点距离可能不是最小的。这个断言不再成立。例如 `A→B(2), A→C(3), B→C(-5)`：Dijkstra 先确定 A→C=3，但实际 A→B→C=2+(-5)=-3 更短。
>
> 🔑 **负权边解决方案**：Bellman-Ford O(VE) 对所有边松弛 V-1 轮；SPFA（Bellman-Ford 的队列优化版）在大多数情况下接近 O(E)，但会被特殊数据卡到 O(VE)。

### 1.4 优缺分析

**优点：**
- 模型通用性强——社交网络、地图导航、依赖关系都可建模为图
- BFS/DFS 统一框架，实现简单
- 许多图问题有成熟的 O(V+E) 解法

**缺点：**
- 图的遍历和搜索比数组/树更复杂（需要 visited 标记防环）
- 存储开销随边数增长；稠密图内存占用大
- 难以在 Rust 中直接表达图（所有权和借用规则复杂），通常用 `Vec<Vec<usize>>` 配合索引

## 三、图的表示

### 邻接表（推荐，LeetCode 最常用）

```rust
// 方式1：Vec<Vec<usize>> （无权图）
let n = 5; // 5个节点
let mut graph = vec![vec![]; n];
// 添加边 u -> v
graph[0].push(1); // 0 → 1
graph[0].push(2); // 0 → 2
graph[1].push(3); // 1 → 3

// 方式2：Vec<Vec<(usize, i32)>> （带权图）
let mut graph = vec![vec![]; n];
graph[0].push((1, 5)); // 0 → 1，权重5
graph[0].push((2, 3)); // 0 → 2，权重3
```

### 从边列表构建图

```rust
fn build_graph(n: usize, edges: &Vec<Vec<i32>>) -> Vec<Vec<usize>> {
    let mut graph = vec![vec![]; n];
    for edge in edges {
        let (u, v) = (edge[0] as usize, edge[1] as usize);
        graph[u].push(v);
        graph[v].push(u); // 无向图要加双向
    }
    graph
}
```

## 四、DFS（深度优先搜索）

**大白话：走迷宫时"一条路走到黑，走不通再回头换路"。**

```rust
fn dfs(graph: &Vec<Vec<usize>>, node: usize, visited: &mut Vec<bool>) {
    visited[node] = true;
    println!("访问: {}", node);

    for &next in &graph[node] {
        if !visited[next] {
            dfs(graph, next, visited);
        }
    }
}

// 使用
let mut visited = vec![false; n];
dfs(&graph, 0, &mut visited);
```

### 应用：判断连通分量数

```rust
fn count_components(n: usize, graph: &Vec<Vec<usize>>) -> i32 {
    let mut visited = vec![false; n];
    let mut count = 0;

    for i in 0..n {
        if !visited[i] {
            dfs(graph, i, &mut visited);
            count += 1; // 每次 DFS 发现一个新的连通分量
        }
    }
    count
}
```

### 应用：岛屿数量（LeetCode 200）

```rust
fn num_islands(mut grid: Vec<Vec<char>>) -> i32 {
    let (m, n) = (grid.len(), grid[0].len());
    let mut count = 0;

    fn dfs(grid: &mut Vec<Vec<char>>, i: usize, j: usize) {
        if i >= grid.len() || j >= grid[0].len() || grid[i][j] == '0' {
            return;
        }
        grid[i][j] = '0'; // 标记已访问（沉岛）
        if i > 0 { dfs(grid, i - 1, j); }
        dfs(grid, i + 1, j);
        if j > 0 { dfs(grid, i, j - 1); }
        dfs(grid, i, j + 1);
    }

    for i in 0..m {
        for j in 0..n {
            if grid[i][j] == '1' {
                dfs(&mut grid, i, j);
                count += 1;
            }
        }
    }
    count
}
```

## 五、BFS（广度优先搜索）

**大白话：像水波一样一圈一圈往外扩散。先访问所有距离为1的点，再访问距离为2的，以此类推。**

```rust
use std::collections::VecDeque;

fn bfs(graph: &Vec<Vec<usize>>, start: usize) -> Vec<i32> {
    let n = graph.len();
    let mut dist = vec![-1i32; n]; // -1 表示未访问
    let mut queue = VecDeque::new();

    dist[start] = 0;
    queue.push_back(start);

    while let Some(node) = queue.pop_front() {
        for &next in &graph[node] {
            if dist[next] == -1 { // 未访问
                dist[next] = dist[node] + 1;
                queue.push_back(next);
            }
        }
    }
    dist // dist[i] = 从 start 到 i 的最短距离
}
```

**BFS 的核心性质：在无权图中，BFS 找到的一定是最短路径！**

### 网格 BFS 模板

```rust
use std::collections::VecDeque;

fn bfs_grid(grid: &Vec<Vec<i32>>, start: (usize, usize)) -> Vec<Vec<i32>> {
    let (m, n) = (grid.len(), grid[0].len());
    let mut dist = vec![vec![-1; n]; m];
    let mut queue = VecDeque::new();
    let dirs = [(0, 1), (0, -1), (1, 0), (-1, 0)]; // 四个方向

    dist[start.0][start.1] = 0;
    queue.push_back(start);

    while let Some((x, y)) = queue.pop_front() {
        for &(dx, dy) in &dirs {
            let nx = x as i32 + dx;
            let ny = y as i32 + dy;
            if nx >= 0 && nx < m as i32 && ny >= 0 && ny < n as i32 {
                let (nx, ny) = (nx as usize, ny as usize);
                if dist[nx][ny] == -1 && grid[nx][ny] != 1 { // 1 代表墙
                    dist[nx][ny] = dist[x][y] + 1;
                    queue.push_back((nx, ny));
                }
            }
        }
    }
    dist
}
```

## 六、拓扑排序

**大白话：给有先后顺序的任务排一个合理的执行顺序。** 比如先穿袜子再穿鞋。

### BFS 拓扑排序（Kahn 算法）

```rust
fn topological_sort(n: usize, edges: &Vec<Vec<i32>>) -> Vec<usize> {
    let mut graph = vec![vec![]; n];
    let mut in_degree = vec![0; n]; // 入度

    for edge in edges {
        let (u, v) = (edge[0] as usize, edge[1] as usize);
        graph[u].push(v);
        in_degree[v] += 1;
    }

    let mut queue: VecDeque<usize> = VecDeque::new();
    // 入度为0的节点先入队
    for i in 0..n {
        if in_degree[i] == 0 {
            queue.push_back(i);
        }
    }

    let mut order = vec![];
    while let Some(node) = queue.pop_front() {
        order.push(node);
        for &next in &graph[node] {
            in_degree[next] -= 1;
            if in_degree[next] == 0 {
                queue.push_back(next);
            }
        }
    }

    if order.len() == n { order } else { vec![] } // 如果不等于 n，说明有环
}
```

### 应用：课程表（LeetCode 207）

```rust
fn can_finish(num_courses: i32, prerequisites: Vec<Vec<i32>>) -> bool {
    let n = num_courses as usize;
    let mut graph = vec![vec![]; n];
    let mut in_degree = vec![0; n];

    for pre in &prerequisites {
        graph[pre[1] as usize].push(pre[0] as usize);
        in_degree[pre[0] as usize] += 1;
    }

    let mut queue: VecDeque<usize> = VecDeque::new();
    for i in 0..n {
        if in_degree[i] == 0 { queue.push_back(i); }
    }

    let mut count = 0;
    while let Some(node) = queue.pop_front() {
        count += 1;
        for &next in &graph[node] {
            in_degree[next] -= 1;
            if in_degree[next] == 0 { queue.push_back(next); }
        }
    }
    count == n // 所有课程都能学完 = 无环
}
```

## 七、最短路径

### Dijkstra 算法（非负权重图）

**大白话：从起点开始，每次选"当前最近的未访问节点"，更新它邻居的距离。**

```rust
use std::collections::BinaryHeap;
use std::cmp::Reverse;

fn dijkstra(graph: &Vec<Vec<(usize, i64)>>, start: usize) -> Vec<i64> {
    let n = graph.len();
    let mut dist = vec![i64::MAX; n];
    let mut heap = BinaryHeap::new();

    dist[start] = 0;
    heap.push(Reverse((0i64, start))); // (距离, 节点)

    while let Some(Reverse((d, u))) = heap.pop() {
        if d > dist[u] { continue; } // 已经有更短的路径了

        for &(v, w) in &graph[u] {
            let new_dist = dist[u] + w;
            if new_dist < dist[v] {
                dist[v] = new_dist;
                heap.push(Reverse((new_dist, v)));
            }
        }
    }
    dist
}
```

### DFS vs BFS 选择

| 场景 | 用 DFS | 用 BFS |
|------|--------|--------|
| 连通性/可达性 | ✅ | ✅ |
| 无权图最短路 | ❌ | ✅（BFS天然找最短路） |
| 路径枚举/回溯 | ✅ | ❌ |
| 拓扑排序 | ✅ | ✅ |
| 层级遍历 | ❌ | ✅ |

## 八、测试题

1. **问：** BFS 为什么能找到最短路径？

   **答：** 因为 BFS 按层扩展，先访问距离为 k 的所有节点，再访问距离为 k+1 的。第一次到达某个节点时，一定是最短路径。

2. **问：** 什么时候用 Dijkstra 而不是 BFS？

   **答：** 当边有不同的权重时。BFS 适用于无权图（或边权为1的图），Dijkstra 适用于边权非负的图。
