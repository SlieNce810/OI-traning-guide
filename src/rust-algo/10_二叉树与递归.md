# 10 - 二叉树与递归

> 树是递归的天堂。理解了树的递归，你就理解了递归的精髓。

## 一、基础知识

### 1.1 核心定义

**二叉树**是每个节点最多有两个子节点（左和右）的树形结构。它是递归定义的：一棵二叉树要么为空，要么由一个根节点和两棵分别称为左子树、右子树的二叉树组成。

关键概念：根节点、叶节点（无孩子）、深度（从根到该节点的边数）、高度（从该节点到最远叶子的边数）、完全二叉树（除最后一层外每层全满）、满二叉树（每个节点有 0 或 2 个孩子）。

### 1.2 工作原理

二叉树操作天然适合递归——因为"整棵树 = 根 + 左子树 + 右子树"，左右子树又是同样结构的树：

- **遍历**：前序（根→左→右）、中序（左→根→右）、后序（左→右→根）、层序（BFS）
- **递归三要素**：终止条件（空节点）、处理当前节点、递归处理子树
- Rust 中 `Rc<RefCell<TreeNode>>` 允许多个拥有者共享节点；`borrow()` 获取不可变借用，`borrow_mut()` 获取可变借用

### 1.3 适用场景与前提

| 场景 | 方法 |
|------|------|
| 遍历/搜索所有节点 | DFS（前中后序）或 BFS（层序） |
| 求深度/高度/直径 | 后序递归，自底向上汇总 |
| 验证 BST | 中序遍历是否为升序 |
| 路径和问题 | DFS 回溯，叶子处判断 |
| 最近公共祖先 | 后序遍历，左右子树各找一个 |

**前提**：树必须是连通的（没有环）。Rust 中 `Rc<RefCell<>>` 需要注意不能循环引用（会导致内存泄漏）。

### 1.4 优缺分析

**优点：**
- 天然支持递归，代码简洁优雅
- 二叉搜索树（BST）查找/插入/删除 O(log n)（平衡情况）
- 既支持深度遍历又支持广度遍历，灵活应对不同问题

**缺点：**
- 递归有栈溢出风险（深度 > 1000），需转为迭代或尾递归
- Rust 中 `Rc<RefCell<>>` 运行时借用检查，误操作会 panic（如同时 `borrow()` 和 `borrow_mut()`）
- 最坏情况退化为链表（如插入有序序列），BST 操作降为 O(n)
- 非递归遍历的代码复杂度和可读性远不如递归

> ⚠️ **递归深度限制**：默认情况下，Rust 的栈大小约 2MB。每次递归调用约消耗几十到几百字节（取决于局部变量）。当树深度超过 ~5000 时，可能发生**栈溢出（Stack Overflow）**。解决方案：转为迭代法（手动维护栈）或用 `#[inline]` 让编译器做尾递归优化。
>
> 🔑 **尾递归（Tail Recursion）**：如果递归调用是函数的最后一步操作，编译器可能把它优化成循环，不消耗栈空间。但 Rust 不保证尾递归优化，所以不要太依赖它。

## 二、算法本质

**大白话：二叉树就是每个节点最多有两个孩子（左和右）的树形结构。**

```
        1          ← 根节点
       / \
      2   3        ← 左子树和右子树
     / \   \
    4   5   6      ← 叶子节点（没有孩子的节点）
```

**递归的本质：把大问题分解成相同结构的小问题。**

树天然适合递归，因为"整棵树 = 根 + 左子树 + 右子树"，左右子树又是同样结构的树。

## 三、二叉树的定义（LeetCode）

```rust
use std::rc::Rc;
use std::cell::RefCell;

#[derive(Debug, PartialEq, Eq)]
pub struct TreeNode {
    pub val: i32,
    pub left: Option<Rc<RefCell<TreeNode>>>,
    pub right: Option<Rc<RefCell<TreeNode>>>,
}

type Tree = Option<Rc<RefCell<TreeNode>>>;
```

**为什么这么复杂？**
- `Option`：可能有节点，可能是空（`None` = 空节点）
- `Rc`（引用计数）：允许多个变量指向同一个节点。`Rc::clone(&node)` 只是增加计数，不会复制整棵树
- `RefCell`：把 Rust 的借用检查从编译期推迟到运行期，允许通过共享引用修改数据

**常用操作速记：**
```rust
// 获取节点内容（不可变借用）
let n = node.borrow();
// 获取节点内容（可变借用，用于修改）
let mut n = node.borrow_mut();
// 注意：同时调用 borrow() 和 borrow_mut() 会运行时 panic！
// 克隆 Rc（便宜操作，只增加引用计数）
let cloned = node.clone(); // 或 Rc::clone(&node)
```

## 四、三种遍历方式

### 前序遍历（根 → 左 → 右）

```rust
fn preorder(root: &Tree, result: &mut Vec<i32>) {
    if let Some(node) = root {
        let node = node.borrow();
        result.push(node.val);       // 先访问根
        preorder(&node.left, result);  // 再左子树
        preorder(&node.right, result); // 最后右子树
    }
}
```

### 中序遍历（左 → 根 → 右）

```rust
fn inorder(root: &Tree, result: &mut Vec<i32>) {
    if let Some(node) = root {
        let node = node.borrow();
        inorder(&node.left, result);   // 先左子树
        result.push(node.val);          // 再访问根
        inorder(&node.right, result);  // 最后右子树
    }
}
// 对于二叉搜索树，中序遍历的结果是升序的！
```

### 后序遍历（左 → 右 → 根）

```rust
fn postorder(root: &Tree, result: &mut Vec<i32>) {
    if let Some(node) = root {
        let node = node.borrow();
        postorder(&node.left, result);  // 先左子树
        postorder(&node.right, result); // 再右子树
        result.push(node.val);           // 最后访问根
    }
}
```

### 层序遍历（BFS）

```rust
use std::collections::VecDeque;

fn level_order(root: Tree) -> Vec<Vec<i32>> {
    let mut result = vec![];
    if root.is_none() { return result; }

    let mut queue = VecDeque::new();
    queue.push_back(root.unwrap());

    while !queue.is_empty() {
        let size = queue.len();
        let mut level = vec![];
        for _ in 0..size {
            let node = queue.pop_front().unwrap();
            let node = node.borrow();
            level.push(node.val);
            if let Some(left) = &node.left {
                queue.push_back(left.clone());
            }
            if let Some(right) = &node.right {
                queue.push_back(right.clone());
            }
        }
        result.push(level);
    }
    result
}
```

### 记忆技巧

**前/中/后序 = "根"在什么位置：**
- **前**序：**根**左右
- **中**序：左**根**右
- **后**序：左右**根**

## 五、递归思维模板

### 模板 1：自顶向下（传递信息给子树）

```rust
fn solve(root: &Tree, info: i32) {
    if let Some(node) = root {
        let node = node.borrow();
        // 处理当前节点（利用从父节点传来的 info）
        let updated_info = info + node.val; // 举例：累加路径和
        solve(&node.left, updated_info);
        solve(&node.right, updated_info);
    }
}
```

### 模板 2：自底向上（收集子树的信息）

```rust
fn solve(root: &Tree) -> i32 {
    match root {
        None => 0, // 空节点的返回值
        Some(node) => {
            let node = node.borrow();
            let left = solve(&node.left);   // 收集左子树信息
            let right = solve(&node.right); // 收集右子树信息
            // 根据左右子树的信息计算当前节点的答案
            left + right + node.val // 举例
        }
    }
}
```

## 六、经典例题

### 例题 1：二叉树的最大深度（LeetCode 104）

```rust
fn max_depth(root: Tree) -> i32 {
    match root {
        None => 0,
        Some(node) => {
            let node = node.borrow();
            let left = max_depth(node.left.clone());
            let right = max_depth(node.right.clone());
            1 + left.max(right)  // 深度 = 1 + max(左深度, 右深度)
        }
    }
}
```

### 例题 2：翻转二叉树（LeetCode 226）

```rust
fn invert_tree(root: Tree) -> Tree {
    if let Some(node) = &root {
        let mut node = node.borrow_mut();
        let left = invert_tree(node.left.take());
        let right = invert_tree(node.right.take());
        node.left = right;   // 左右交换
        node.right = left;
    }
    root
}
```

### 例题 3：对称二叉树（LeetCode 101）

```rust
fn is_symmetric(root: Tree) -> bool {
    fn check(left: &Tree, right: &Tree) -> bool {
        match (left, right) {
            (None, None) => true,
            (Some(l), Some(r)) => {
                let (l, r) = (l.borrow(), r.borrow());
                l.val == r.val
                    && check(&l.left, &r.right)  // 左的左 = 右的右
                    && check(&l.right, &r.left)   // 左的右 = 右的左
            }
            _ => false,
        }
    }
    match &root {
        None => true,
        Some(node) => {
            let node = node.borrow();
            check(&node.left, &node.right)
        }
    }
}
```

### 例题 4：二叉树的直径（LeetCode 543）

> **LeetCode 543. 二叉树的直径：** 给定二叉树根节点 `root`，返回树的直径（任意两节点间最长路径的边数）。路径可能不经过根。
>
> **示例：** `root = [1,2,3,4,5]` → `3`（路径 `[4,2,1,3]` 或 `[5,2,1,3]`）
> **约束：** 节点数 `[1, 10^4]`, `-100 <= Node.val <= 100`

```rust
fn diameter_of_binary_tree(root: Tree) -> i32 {
    let mut ans = 0;

    fn depth(node: &Tree, ans: &mut i32) -> i32 {
        match node {
            None => 0,
            Some(n) => {
                let n = n.borrow();
                let left = depth(&n.left, ans);
                let right = depth(&n.right, ans);
                *ans = (*ans).max(left + right); // 经过当前节点的最长路径
                1 + left.max(right)
            }
        }
    }

    depth(&root, &mut ans);
    ans
}
```

### 例题 5：验证二叉搜索树（LeetCode 98）

> **LeetCode 98. 验证二叉搜索树：** 判断二叉树是否为有效 BST（左子树所有节点值 < 根 < 右子树所有节点值，且左右子树也必须是 BST）。
>
> **示例：** `root = [2,1,3]` → `true`；`root = [5,1,4,null,null,3,6]` → `false`
> **约束：** 节点数 `[1, 10^4]`, `-2^31 <= Node.val <= 2^31-1`

```rust
fn is_valid_bst(root: Tree) -> bool {
    fn validate(node: &Tree, min: i64, max: i64) -> bool {
        match node {
            None => true,
            Some(n) => {
                let n = n.borrow();
                let val = n.val as i64;
                val > min && val < max
                    && validate(&n.left, min, val)
                    && validate(&n.right, val, max)
            }
        }
    }
    validate(&root, i64::MIN, i64::MAX)
}
```

## 七、二叉搜索树 (BST)

**性质：左 < 根 < 右，中序遍历是升序的。**

```rust
// BST 查找
fn search_bst(root: &Tree, val: i32) -> Tree {
    match root {
        None => None,
        Some(node) => {
            let node = node.borrow();
            if val == node.val {
                root.clone()
            } else if val < node.val {
                search_bst(&node.left, val)
            } else {
                search_bst(&node.right, val)
            }
        }
    }
}
```

## 八、测试题

1. **问：** 前序和中序遍历能唯一确定一棵二叉树吗？

   **答：** 能！前序的第一个元素是根，在中序中找到根的位置，就能分出左右子树，然后递归构建。

2. **问：** 递归的时间复杂度怎么分析？

   **答：** 通常每个节点访问一次，所以是 O(n)。空间复杂度取决于递归深度（即树的高度），最坏 O(n)（链状树），平均 O(log n)（平衡树）。
