# 05 - 单调栈与单调队列

> "比我矮的都被我挡住了，后面的人不需要看他们。"——单调栈的核心直觉。

## 一、基础知识

### 1.1 核心定义

**单调栈**：元素值始终保持单调（递增/递减）的栈结构。入栈前弹出破坏单调性的元素。**单调队列**：在双端队列中维护窗口内单调性。

两者本质相同——O(n) 时间内为每个元素找到"第一个更大/更小邻居"。

### 1.2 工作原理

核心是**消除无用元素**：求「下一个更大元素」时维护递减栈。遍历到 nums[i] 时，弹出所有更小的栈顶——对之后元素来说，nums[i] 比它们"更高且更近"，被弹出的不再有用。

每个元素最多入栈一次、出栈一次 → **均摊 O(n)**。两重循环（for + while）的总执行次数 = n + n = 2n。

> 💡 **为什么存索引而不是存值？** 因为我们需要知道答案要写回哪个位置（`ans[top] = ...`）。如果只存值，只能知道"有一个元素的下一个更大值是 X"，但不知道"这个元素在原数组的哪个位置"。
> 
> 🔑 **"均摊 O(n)"直觉**：想象一个停车场，每辆车（元素）最多进一次、出一次。虽然看起来有两层循环（每来一辆新车都可能赶走多辆旧车），但每辆车只被赶走一次。总操作 = n 次进 + ≤ n 次出 = O(n)。

### 1.3 适用场景

| 问题 | 结构 | 单调方向 |
|------|------|---------|
| 下一个更大元素 | 单调栈 | 递减（栈底大，栈顶小） |
| 下一个更小元素 | 单调栈 | 递增 |
| 柱状图最大矩形 | 单调递增栈 | 找每个柱子的左右边界 |
| 接雨水 | 单调递减栈 | 形成凹槽时计算 |
| 滑动窗口最大值 | 单调递减队列 | 队头始终最大 |

## 二、从暴力到优化

以「每日温度」(LeetCode 739) 为例。

**暴力 O(n²)**：对每天 i，向右扫描找第一个更高温度。

```rust
fn brute(temps: &[i32]) -> Vec<i32> {
    let mut ans = vec![0; temps.len()];
    for i in 0..temps.len() {
        for j in i+1..temps.len() {
            if temps[j] > temps[i] { ans[i] = (j-i) as i32; break; }
        }
    }
    ans
}
```

**为什么慢**：递减数组 `[99,98,...]` 时，每个 i 都扫到末尾。大量重复比较。

**优化思路**：从右往左看，维护一个"未找到答案的元素索引"列表。这个列表的值保持递减——因为如果一个元素更高且更靠左，它会把右边更矮的元素"挡住"。

```rust
fn daily_temperatures(temps: Vec<i32>) -> Vec<i32> {
    let n = temps.len();
    let mut ans = vec![0i32; n];
    // 存索引而非值：需要把答案写回 ans[top]
    let mut stack: Vec<usize> = vec![];

    for i in 0..n {
        // 当前元素是栈中所有更小元素的"右边第一个更大"
        while let Some(&top) = stack.last() {
            if temps[i] > temps[top] {
                stack.pop();
                ans[top] = (i - top) as i32;
            } else { break; } // 栈内保持递减，一旦遇到不小于的，后面的也都满足条件
        }
        stack.push(i);
    }
    ans
}
```

**手动模拟** `temps = [73,74,75,71,69,72,76,73]`：
```
i=0(73): stack=[] → [0]              ans=[0,0,0,0,0,0,0,0]
i=1(74): 74>73 → pop1 ans[0]=1 → [1]
i=2(75): 75>74 → pop1 ans[1]=1 → [2]
i=3(71): 71<75 → [2,3]
i=4(69): 69<71 → [2,3,4]
i=5(72): 72>69 → pop1 ans[4]=1; 72>71 → pop1 ans[3]=2; 72<75 → stop → [2,5]
i=6(76): 76>72 → pop1 ans[5]=1; 76>75 → pop1 ans[2]=4 → [6]
i=7(73): 73<76 → [6,7]
结果: [1,1,4,2,1,1,0,0] ✓
```

## 三、单调栈核心模板

```rust
/// 右边第一个更大元素
fn next_greater(nums: &[i32]) -> Vec<i32> {
    let n = nums.len();
    let mut ans = vec![-1i32; n]; // -1 表示不存在
    let mut stack: Vec<usize> = vec![];

    for i in 0..n {
        // 弹出所有被当前元素"击败"的：它们右边第一个更大就是 nums[i]
        while let Some(&top) = stack.last() {
            if nums[i] > nums[top] {
                ans[top] = nums[i];
                stack.pop();
            } else { break; }
        }
        stack.push(i);
    }
    ans
}
```

**四种变体**：

| 问题 | 栈类型 | 比较条件 |
|------|--------|---------|
| 右边第一个更大 | 递减栈 | `nums[i] > nums[top]` |
| 右边第一个更小 | 递增栈 | `nums[i] < nums[top]` |
| 左边第一个更大 | 递减栈（反向遍历） | 对称 |
| 左边第一个更小 | 递增栈（正向遍历，栈内即左边） | 对称 |

## 四、单调队列（滑动窗口最大值 LeetCode 239）

```rust
use std::collections::VecDeque;

fn max_sliding_window(nums: Vec<i32>, k: i32) -> Vec<i32> {
    let k = k as usize;
    let n = nums.len();
    let mut ans = Vec::with_capacity(n - k + 1);
    // 存索引：需要判断队头是否滑出窗口
    let mut deq: VecDeque<usize> = VecDeque::new();

    for i in 0..n {
        // 队尾维护递减：比当前元素小的都不可能是后续窗口的最大值
        while let Some(&back) = deq.back() {
            if nums[back] <= nums[i] { deq.pop_back(); }
            else { break; }
        }
        deq.push_back(i);

        // 队头滑出窗口则移除
        if deq.front() == Some(&(i.wrapping_sub(k))) {
            deq.pop_front();
        }

        // 窗口形成后才记录答案
        if i >= k - 1 {
            ans.push(nums[*deq.front().unwrap()]);
        }
    }
    ans
}
```

## 五、接雨水 (LeetCode 42)

```rust
fn trap(height: Vec<i32>) -> i32 {
    let mut stack: Vec<usize> = vec![]; // 递减栈
    let mut water = 0i32;

    for i in 0..height.len() {
        // 当前高度比栈顶高 → 形成凹槽，计算积水量
        while let Some(&mid) = stack.last() {
            if height[i] <= height[mid] { break; }
            stack.pop();
            if let Some(&left) = stack.last() {
                // 凹槽宽度 × 凹槽深度（左右壁的较矮者 - 槽底高度）
                let w = (i - left - 1) as i32;
                let h = height[left].min(height[i]) - height[mid];
                water += w * h;
            }
        }
        stack.push(i);
    }
    water
}
```

## 六、常见错误

| 错误 | 后果 |
|------|------|
| 栈存值而不是索引 | 无法回写答案位置 |
| 忘记维护单调顺序 | 弹出条件写反 |
| 单调队列忘记弹出过期队头 | 返回窗口外的元素 |
| 接雨水忘记检查 `stack.last()` 是否还有左壁 | 空栈时越界 |
