# 07 - 哈希表

> 空间换时间的终极利器。"查找"从 O(n) 降到 O(1) 的秘密。

## 一、基础知识

### 1.1 核心定义

**哈希表**（Hash Table）是一种通过**哈希函数**将键（key）映射到存储桶（bucket）的数据结构。支持 O(1) 平均时间的插入、删除和查找。

Rust 标准库提供两种实现：`HashMap<K, V>`（键值映射）和 `HashSet<T>`（不重复集合）。算法中常用于**计数统计**、**去重**、**快速查找**。

### 1.2 工作原理

哈希表的核心是一个数组（桶），通过对 key 调用哈希函数得到索引：

1. `hash(key)` → 数组索引 → 将 value 存入对应位置
2. 不同 key 可能映射到同一索引（**哈希冲突**），通常用链表法（拉链法）解决：同一索引处维护一个链表
3. 当装载因子（元素数 / 桶数）过高时，触发**rehash**：扩容数组并重新分配所有元素
4. 理想情况下每个桶只有 0~1 个元素，查找只需一次哈希计算 + 一次数组访问 → O(1)

> 💡 **哈希冲突解决的两种方式**：
> - **拉链法（Separate Chaining）**：每个桶是一个链表，冲突时追加到链表尾部。Java 的 HashMap 使用此方法（当链表过长时转为红黑树）。
> - **开放寻址法（Open Addressing）**：冲突时按某种规则探测下一个空桶（如线性探测 `i+1, i+2, ...`）。Python 的 dict、Rust 的 HashMap（从 1.36 版本起使用 Swiss Table / hashbrown 实现）都用此方法。
>
> ⚠️ **为什么装载因子高了要 rehash？** 假设 100 个桶存了 80 个元素（装载因子 0.8），平均每个桶 0.8 个元素。但如果扩展到 200 个桶（装载因子 0.4），每个桶只有 0.4 个元素，链表更短，查找更快。代价是占用更多内存——这就是"空间换时间"的本质。
>
> **Rust HashMap 默认使用的哈希算法**是 SipHash-1-3，它提供了 HashDoS 攻击防护（避免恶意输入制造大量冲突），但比简单哈希稍微慢一点。对于算法竞赛，通常不需要关心这个细节。

### 1.3 适用场景与前提

| 场景 | 技术 | 说明 |
|------|------|------|
| "是否存在" | HashSet | O(1) 查询某个值是否出现过 |
| 频次计数 | `map.entry(k).or_insert(0) += 1` | 统计每个元素出现次数 |
| 两数之和 | 遍历时存 `target - num → index` | 找互补值 |
| 前缀和 + 哈希 | `map` 映射前缀和 → 索引 | 和为 K 的子数组 |
| 字母异位词分组 | 排序后字符串作 key | 将同构字符串聚在一起 |
| 缓存 / 记忆化 | `memo.insert(state, result)` | 避免重复计算 |

**前提**：key 必须实现了 `Hash` 和 `Eq` trait（Rust 自动为基本类型实现）。查找不要求有序——如果需要有序遍历，用 `BTreeMap`（O(log n)）。

### 1.4 优缺分析

**优点：**
- 查找/插入/删除 O(1) 平均时间，远快于线性遍历
- 空间换时间的经典实现，思维模型简单直观
- Rust 的 `entry().or_insert()` API 一次查找即可完成"存在则更新，不存在则插入"

**缺点：**
- 最坏情况下（大量哈希冲突）降级为 O(n)，依赖好的哈希函数
- 无序存储，无法按 key 顺序遍历（需要 `BTreeMap`）
- 空间开销大——需要预分配桶数组，装载因子低时浪费内存
- 哈希计算本身有常数开销，数据量小时不如暴力遍历或固定大小数组（如 `[0; 26]`）

## 二、算法本质

**大白话：哈希表就是一个"字典"——给一个 key，瞬间找到对应的 value。**

就像查新华字典，通过拼音（key）直接翻到对应的页码（value），不用从头翻到尾。

## 三、Rust 中的哈希表

### 2.1 HashMap

```rust
use std::collections::HashMap;

let mut map: HashMap<String, i32> = HashMap::new();

// 插入
map.insert("apple".to_string(), 3);
map.insert("banana".to_string(), 5);

// 查询
if let Some(&count) = map.get("apple") {
    println!("apple: {}", count); // apple: 3
}

// 是否存在
map.contains_key("apple"); // true

// 删除
map.remove("apple");

// 遍历
for (key, value) in &map {
    println!("{}: {}", key, value);
}
```

### 2.2 计数器模式（超高频！）

```rust
use std::collections::HashMap;

// 统计每个元素出现的次数
fn count_elements(arr: &[i32]) -> HashMap<i32, i32> {
    let mut counter = HashMap::new();
    for &x in arr {
        *counter.entry(x).or_insert(0) += 1;
    }
    counter
}
```

**`entry().or_insert()` 是 Rust 最优雅的计数方式。** 含义：
- `entry(x)`：查找 key=x
- `.or_insert(0)`：如果不存在，插入默认值 0
- `+= 1`：对值加 1

### 2.3 HashSet

```rust
use std::collections::HashSet;

let mut set: HashSet<i32> = HashSet::new();

set.insert(1);
set.insert(2);
set.insert(3);
set.insert(1); // 重复，不会重复插入

set.contains(&2); // true
set.len(); // 3

// 集合运算
let a: HashSet<i32> = HashSet::from([1, 2, 3]);
let b: HashSet<i32> = HashSet::from([2, 3, 4]);
// .copied() 把 &i32 转成 i32，这样结果是 HashSet<i32>
let intersection: HashSet<i32> = a.intersection(&b).copied().collect(); // {2, 3}
let union: HashSet<i32> = a.union(&b).copied().collect(); // {1, 2, 3, 4}
```

## 四、哈希表的常见用法

### 用法 1：快速查找——两数之和（LeetCode 1）

```rust
use std::collections::HashMap;

fn two_sum(nums: Vec<i32>, target: i32) -> Vec<i32> {
    let mut map = HashMap::new();
    for (i, &num) in nums.iter().enumerate() {
        if let Some(&j) = map.get(&(target - num)) {
            return vec![j as i32, i as i32];
        }
        map.insert(num, i);
    }
    vec![]
}
```

### 用法 2：去重/存在性检查

```rust
use std::collections::HashSet;

fn contains_duplicate(nums: Vec<i32>) -> bool {
    let set: HashSet<_> = nums.iter().collect();
    set.len() != nums.len()
}
```

### 用法 3：LeetCode 49. 字母异位词分组

> 给定字符串数组 `strs`，将字母异位词（字母相同、顺序不同）分到同一组。
>
> **示例：** `strs = ["eat","tea","tan","ate","nat","bat"]` → `[["bat"],["nat","tan"],["ate","eat","tea"]]`
> **约束：** `1 <= strs.length <= 10^4`

```rust
use std::collections::HashMap;

fn group_anagrams(strs: Vec<String>) -> Vec<Vec<String>> {
    let mut map: HashMap<Vec<u8>, Vec<String>> = HashMap::new();
    for s in strs {
        let mut key: Vec<u8> = s.bytes().collect();
        key.sort(); // 排序后的字符串作为 key
        map.entry(key).or_default().push(s);
    }
    map.into_values().collect()
}
```

### 用法 4：LeetCode 560. 和为 K 的子数组

> 整数数组 `nums` 和整数 `k`，统计和为 `k` 的连续子数组个数（前缀和 + 哈希表）。
>
> **示例：** `nums = [1,1,1], k = 2` → `2`；`nums = [1,2,3], k = 3` → `2`
> **约束：** `1 <= nums.length <= 2*10^4`

```rust
use std::collections::HashMap;

fn subarray_sum(nums: Vec<i32>, k: i32) -> i32 {
    let mut count = 0;
    let mut prefix_sum = 0;
    let mut map = HashMap::new();
    map.insert(0, 1); // 前缀和为0出现了1次

    for &num in &nums {
        prefix_sum += num;
        // 如果存在前缀和 = prefix_sum - k，说明有一段子数组和为 k
        if let Some(&cnt) = map.get(&(prefix_sum - k)) {
            count += cnt;
        }
        *map.entry(prefix_sum).or_insert(0) += 1;
    }
    count
}
```

**这是"前缀和 + 哈希表"的经典组合，非常重要！**

### 用法 5：数组当哈希表（字符计数）

当 key 的范围很小时（如 26 个字母），用数组代替 HashMap 更快。

```rust
fn is_anagram(s: String, t: String) -> bool {
    let mut count = [0i32; 26];
    for b in s.bytes() { count[(b - b'a') as usize] += 1; }
    for b in t.bytes() { count[(b - b'a') as usize] -= 1; }
    count.iter().all(|&c| c == 0)
}
```

## 五、经典例题

### 例题：LeetCode 128. 最长连续序列

> 未排序数组 `nums`，找出最长连续序列的长度（要求 O(n) 时间）。
>
> **示例：** `nums = [100,4,200,1,3,2]` → `4`（`[1,2,3,4]`）
> **约束：** `0 <= nums.length <= 10^5`, `-10^9 <= nums[i] <= 10^9`

```rust
use std::collections::HashSet;

fn longest_consecutive(nums: Vec<i32>) -> i32 {
    let set: HashSet<i32> = nums.into_iter().collect();
    let mut ans = 0;

    for &num in &set {
        // 只从序列的起点开始计数（num-1 不在集合中）
        if !set.contains(&(num - 1)) {
            let mut current = num;
            let mut length = 1;
            while set.contains(&(current + 1)) {
                current += 1;
                length += 1;
            }
            ans = ans.max(length);
        }
    }
    ans
}
```

**关键技巧：** 只从序列的起点开始计数，避免重复工作，时间复杂度 O(n)。

## 六、测试题

1. **问：** HashMap 和 BTreeMap 的区别？

   **答：** HashMap 查找 O(1) 但无序；BTreeMap 查找 O(log n) 但有序。需要排序或范围查询时用 BTreeMap。

2. **问：** 什么时候用数组代替 HashMap？

   **答：** 当 key 的范围小且连续时（如字母 a-z，数字 0-9），用数组 `[0; 26]` 更快更省空间。
