# 03 - 二分查找

> 每次砍一半，把 O(n) 变成 O(log n)。对数级复杂度在竞赛中就是"随便跑"的代名词。

## 一、基础知识

### 1.1 核心定义

**二分查找**在有序序列中通过不断将搜索范围减半来快速定位目标值。更广义的「二分答案」则对答案的值域进行二分，每次验证候选答案是否可行。

本质上利用了**单调性**：如果某位置不满足条件，它一侧的所有位置也必然不满足，每一步可以安全丢弃一半搜索空间。

### 1.2 工作原理

每一步将搜索区间分为两半，根据条件排除一半。二分答案的核心是将"求解"转化为"验证"——check(mid) 返回 true 则所有 >= mid 都可行（单调），安全缩小区间。

### 1.3 适用场景与前提

| 场景 | 前提 | 例子 |
|------|------|------|
| 有序数组查找 | 数组单调有序 | lower_bound / upper_bound |
| 二分答案 | check(x) 有单调性 | 珂珂吃香蕉、D天包裹 |
| 旋转数组 | 一分为二后必有一半有序 | LeetCode 33, 153 |

### 1.4 优缺分析

**优点**：O(log n)，n=10⁹ 仅约 30 次比较；模板固定不易出错
**缺点**：必须有序/有单调性；边界处理（`lo < hi` vs `lo <= hi`）容易踩坑

## 二、从暴力到优化

暴力 O(n) 逐个查找没利用"有序"信息。

**关键洞察**：我们只需要知道中间值和 target 的关系，就能排除一半。

```rust
// 暴力 O(n)：从左扫到右
fn linear_search(arr: &[i32], target: i32) -> Option<usize> {
    for (i, &v) in arr.iter().enumerate() {
        if v == target { return Some(i); }
    }
    None
}

// 二分 O(log n)：每次砍半
fn binary_search(arr: &[i32], target: i32) -> Option<usize> {
    let (mut lo, mut hi) = (0, arr.len()); // [lo, hi) 左闭右开
    while lo < hi {
        // 防溢出：不用 (lo+hi)/2 避免跨过 i32 上限
        let mid = lo + (hi - lo) / 2;
        if arr[mid] == target {
            return Some(mid);
        }
        // mid 及左边全排除（都比 target 小）
        if arr[mid] < target { lo = mid + 1; }
        // mid 及右边全排除（都比 target 大）
        else { hi = mid; }
    }
    None
}
```

## 三、二分查找标准写法

### 左闭右开 `[lo, hi)`（推荐）

```rust
/// 在有序数组中找第一个 >= target 的位置（lower_bound）
fn lower_bound(arr: &[i32], target: i32) -> usize {
    let (mut lo, mut hi) = (0, arr.len());
    while lo < hi {
        let mid = lo + (hi - lo) / 2;
        // arr[mid] < target → 排除左半（mid 及左边必不满足）
        if arr[mid] < target {
            lo = mid + 1;
        // arr[mid] >= target → mid 可能是答案，保留；缩小右边界
        } else {
            hi = mid;
        }
    }
    lo // lo == hi，即分界点
}
```

**过程模拟** `arr = [1,3,5,7,9], target = 5`：
```
lo=0,hi=5 → mid=2, arr[2]=5≥5 → hi=2
lo=0,hi=2 → mid=1, arr[1]=3<5 → lo=2
lo=hi=2 → 答案 2 即 arr[2]=5 ✓
```

**Rust 标准库**：`arr.partition_point(|&x| x < target)` 等价于 `lower_bound`。

### 3.1 upper_bound（第一个 > target 的位置）

```rust
/// 在有序数组中找第一个 > target 的位置（upper_bound）
fn upper_bound(arr: &[i32], target: i32) -> usize {
    let (mut lo, mut hi) = (0, arr.len());
    while lo < hi {
        let mid = lo + (hi - lo) / 2;
        // arr[mid] <= target → 排除左半（mid 及左边 ≤ target，不满足）
        if arr[mid] <= target {
            lo = mid + 1;
        // arr[mid] > target → mid 可能是答案，保留
        } else {
            hi = mid;
        }
    }
    lo
}
```

> **🧭 开区间 vs 闭区间决策指南**：
> 
> | 写法 | 循环条件 | mid 落在左侧 | mid 落在右侧 | 适用场景 |
> |------|---------|-------------|-------------|---------|
> | `[lo, hi)` | `lo < hi` | `lo = mid + 1` | `hi = mid` | **推荐**：边界干净，不易错 |
> | `[lo, hi]` | `lo <= hi` | `lo = mid + 1` | `hi = mid - 1` | 需要精确查找某个值时 |
> 
> **为什么推荐左闭右开**：因为 `hi` 始终指向"第一个不满足条件的元素"，循环结束时 `lo == hi` 就是答案位置，不需要额外判断。

## 四、二分答案

**核心思想**：不对数组二分，对答案值域二分。猜一个答案 mid，用 check(mid) 验证。

```rust
fn binary_search_answer(lo: i64, hi: i64) -> i64 {
    let (mut lo, mut hi) = (lo, hi);
    while lo < hi {
        let mid = lo + (hi - lo) / 2;
        // check(mid) 返回 true 表示 mid 可行，尝试更小的答案
        if check(mid) { hi = mid; }
        else { lo = mid + 1; }
    }
    lo
}
```

**例题**：LeetCode 875. 爱吃香蕉的珂珂——有 `n` 堆香蕉，`h` 小时吃完，每小时选一堆吃掉 `k` 根。求最小速度 `k`。

```rust
fn min_eating_speed(piles: Vec<i32>, h: i32) -> i32 {
    let mut lo = 1i64;
    // 上界：最大堆的香蕉数，因为速度再大也没意义（每小时只能吃一堆）
    let mut hi = *piles.iter().max().unwrap() as i64;

    while lo < hi {
        let mid = lo + (hi - lo) / 2;
        // 以速度 mid 吃完所有香蕉需要的小时数
        let hours: i64 = piles.iter()
            .map(|&p| (p as i64 + mid - 1) / mid) // 向上取整
            .sum();
        // 能在 h 小时内吃完 → 尝试更慢的速度
        if hours <= h as i64 { hi = mid; }
        // 吃不完 → 速度必须更快
        else { lo = mid + 1; }
    }
    lo as i32
}
```

## 五、搜索旋转排序数组 (LeetCode 33)

```rust
fn search(nums: Vec<i32>, target: i32) -> i32 {
    let (mut lo, mut hi) = (0i32, nums.len() as i32 - 1);
    while lo <= hi {
        let mid = lo + (hi - lo) / 2;
        let m = mid as usize;
        if nums[m] == target { return mid; }

        // 判断哪一半有序：nums[lo] <= nums[m] 表示左半有序
        if nums[lo as usize] <= nums[m] {
            // 左半有序且 target 落在左半范围内 → 在左半找
            if nums[lo as usize] <= target && target < nums[m] {
                hi = mid - 1;
            } else { lo = mid + 1; }
        } else {
            // 右半有序且 target 落在右半范围内 → 在右半找
            if nums[m] < target && target <= nums[hi as usize] {
                lo = mid + 1;
            } else { hi = mid - 1; }
        }
    }
    -1
}
```

## 六、常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| `while lo <= hi` 用 `while lo < hi` | 可能死循环或漏判 | 左闭右开用 `<`，左闭右闭用 `<=` |
| `mid = (lo + hi) / 2` | 大数溢出 | `lo + (hi - lo) / 2` |
| 二分答案中 check 函数 O(n) 太慢 | 总复杂度变 O(n log n) | 优化 check 为 O(1) 或 O(log n) |
| 旋转数组中直接使用标准二分 | 无条件判断 | 先判断哪一半有序 |

## 七、面试识别速查

## 八、测试题

1. `lo + (hi - lo) / 2` vs `(lo + hi) / 2`：后者当 lo/hi 很大时可能溢出。
2. 什么条件能用二分答案：check(x) 有单调性——x 可行则更极端的值也一定可行（或一定不可行）。
