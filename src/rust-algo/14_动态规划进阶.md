# 14 - 动态规划进阶

> 区间DP、数位DP、状压DP、树形DP——DP 的高级形态。你已经做过数位DP了（6/5）！

## 一、基础知识

### 1.1 核心定义

动态规划进阶篇涵盖四种经典高级 DP 模式：

- **区间 DP**：状态 `dp[i][j]` 定义在区间 `[i, j]` 上，通过枚举分割点将区间一分为二来转移
- **数位 DP**：统计 `[l, r]` 范围内满足数位条件的数字个数/总和，通过逐位构造数字 + 记忆化实现
- **状压 DP**：用二进制掩码（bitmask）表示"已选集合"，状态转移时切换掩码位
- **树形 DP**：在树上做 DP，通常后序遍历（先处理子树，再汇总到根节点）

### 1.2 工作原理

- **区间 DP**：`dp[i][j] = max/min(dp[i][k] + dp[k+1][j] + cost(i,k,j))`。按区间长度递增递推，保证子区间已计算完毕。
- **数位 DP**：`f(pos, tight, ...)` 递归构造数字，tight 控制上限，记忆化避免重复。核心是 `f(r) - f(l-1)` 前缀差分。
- **状压 DP**：`dp[mask][i]` = 已访问集合为 mask，当前在 i 的最优解。遍历所有 mask（O(2ⁿ)），对每个 mask 枚举下一步。
- **树形 DP**：在 DFS 回溯时计算 `dp[node]`，子节点结果已就绪。常用于"选或不选"（树的独立集）。

### 1.3 适用场景与前提

| DP 类型 | 识别信号 | 复杂度 |
|--------|---------|--------|
| 区间 DP | "合并石子"、"戳气球"——区间操作 | O(n³) |
| 数位 DP | 统计 [l, r] 内满足数位条件的数字 | O(位数 × 10 × 状态数) |
| 状压 DP | n ≤ 20，集合选择问题 | O(n·2ⁿ) |
| 树形 DP | 树上最优选择、路径统计 | O(n) 或 O(n²) |

**前提**：数位 DP 要求数字范围可表达为字符串；状压 DP 要求 n 很小（否则 2ⁿ 不可行）；区间 DP 的 cost 函数要求 O(1)；树形 DP 要求子节点与父节点的状态可合并。

### 1.4 优缺分析

**优点：**
- 区间 DP 通用性强，合并类问题都可套用
- 数位 DP 将 O(range) 暴力降为 O(log n × 常数)，处理 10¹⁵ 以上的范围
- 状压 DP 是处理小规模组合优化的标准方案

**缺点：**
- 区间 DP O(n³) 在 n>500 时较慢，需优化到 O(n²)
- 状压 DP 只能处理 n≤20，指数级增长无法扩展
- 数位 DP 的套路化代码容易写，但调试困难（状态转移的细节繁多）
- 树形 DP 依赖树的遍历顺序，图上的环会导致算法失效

## 二、区间 DP

### 算法本质

**大白话：在一个区间上做决策，每次把区间分成两半来考虑。**

状态定义：`dp[i][j]` = 区间 `[i, j]` 的最优解

### 通用模板

```rust
fn interval_dp(arr: &[i32]) -> i32 {
    let n = arr.len();
    let mut dp = vec![vec![0; n]; n];

    // 枚举区间长度
    for len in 2..=n {
        // 枚举左端点
        for i in 0..=n - len {
            let j = i + len - 1; // 右端点
            dp[i][j] = i32::MAX; // 初始化

            // 枚举分割点
            for k in i..j {
                dp[i][j] = dp[i][j].min(
                    dp[i][k] + dp[k + 1][j] + cost(i, k, j) // cost 是合并代价，根据题目定义
                );
            }
        }
    }
    dp[0][n - 1]
}
```

### 例题：LeetCode 312. 戳气球

> `n` 个气球，戳破第 `i` 个可得 `nums[left]*nums[i]*nums[right]` 金币（`left/right` 为相邻未戳破气球，越界视为 1）。求最大金币。
>
> **示例：** `nums = [3,1,5,8]` → `167`
> **约束：** `n <= 300`, `0 <= nums[i] <= 100`

```rust
fn max_coins(nums: Vec<i32>) -> i32 {
    // 在两端加上虚拟气球 1
    let mut arr = vec![1];
    arr.extend(&nums);
    arr.push(1);
    let n = arr.len();

    // dp[i][j] = 戳破 (i, j) 区间内所有气球的最大金币
    let mut dp = vec![vec![0; n]; n];

    for len in 2..n {
        for i in 0..n - len {
            let j = i + len;
            for k in i + 1..j {
                // k 是最后一个被戳破的气球
                dp[i][j] = dp[i][j].max(
                    dp[i][k] + dp[k][j] + arr[i] * arr[k] * arr[j]
                );
            }
        }
    }
    dp[0][n - 1]
}
```

### 例题：最长回文子序列（LeetCode 516）

```rust
fn longest_palindrome_subseq(s: String) -> i32 {
    let s = s.as_bytes();
    let n = s.len();
    let mut dp = vec![vec![0; n]; n];

    for i in (0..n).rev() {
        dp[i][i] = 1; // 单个字符是回文
        for j in i + 1..n {
            if s[i] == s[j] {
                dp[i][j] = dp[i + 1][j - 1] + 2;
            } else {
                dp[i][j] = dp[i + 1][j].max(dp[i][j - 1]);
            }
        }
    }
    dp[0][n - 1]
}
```

## 三、数位 DP

### 算法本质

**大白话：在 [0, n] 范围内，统计满足某种条件的数的个数。逐位构造数字，跟踪是否还"贴着上界"。**

**你已经做过了！**（6/5 的 LC 3163 数位DP）

### 核心思想

对于 "1 到 n 中有多少个数满足条件X"：
- 从最高位开始，逐位确定
- 维护一个 `is_limit` 标志：当前是否还受上界 n 的限制
- 维护一个 `is_num` 标志：前面是否已经选了非零数字（处理前导零）

### 通用模板

```rust
fn count_special_numbers(n: i32) -> i32 {
    let digits: Vec<u8> = n.to_string().bytes().map(|b| b - b'0').collect();
    let m = digits.len();

    // memo[pos][state] = 从第 pos 位开始，状态为 state 的方案数
    // state_size 根据题目定义，比如用 1024 表示 bitmask 状态空间
    let state_size = 1024;
    let mut memo = vec![vec![-1i32; state_size]; m];

    fn dfs(
        pos: usize,
        state: usize,  // 根据题目定义的状态
        is_limit: bool, // 是否受上界限制
        is_num: bool,   // 前面是否已选数字
        digits: &[u8],
        memo: &mut Vec<Vec<i32>>,
    ) -> i32 {
        if pos == digits.len() {
            return if is_num { 1 } else { 0 }; // 是否构成了有效数字
        }

        // 只有不受限且已经是有效数字时才能用缓存
        if !is_limit && is_num && memo[pos][state] != -1 {
            return memo[pos][state];
        }

        let mut result = 0;

        // 可以选择跳过当前位（即前导零）
        if !is_num {
            result += dfs(pos + 1, state, false, false, digits, memo);
        }

        // 当前位可选的数字范围
        let lo = if is_num { 0 } else { 1 }; // 已经有数字了可以选0，否则从1开始
        let hi = if is_limit { digits[pos] } else { 9 };

        for d in lo..=hi {
            // 根据题目条件，更新 state
            let new_state = state + d as usize; // 举例：根据题目条件更新
            result += dfs(
                pos + 1,
                new_state,
                is_limit && d == hi, // 只有选了最大值才继续受限
                true,
                digits,
                memo,
            );
        }

        if !is_limit && is_num {
            memo[pos][state] = result;
        }
        result
    }

    dfs(0, 0, true, false, &digits, &mut memo)
}
```

### 例题：LeetCode 902. 最大为 N 的数字组合

> 给定数码集合 `digits` 和数字 `n`，统计 `[1, n]` 中能用 `digits` 中的数码组成的数字个数（数位 DP）。
>
> **示例：** `digits = ["1","3","5","7"], n = 100` → `20`
> **约束：** `1 <= digits.length <= 9`, `1 <= n <= 10^9`

```rust
fn at_most_n_given_digit_set(digits: Vec<String>, n: i32) -> i32 {
    let s: Vec<u8> = n.to_string().bytes().map(|b| b - b'0').collect();
    let ds: Vec<u8> = digits.iter().map(|d| d.as_bytes()[0] - b'0').collect();
    let m = s.len();
    let mut memo = vec![-1i32; m];

    fn dfs(pos: usize, is_limit: bool, is_num: bool,
           s: &[u8], ds: &[u8], memo: &mut Vec<i32>) -> i32 {
        if pos == s.len() {
            return if is_num { 1 } else { 0 };
        }
        if !is_limit && is_num && memo[pos] != -1 {
            return memo[pos];
        }
        let mut res = 0;
        if !is_num {
            res += dfs(pos + 1, false, false, s, ds, memo);
        }
        let hi = if is_limit { s[pos] } else { 9 };
        for &d in ds {
            if d > hi { break; }
            res += dfs(pos + 1, is_limit && d == hi, true, s, ds, memo);
        }
        if !is_limit && is_num { memo[pos] = res; }
        res
    }

    dfs(0, true, false, &s, &ds, &mut memo)
}
```

## 四、状态压缩 DP（状压DP）

### 算法本质

**大白话：用一个整数的二进制来表示"选了哪些元素"的状态。**

适用条件：n 很小（通常 n ≤ 20），需要枚举所有子集。

### 例题：最短超级串（旅行商问题变体）

```rust
// dp[mask][i] = 访问了 mask 表示的城市集合，当前在城市 i 的最短路径
fn tsp(dist: &Vec<Vec<i32>>) -> i32 {
    let n = dist.len();
    let full = (1 << n) - 1;
    let mut dp = vec![vec![i32::MAX; n]; 1 << n];

    dp[1][0] = 0; // 从城市0出发

    for mask in 1..=full {
        for u in 0..n {
            if dp[mask][u] == i32::MAX { continue; }
            if mask & (1 << u) == 0 { continue; }

            for v in 0..n {
                if mask & (1 << v) != 0 { continue; } // 已访问
                let new_mask = mask | (1 << v);
                dp[new_mask][v] = dp[new_mask][v].min(dp[mask][u] + dist[u][v]);
            }
        }
    }

    // 回到起点
    let mut ans = i32::MAX;
    for u in 0..n {
        if dp[full][u] != i32::MAX {
            ans = ans.min(dp[full][u].saturating_add(dist[u][0]));
        }
    }
    ans
}
```

### 例题：划分为K个相等的子集（LeetCode 698）

```rust
fn can_partition_k_subsets(nums: Vec<i32>, k: i32) -> bool {
    let sum: i32 = nums.iter().sum();
    if sum % k != 0 { return false; }
    let target = sum / k;

    let n = nums.len();
    let mut dp = vec![-1i32; 1 << n]; // -1: 未访问, 0: 不可行, 1: 可行
    dp[0] = 0; // 空集，当前桶的和为0

    // dp[mask] = 选了 mask 中的数后，当前这个桶里的和 (mod target)
    let mut current_sum = vec![0i32; 1 << n];

    for mask in 0..(1u32 << n) {
        if dp[mask as usize] == -1 { continue; }
        for i in 0..n {
            if mask & (1 << i) != 0 { continue; } // 已选
            let new_mask = mask | (1 << i);
            if dp[new_mask as usize] != -1 { continue; }

            let bucket_sum = current_sum[mask as usize] + nums[i];
            if bucket_sum <= target {
                dp[new_mask as usize] = 1;
                current_sum[new_mask as usize] = bucket_sum % target;
            }
        }
    }
    dp[(1 << n) - 1] == 1
}
```

## 五、树形 DP

### 算法本质

**大白话：在树上做 DP，通常是"从叶子往根汇总信息"（后序遍历）。**

### 例题：打家劫舍 III（LeetCode 337）

> 二叉树上偷东西，不能偷相邻节点。

```rust
use std::rc::Rc;
use std::cell::RefCell;

fn rob(root: Option<Rc<RefCell<TreeNode>>>) -> i32 {
    // 返回 (偷当前节点的最大值, 不偷当前节点的最大值)
    fn dfs(node: &Option<Rc<RefCell<TreeNode>>>) -> (i32, i32) {
        match node {
            None => (0, 0),
            Some(n) => {
                let n = n.borrow();
                let (left_rob, left_skip) = dfs(&n.left);
                let (right_rob, right_skip) = dfs(&n.right);

                let rob_this = n.val + left_skip + right_skip;
                let skip_this = left_rob.max(left_skip) + right_rob.max(right_skip);

                (rob_this, skip_this)
            }
        }
    }

    let (rob, skip) = dfs(&root);
    rob.max(skip)
}
```

## 六、DP 类型速查表

| DP 类型 | 状态特征 | 典型问题 |
|---------|---------|---------|
| 线性 DP | dp[i] | 爬楼梯、打家劫舍、LIS |
| 双序列 DP | dp[i][j] 两个字符串 | 编辑距离、LCS |
| 背包 DP | dp[容量] | 零钱兑换、分割等和子集 |
| 区间 DP | dp[i][j] 区间 | 戳气球、回文子序列 |
| 数位 DP | 逐位构造 + is_limit | 统计特殊数字 |
| 状压 DP | dp[bitmask] | TSP、集合划分 |
| 树形 DP | DFS 后序 | 树上打家劫舍 |

## 七、测试题

1. **问：** 数位 DP 中 `is_limit` 的作用是什么？

   **答：** 标记当前位是否还受上界 n 的限制。如果前面的位都取到了上界的值，当前位最大只能取到 n 对应位的值。否则可以自由取 0-9。

2. **问：** 状压 DP 适用的 n 的范围？

   **答：** n ≤ 20（因为 2^20 = 100万），一般 n ≤ 15 比较安全。
