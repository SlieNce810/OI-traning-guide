# 01 - 复杂度分析与基础

> 学算法的第一步：学会判断代码"快不快"、"省不省内存"。

## 一、基础知识

### 1.1 核心定义

**算法复杂度**是对算法运行效率的数学度量，分为两类：

- **时间复杂度**：算法执行所需的时间随输入规模增长的趋势。不关心具体秒数，关心的是"当 n 翻倍时，运行时间增长了多少倍"。
- **空间复杂度**：算法执行所需的额外内存随输入规模增长的趋势。同样只关心增长趋势。

这种度量方式称为**渐进分析**（Asymptotic Analysis），用大 O 记号表示。它"抓大放小"——忽略常数因子和低阶项，只看最高阶的增长速度。

> **「抓大放小」直观理解**：假设你有 `3n² + 100n + 500` 次操作。当 n=1000 时，`3n²=3,000,000`，`100n=100,000`，`500` 这三项中 `3n²` 占绝对主导（97%）。随着 n 继续增大，后两项的占比趋近于零。因此我们只保留最高阶项并去掉系数，得到 `O(n²)`。

> **📖 术语说明**：
> - **渐进**（Asymptotic）：当 n 趋向无穷大时的行为。我们只关心 n 很大时的趋势，不关心 n 很小的时候。
> - **大 O 记号**（Big-O Notation）：数学上 `f(n) = O(g(n))` 表示存在常数 C 和 N，使得对所有 n ≥ N，都有 `f(n) ≤ C·g(n)`。通俗地说就是"f 的增长速度不超过 g 的某个常数倍"。

### 1.2 工作原理

复杂度分析基于 **RAM (Random Access Machine)** 计算模型：假设每条基本指令（算术运算、内存读写、分支跳转）耗时相同。我们统计这些基本操作的执行次数，再提取其渐进增长率。

> **💡 类比理解 RAM 模型**：想象你是一个厨师，切菜、翻锅、加调料各算"一步"。不管切什么菜，不管锅多大，每一步都计为 1。RAM 模型就是这么简单粗暴——它忽略 CPU 缓存、流水线、SIMD 等真实硬件的复杂性，让我们专注于"算法本身的效率"。

**分析步骤：**
1. 识别输入规模变量 n（数组长度、数字大小等）
2. 逐行计算基本操作的执行次数（循环次数 × 循环体内操作次数）
3. 保留增长最快项的系数为 1，去掉低阶项和常数

> **🔑 均摊分析（Amortized Analysis）**：有些操作单次看很贵，但多次平均下来很便宜。例如滑动窗口中 `left` 指针每次收缩是 O(1)，整个算法中 `left` 总共移动 ≤ n 次，所以总复杂度是 O(n) 而非 O(n²)。详见 [02_双指针](./02_双指针与滑动窗口.md) 中的滑动窗口部分。

### 1.3 适用场景

| 场景 | 需要的复杂度 | 原因 |
|------|-------------|------|
| LeetCode 在线判题 | n ≤ 10⁶ → O(n)；n ≤ 10⁵ → O(n log n) | 判题机每秒约 10⁷~10⁸ 次运算 |
| 实时系统 | 必须是 O(1) 或 O(log n) | 必须在固定时间内响应 |
| 数据分析（离线） | O(n log n) 甚至 O(n²) 可以接受 | 没有实时性要求，数据量可控 |
| 竞赛编程 | 根据数据范围反推算法 | 看到 10⁵ 说明需要 O(n log n)，看到 20 说明可以 O(2ⁿ) |

**关键前提**：复杂度分析只关心趋势，不关心具体常数。一个 O(n) 但常数因子为 1000 的算法，在 n < 1000 时可能比 O(n²) 的算法还慢。

### 1.4 优缺分析

**优点：**
- **通用性好**：和语言、硬件无关，纯数学推导，任何编程语言都适用
- **指导性强**：看到数据范围就能快速排除不可行的算法（"如果 n=10⁵，O(n²) 一定超时"）
- **可叠加**：多步操作的复杂度可以组合（加法规则和乘法规则）

**缺点：**
- **忽略常数**：O(n) 可能因为常数因子 1000 比 O(n log n) 的常数因子 2 更慢
- **不反映实际运行时间**：不考虑缓存命中、分支预测、内存带宽等硬件因素
- **最坏情况不一定代表典型情况**：快排的最坏复杂度是 O(n²)，但实际几乎不会遇到，平均是 O(n log n)

## 二、什么是复杂度？

**一句话：复杂度就是衡量你的代码跑得快不快、占内存多不多的标尺。**

想象你要在一堆扑克牌里找到红心A：
- 方法一：一张张翻，最多翻完所有牌 → **O(n)**
- 方法二：如果牌是排好序的，每次看中间那张，范围缩小一半 → **O(log n)**

复杂度不关心具体花了几秒，而是关心 **当数据量 n 变大时，时间增长的趋势**。

## 三、时间复杂度

### 3.1 常见复杂度（从快到慢）

| 复杂度 | 名称 | 直觉 | n=10^6 时大约运算次数 |
|--------|------|------|---------------------|
| O(1) | 常数 | 不管数据多大，都是固定时间 | 1 |
| O(log n) | 对数 | 每次砍一半 | 20 |
| O(n) | 线性 | 看一遍数据 | 10^6 |
| O(n log n) | 线性对数 | 排序的标准速度 | 2×10^7 |
| O(n²) | 平方 | 两层循环嵌套 | 10^12（超时！）|
| O(2^n) | 指数 | 每个元素选或不选 | 天文数字 |

### 3.2 怎么判断复杂度？

**核心规则：看循环嵌套的层数和循环范围。**

```rust
// O(1) - 常数时间
fn get_first(arr: &[i32]) -> i32 {
    arr[0]  // 不管数组多长，只取第一个
}

// O(n) - 线性时间
fn sum(arr: &[i32]) -> i32 {
    let mut s = 0;
    for &x in arr {  // 遍历一次
        s += x;
    }
    s
}

// O(n²) - 平方时间
fn has_duplicate(arr: &[i32]) -> bool {
    for i in 0..arr.len() {       // 外层 n 次
        for j in i+1..arr.len() { // 内层也接近 n 次
            if arr[i] == arr[j] {
                return true;
            }
        }
    }
    false
}

// O(log n) - 对数时间
fn binary_search(arr: &[i32], target: i32) -> bool {
    let (mut lo, mut hi) = (0, arr.len());
    while lo < hi {          // 每次范围缩小一半
        let mid = lo + (hi - lo) / 2;
        if arr[mid] == target { return true; }
        if arr[mid] < target { lo = mid + 1; }
        else { hi = mid; }
    }
    false
}
```

### 3.3 实用判断口诀

- **一个循环 0..n** → O(n)
- **两层循环嵌套** → O(n²)
- **每次范围缩小一半** → O(log n)
- **一层循环 × 二分** → O(n log n)
- **递归 + 每次砍一半** → O(log n)
- **排序** → O(n log n)

> **🔑 递归算法的复杂度分析——主定理（Master Theorem）**：
> 
> 对于形如 `T(n) = a·T(n/b) + f(n)` 的递归：
> - 归并排序：`a=2, b=2, f(n)=O(n)` → `T(n) = O(n log n)`
> - 二分查找：`a=1, b=2, f(n)=O(1)` → `T(n) = O(log n)`
> - 二叉树遍历：`a=2, b=2, f(n)=O(1)` → `T(n) = O(n)`
> 
> **记忆技巧**：递归每层总工作量 × 层数。归并排序每层 O(n) 共 log n 层 = O(n log n)。

### 3.4 LeetCode 的时间限制经验

一般来说，LeetCode 允许的运算次数约为 **10^7 ~ 10^8** 次。

| n 的范围 | 能用的最大复杂度 |
|---------|----------------|
| n ≤ 20 | O(2^n) 或 O(n!) |
| n ≤ 1000 | O(n²) |
| n ≤ 10^5 | O(n log n) |
| n ≤ 10^6 | O(n) |
| n ≤ 10^9 | O(log n) 或 O(1) |

**这个表非常重要！看到题目的数据范围，就能倒推应该用什么复杂度的算法。**

## 四、空间复杂度

**一句话：你的代码额外开了多少内存。**

```rust
// O(1) 空间 - 只用了几个变量
fn max_val(arr: &[i32]) -> i32 {
    let mut mx = arr[0];
    for &x in &arr[1..] {
        mx = mx.max(x);
    }
    mx
}

// O(n) 空间 - 开了一个同等大小的数组
fn prefix_sum(arr: &[i32]) -> Vec<i32> {
    let mut pre = vec![0; arr.len() + 1];
    for i in 0..arr.len() {
        pre[i + 1] = pre[i] + arr[i];
    }
    pre
}
```

### 5.4 迭代器（Rust 的瑞士军刀）

> 迭代器是 Rust 中处理集合的核心方式。`v.iter()` 创建迭代器，后面可以链式调用各种操作。
> 闭包 `|x|` 是匿名函数的写法，`|` 之间是参数，后面是函数体。

```rust
let v = vec![1, 2, 3, 4, 5];

// 求和
let sum: i32 = v.iter().sum();

// 过滤 + 映射
// |&&x| 中的 && 是因为 filter 给的是 &&i32（迭代器给 &i32，filter 再加一层 &）
// 刷题中如果搞不清楚 & 的层数，可以写 |x| **x % 2 == 0 来解引用
let even_squares: Vec<i32> = v.iter()
    .filter(|&&x| x % 2 == 0)  // 保留偶数
    .map(|&x| x * x)            // 求平方
    .collect();                  // 收集成 Vec

// 最大值
let mx = v.iter().max().unwrap();

// 枚举（带索引）
for (i, &x) in v.iter().enumerate() {
    println!("v[{}] = {}", i, x);
}
```

## 六、前缀和（最基础的优化技巧）

**问题：** 反复查询数组某个区间 `[l, r]` 的和。

**暴力：** 每次查询遍历区间求和 → O(n) 每次查询
**前缀和：** 预处理一次后，每次查询 O(1)

```rust
// 构建前缀和数组
fn build_prefix_sum(arr: &[i32]) -> Vec<i32> {
    let n = arr.len();
    let mut pre = vec![0; n + 1];
    for i in 0..n {
        pre[i + 1] = pre[i] + arr[i];
    }
    pre
    // pre[i] = arr[0] + arr[1] + ... + arr[i-1]
}

// 查询区间和 [l, r]（0-indexed）
fn range_sum(pre: &[i32], l: usize, r: usize) -> i32 {
    pre[r + 1] - pre[l]
}
```

**直觉：** 就像里程碑。要知道A站到B站的距离，不用重新量，只需要用B站的里程数减去A站的里程数。

## 七、差分数组

**问题：** 反复对数组某个区间 `[l, r]` 所有元素加上一个值。

```rust
// 区间加操作：给 [l, r] 每个元素加 val
fn range_add(diff: &mut Vec<i32>, l: usize, r: usize, val: i32) {
    diff[l] += val;
    if r + 1 < diff.len() {
        diff[r + 1] -= val;
    }
}

// 从差分数组还原原数组
fn restore(diff: &[i32]) -> Vec<i32> {
    let mut arr = vec![0; diff.len()];
    arr[0] = diff[0];
    for i in 1..diff.len() {
        arr[i] = arr[i - 1] + diff[i];
    }
    arr
}
```

**直觉：** 差分是前缀和的逆操作。前缀和是"累加"，差分是"只记录变化量"。

## 八、例题

### 例题：LeetCode 1. 两数之和

> 给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出和为目标值 `target` 的那两个整数，并返回它们的数组下标。假设每种输入只会对应一个答案，不能使用两次相同的元素。
>
> **示例 1：** 输入 `nums = [2,7,11,15], target = 9` → 输出 `[0,1]`
> **示例 2：** 输入 `nums = [3,2,4], target = 6` → 输出 `[1,2]`
> **约束：** `2 <= nums.length <= 10^4`, `-10^9 <= nums[i] <= 10^9`

```rust
use std::collections::HashMap;

fn two_sum(nums: Vec<i32>, target: i32) -> Vec<i32> {
    let mut map = HashMap::new();
    for (i, &num) in nums.iter().enumerate() {
        let complement = target - num;
        if let Some(&j) = map.get(&complement) {
            return vec![j as i32, i as i32];
        }
        map.insert(num, i);
    }
    vec![]
}
```

**复杂度分析：**
- 时间：O(n)，遍历一次数组
- 空间：O(n)，哈希表最多存 n 个元素

## 九、测试题（检验理解）

1. 以下代码的时间复杂度是多少？
```rust
for i in 0..n {
    for j in i..n {
        // 操作
    }
}
```
答案：O(n²)。内层循环虽然从 i 开始，但总次数是 n + (n-1) + ... + 1 = n(n+1)/2，还是 O(n²)。

2. 以下代码的时间复杂度是多少？
```rust
let mut i = 1;
while i < n {
    i *= 2;
}
```
答案：O(log n)。i 每次翻倍，到 n 需要 log₂(n) 次。

3. 如果 n = 100000，以下哪个复杂度会超时？
   - A) O(n log n) → 约 170万 ✅ 不超时
   - B) O(n²) → 约 100亿 ❌ **超时！**
   - C) O(n√n) → 约 3100万 ✅ 不超时
