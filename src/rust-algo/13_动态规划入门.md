# 13 - 动态规划入门

> 动态规划（DP）是算法中最重要也最难的专题。掌握它，你就能解决一大半的算法题。

## 一、基础知识

### 1.1 核心定义

**动态规划**（Dynamic Programming, DP）是一种通过将问题分解为重叠子问题，并存储子问题的结果来避免重复计算的算法范式。名字中的"规划"不是编程，而是"填表"（用表格记录中间结果）。

DP 的两个核心特质：
- **最优子结构**：原问题的最优解可以由子问题的最优解构成
- **重叠子问题**：不同分支会反复遇到相同的子问题（这是 DP 区别于分治的关键——分治的子问题不重叠）

### 1.2 工作原理

DP 的两种实现思路：

1. **自顶向下（记忆化搜索）**：从大问题递归，用数组/HashMap 记录已计算结果。遇到重复子问题时直接返回缓存值。
2. **自底向上（递推）**：从最小的子问题开始，逐步递推到原问题。通常用循环 + DP 数组实现。

三步法解 DP 题：
1. 定义状态：`dp[i]` 或 `dp[i][j]` 代表什么含义？
2. 推导转移方程：`dp[i]` 怎样从更小的子问题推导出来？
3. 确定边界：最小的子问题答案是什么？

### 1.3 适用场景与前提

| DP 类型 | 特征 | 例子 |
|--------|------|------|
| 线性 DP | 单维度状态，沿数组推进 | 打家劫舍、爬楼梯 |
| 背包 DP | 选择物品，容量限制 | 0-1背包、完全背包 |
| 双序列 DP | 两个字符串/数组的最优匹配 | LCS、编辑距离 |
| LIS | 最长递增子序列 | O(n²) 或 O(n log n) |

**前提**：
- 问题能分解为子问题，且子问题之间有重叠（否则递归/分治就够了）
- 状态转移具有无后效性：当前状态只依赖之前的状态，不依赖之后的决策
- 状态维度不能太多（通常 ≤3，否则时空复杂度爆炸）

> 🔑 **无后效性（No Aftereffect）详解**：这是 DP 的核心前提。意思是——一旦确定了某个子问题的状态，这个状态之后怎么来的（即"历史"）不影响后续的决策。后续决策只关心当前状态是什么，不关心你是怎么到这个状态的。
> 
> **正例**（无后效性 → 可用 DP）：打家劫舍中，`dp[i]` 只关心"前 i 个房子的最优方案"，不关心具体偷了哪几个。
> **反例**（有后效性 → 需增加状态维度）：旅行商问题中如果加约束"不能连续访问同一颜色的城市"，则需要记住最近访问的城市颜色——这会增加状态维度（状压 DP 可以通过增加状态位来解决）。注意：标准 TSP 本身也是通过状压 DP（增加"已访问集合"维度）消除后效性的。

### 1.4 优缺分析

**优点：**
- 将指数级暴力降为多项式时间，是解决最优化问题的杀手锏
- 通用性强——能用 DP 的题目类型广泛（数组、字符串、树、图）
- 一旦写出正确状态转移方程，代码通常很短

**缺点：**
- 状态定义和转移方程的推导是最大难点，没有固定套路
- 空间复杂度可能很高（O(n²) 或更高），需要滚动数组优化
- 容易和贪心混淆——不确定时要用 DP 验证
- 调试困难——中间状态不直观

## 二、算法本质

**大白话：动态规划就是"记住以前算过的结果，避免重复计算"。**

举个例子：计算斐波那契数列 fib(5)。

**暴力递归（大量重复计算）：**
```
fib(5) = fib(4) + fib(3)
fib(4) = fib(3) + fib(2)  ← fib(3) 被算了两次！
fib(3) = fib(2) + fib(1)  ← fib(2) 被算了三次！
```

**动态规划（记住结果，只算一次）：**
```
fib(1) = 1
fib(2) = 1
fib(3) = fib(2) + fib(1) = 2  ← 查表，不重复算
fib(4) = fib(3) + fib(2) = 3
fib(5) = fib(4) + fib(3) = 5
```

## 三、DP 的两种实现方式

### 方式一：记忆化搜索（自顶向下）

从大问题出发，递归拆分，用数组/HashMap 记录已算过的子问题。

```rust
fn fib_memo(n: usize, memo: &mut Vec<i64>) -> i64 {
    if n <= 1 { return n as i64; }
    if memo[n] != -1 { return memo[n]; } // 已算过，直接返回

    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo);
    memo[n]
}

// 使用
let mut memo = vec![-1i64; n + 1];
let result = fib_memo(n, &mut memo);
```

### 方式二：递推（自底向上）

从最小的子问题开始，逐步构建答案。

```rust
fn fib_dp(n: usize) -> i64 {
    if n <= 1 { return n as i64; }
    let mut dp = vec![0i64; n + 1];
    dp[0] = 0;
    dp[1] = 1;
    for i in 2..=n {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    dp[n]
}
```

### 哪种更好？

| | 记忆化搜索 | 递推 |
|--|----------|-----|
| 思维方式 | 自然（先想大问题） | 需要想清楚顺序 |
| 代码风格 | 递归 | 循环 |
| 效率 | 稍慢（递归开销） | 稍快 |
| 适合场景 | 状态空间不规则 | 状态空间规则 |

**建议：先用记忆化搜索想清楚，再翻译成递推。**

## 四、解 DP 题的三步法

**每道 DP 题都要回答三个问题：**

1. **状态定义：** `dp[i]` 或 `dp[i][j]` 代表什么？
2. **状态转移方程：** `dp[i]` 怎么从更小的问题推导出来？
3. **初始条件和边界：** 最小的子问题的答案是什么？

## 五、线性 DP

### 例题 1：LeetCode 70. 爬楼梯

> 爬 `n` 阶楼梯，每次可爬 1 或 2 阶，有多少种不同方法？
>
> **示例：** `n = 2` → `2`；`n = 3` → `3`
> **约束：** `1 <= n <= 45`

**三步法：**
1. 状态：`dp[i]` = 到第 i 层的方法数
2. 转移：`dp[i] = dp[i-1] + dp[i-2]`（从 i-1 爬1步 或 从 i-2 爬2步）
3. 初始：`dp[0] = 1, dp[1] = 1`

```rust
fn climb_stairs(n: i32) -> i32 {
    if n <= 1 { return 1; }
    let n = n as usize;
    let (mut a, mut b) = (1, 1);
    for _ in 2..=n {
        let c = a + b;
        a = b;
        b = c;
    }
    b
}
```

### 例题 2：LeetCode 198. 打家劫舍

> 沿街偷窃，相邻房屋有警报，求不触发警报能偷到的最高金额。
>
> **示例：** `nums = [1,2,3,1]` → `4`；`nums = [2,7,9,3,1]` → `12`
> **约束：** `1 <= nums.length <= 100`, `0 <= nums[i] <= 400`

**三步法：**
1. 状态：`dp[i]` = 前 i 个房子能偷的最大金额
2. 转移：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
   - 不偷第 i 个：`dp[i-1]`
   - 偷第 i 个：`dp[i-2] + nums[i]`（不能偷 i-1）
3. 初始：`dp[0] = nums[0], dp[1] = max(nums[0], nums[1])`

```rust
fn rob(nums: Vec<i32>) -> i32 {
    let n = nums.len();
    if n == 1 { return nums[0]; }

    let mut dp = vec![0; n];
    dp[0] = nums[0];
    dp[1] = nums[0].max(nums[1]);
    for i in 2..n {
        dp[i] = dp[i - 1].max(dp[i - 2] + nums[i]);
    }
    dp[n - 1]
}

// 空间优化版（只需要前两个值）
fn rob_optimized(nums: Vec<i32>) -> i32 {
    let (mut prev2, mut prev1) = (0, 0);
    for &num in &nums {
        let current = prev1.max(prev2 + num);
        prev2 = prev1;
        prev1 = current;
    }
    prev1
}
```

### 例题 3：LeetCode 300. 最长递增子序列

> 找数组中最长严格递增子序列的长度（元素不必连续）。
>
> **示例：** `nums = [10,9,2,5,3,7,101,18]` → `4`（`[2,3,7,101]`）
> **约束：** `1 <= nums.length <= 2500`, `-10^4 <= nums[i] <= 10^4`

**方法一：O(n²) DP**

```rust
fn length_of_lis(nums: Vec<i32>) -> i32 {
    let n = nums.len();
    let mut dp = vec![1; n]; // dp[i] = 以 nums[i] 结尾的 LIS 长度

    for i in 1..n {
        for j in 0..i {
            if nums[j] < nums[i] {
                dp[i] = dp[i].max(dp[j] + 1);
            }
        }
    }
    *dp.iter().max().unwrap()
}
```

**方法二：O(n log n) 贪心 + 二分**

```rust
fn length_of_lis_fast(nums: Vec<i32>) -> i32 {
    let mut tails = vec![]; // tails[i] = 长度为 i+1 的 LIS 的末尾最小值

    for &num in &nums {
        let pos = tails.partition_point(|&x| x < num);
        if pos == tails.len() {
            tails.push(num); // 比所有末尾都大，LIS 长度+1
        } else {
            tails[pos] = num; // 替换为更小的值（贪心）
        }
    }
    tails.len() as i32
}
```

**直觉：** 我们维护一个数组 tails，让每个位置的值尽可能小，这样后面才有更多机会接上更大的数。

## 六、背包 DP

### 0-1 背包

> 有 n 个物品，每个物品有重量 w 和价值 v。背包容量 W，每个物品只能选一次，求最大价值。

```rust
fn knapsack_01(weights: &[i32], values: &[i32], capacity: usize) -> i32 {
    let n = weights.len();
    // dp[j] = 容量为 j 时的最大价值
    let mut dp = vec![0; capacity + 1];

    for i in 0..n {
        let w = weights[i] as usize;
        let v = values[i];
        // 倒序遍历！保证每个物品只用一次
        for j in (w..=capacity).rev() {
            dp[j] = dp[j].max(dp[j - w] + v);
        }
    }
    dp[capacity]
}
```

**为什么倒序？** 如果正序，`dp[j-w]` 可能已经被更新（包含了物品 i），导致物品 i 被选了多次。

### 完全背包

> 和 0-1 背包一样，但每个物品可以选无限次。

```rust
fn knapsack_complete(weights: &[i32], values: &[i32], capacity: usize) -> i32 {
    let n = weights.len();
    let mut dp = vec![0; capacity + 1];

    for i in 0..n {
        let w = weights[i] as usize;
        let v = values[i];
        // 正序遍历！允许物品被重复选择
        for j in w..=capacity {
            dp[j] = dp[j].max(dp[j - w] + v);
        }
    }
    dp[capacity]
}
```

### 例题：LeetCode 322. 零钱兑换

> 用不同面额硬币（无限量）凑成 `amount`，求最少硬币数。无法凑成返回 -1（完全背包）。
>
> **示例：** `coins = [1,2,5], amount = 11` → `3`；`coins = [2], amount = 3` → `-1`
> **约束：** `1 <= coins.length <= 12`, `0 <= amount <= 10^4`

```rust
fn coin_change(coins: Vec<i32>, amount: i32) -> i32 {
    let amount = amount as usize;
    let mut dp = vec![i32::MAX; amount + 1];
    dp[0] = 0; // 凑0元需要0个硬币

    for &coin in &coins {
        let c = coin as usize;
        for j in c..=amount {
            if dp[j - c] != i32::MAX {
                dp[j] = dp[j].min(dp[j - c] + 1);
            }
        }
    }
    if dp[amount] == i32::MAX { -1 } else { dp[amount] }
}
```

### 例题：LeetCode 416. 分割等和子集

> 判断能否将正整数数组分成两个和相等的子集（等价于 0-1 背包存在性：能否凑成 sum/2）。
>
> **示例：** `nums = [1,5,11,5]` → `true`；`nums = [1,2,3,5]` → `false`
> **约束：** `1 <= nums.length <= 200`, `1 <= nums[i] <= 100`

```rust
fn can_partition(nums: Vec<i32>) -> bool {
    let sum: i32 = nums.iter().sum();
    if sum % 2 != 0 { return false; }
    let target = sum as usize / 2;

    let mut dp = vec![false; target + 1];
    dp[0] = true;

    for &num in &nums {
        let n = num as usize;
        for j in (n..=target).rev() {
            dp[j] = dp[j] || dp[j - n];
        }
    }
    dp[target]
}
```

## 七、记忆化搜索模板

很多 DP 题用记忆化搜索写起来更直观。

```rust
fn solve(nums: &[i32]) -> i32 {
    let n = nums.len();
    let mut memo = vec![vec![-1; n]; n]; // 根据状态维度定义

    fn dfs(nums: &[i32], i: usize, j: usize, memo: &mut Vec<Vec<i32>>) -> i32 {
        // 边界条件
        if i > j { return 0; }
        // 查缓存
        if memo[i][j] != -1 { return memo[i][j]; }
        // 递归计算（根据具体题目写状态转移）
        let result = todo!("在这里写状态转移方程");
        memo[i][j] = result;
        result
    }

    dfs(nums, 0, n - 1, &mut memo)
}
```

## 八、空间优化技巧

当 `dp[i]` 只依赖 `dp[i-1]`（前一行），可以用滚动数组优化空间。

```rust
// 原始版本：O(n) 空间
let mut dp = vec![0; n];

// 优化版本：O(1) 空间，只保留需要的变量
let (mut prev, mut curr) = (0, 0);
```

## 九、测试题

1. **问：** DP 和贪心的区别？

   **答：** DP 考虑所有可能的方案，找最优解（一定正确）。贪心只选当前最优（不一定全局最优）。如果贪心能解决就用贪心（更快），否则用 DP。

2. **问：** 0-1 背包和完全背包在代码上的区别？

   **答：** 只有一行不同！0-1 背包**倒序**遍历容量（保证物品只用一次），完全背包**正序**遍历（允许重复使用）。

3. **问：** 什么时候用记忆化搜索？什么时候用递推？

   **答：** 状态空间不规则、不好确定遍历顺序时用记忆化搜索。状态规则、维度低时用递推更高效。
