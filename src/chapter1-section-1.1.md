# 1.1 思维的体操

## 例题4  墓地雕塑（Graveyard, NEERC 2006, CodeForces Gym100287G）

### 题目描述

在一个周长为 10000 的圆形墓地上，最初均匀分布着 n 个雕塑。现在需要再添加 m 个新雕塑，使得所有 n+m 个雕塑在圆周上均匀分布。需要移动原有的 n 个雕塑（新雕塑位置可任意选择），求最小的总移动距离。结果保留 4 位小数（即把圆周长度 10000 单位化后，输出四舍五入到 4 位小数）。

**输入格式**：多组数据，每组一行两个整数 n 和 m（2 ≤ n ≤ 1000，1 ≤ m ≤ 1000）。文件读到 EOF 结束。

**输出格式**：每组数据输出一行，一个保留 4 位小数的实数，表示最小总移动距离。

### 解题思路

**1. 问题的对称性分析**

由于雕塑是圆形排列的，圆形具有旋转对称性。因此，我们可以选择**固定一个原有的雕塑不移动**，只移动其余 n-1 个雕塑。这个选择不会影响最终答案——因为无论选择哪个雕塑固定，相对移动距离是一样的。

**2. 坐标缩放的巧思 —— 把"均匀分布"变成"对齐整数"**

原问题圆周长 10000，雕塑位置和距离都是浮点数，不方便。核心技巧：把圆周缩放，让最终 target 位置恰好落在整数坐标上。

具体思路：最终有 n+m 个雕塑均匀分布，我们可以把圆周长度**视为** n+m。这样一来，n+m 个均匀分布的点恰好就是位置 0, 1, 2, ..., n+m-1。每个目标位置都是**整数**！

**示例（n=2, m=2, 圆周缩放到 4）：**
- 原来：2 个雕塑在位置 0 和 5000（周长为 10000）
- 缩放后：圆周缩放到长度 4，2 个原有雕塑在位置 0 和 `1/2 × 4 = 2`
- 最终目标：4 个雕塑均匀放在 0, 1, 2, 3
- 原来在位置 2 的雕塑，目标是最近的整数位置（2），不用移动 → 贪心正确
- 结果还原：缩放中 0 距离还原到原始 = 0/4 × 10000 = 0

**3. 贪心确定目标位置**

缩放后，第 i 个原有雕塑（i = 1, 2, ..., n-1）在缩放坐标系中的位置是：
**`pos = (i / n) × (n+m)`**

这个 pos 表示"如果原来均匀分布，雕塑应该在缩放坐标系中的哪个位置"。由于最终目标位置就是整数 0, 1, 2, ..., n+m-1，雕塑只需移动到最近的整数位置：`round(pos)`。

**为什么四舍五入（贪心）总是最优？** 因为目标位置是等间距的整数点，每个雕塑独立地找最近的整数就是全局最优。

**4. 计算移动距离**

缩放坐标系中的距离 = |pos - round(pos)|

但它只是比例，需要还原到周长 10000 的真实世界：
真实距离 = `|pos - round(pos)| / (n+m) × 10000`

**5. 总结公式**

```
for i = 1 to n-1:
    pos = i / n * (n+m)
    ans += |pos - round(pos)| / (n+m)
ans *= 10000
```

**直观理解**：如果把圆周"拉长"或"缩短"到 n+m 的长度，那么均匀放置 n+m 个雕塑就变成了"在整数位置上放雕塑"。缩放是个数学技巧，让浮点数判断变成了整数判断。

### 算法方法

**1. 坐标缩放变换**
- 将周长为 10000 的圆缩放为单位长度的圆再缩放为 n+m，简化计算
- 通过浮点数运算保持精度

**2. 贪心策略**
- 每个原有雕塑移动到最近的均匀分布位置（四舍五入）
- 由于圆形对称性，固定一个雕塑不动不影响全局最优

### 复杂度分析

- **时间复杂度**：O(n)。对于每个测试用例，只需要一重循环遍历 n-1 个雕塑，每个雕塑 O(1) 计算距离。总复杂度为 O(n)。
- **空间复杂度**：O(1)。只使用常数个变量（n, m, ans, pos），不需要额外数组。

```cpp
// 例题4  墓地雕塑（Graveyard, NEERC 2006, CodeForces Gym100287G）
// Rujia Liu
#include<cstdio>
#include<cmath>
using namespace std;

int main() {
  // NEERC 2006 使用文件输入输出
  freopen("graveyard.in", "r", stdin);
  freopen("graveyard.out","w",stdout);

  // 多组测试数据，每组两个整数 n(原有雕塑数) 和 m(新增雕塑数)
  for(int n, m; scanf("%d%d", &n, &m) == 2; ) {
    double ans = 0.0;  // 累计最小总移动距离（在缩放坐标系中）
    // ---- 贪心阶段：逐个处理需要移动的雕塑 ----
    for(int i = 1; i < n; i++) {
      // 坐标缩放：将圆周长度从 (n+m) 映射，计算第i个原有雕塑在缩放坐标系中的理想位置
      // pos 不一定是整数，因为原有位置在新均匀分布中不一定对齐
      double pos = (double)i / n * (n+m); //计算每个需要移动的雕塑的坐标
      // 四舍五入到最近整数：floor(pos+0.5) 即 round(pos)
      // |pos - round(pos)| 是缩放坐标系中的移动距离
      // 除以 (n+m) 还原到原始长度比例
      ans += fabs(pos - floor(pos+0.5)) / (n+m); //累加移动距离
    }
    // 乘以 10000 映射回原始周长的实际距离，保留 4 位小数输出
    printf("%.4lf\n", ans*10000); //等比例扩大坐标
  }
  return 0;
}
```

## 例题6  立方体成像（Image Is Everything, World Finals 2004, UVa1030）

### 题目描述

有一个 n×n×n（1 ≤ n ≤ 10）的立方体，其中一些小方块被涂上了颜色（颜色用大写字母 A-Z 表示），其余方块可能是透明的。我们从立方体的 6 个方向（前、后、左、右、上、下）分别拍一张照片，每张照片是一个 n×n 的网格，记录从该方向看过去看到的第一个非透明方块的颜色，如果视线穿透所有方块则是透明（用 `.` 表示）。初始时所有方块的颜色是未知的，目标是确定立方体中**最多可能有多少个不透明的方块**（即不是 `.` 的方块数）。

**输入格式**：多组数据。每组第一行为整数 n，n=0 时结束。接下来依次给出 6 个视图，每个视图是一个 n×n 的字符矩阵（没有空格），字符为大写字母或 `.`。

**输出格式**：每组输出一行 `Maximum weight: X gram(s)`，其中 X 为最多的非透明方块数。

### 解题思路

**几何直观：我们从 6 个方向拍照片**

想象一个透明的 n×n×n 立方体，里面有些方块涂了颜色（不透明），有些是空的。从 6 个面（前、后、左、右、上、下）拍照片，每张照片记录视线碰到的第一个有色方块的颜色——如果视线穿透整个立方体都没碰到东西，照片上就是 `.`。

**核心挑战**：只凭 6 张照片，能推测里面最多有多少方块吗？

**1. 三维坐标变换 —— 把"照片上的像素"映射到"立方体里的位置"**

这是一个关键难点。想象你站在立方体前面，照片上的第 i 行第 j 列——这个像素对应立方体里哪条视线？

`get(k, i, j, len, x, y, z)` 函数就是做这个映射的。6 张照片对应 6 种映射公式：
- **前视图（k=0）**：从 x 轴正方向看进去，x 从 0 到 n-1（从近到远），y=j，z=i
- **左视图（k=1）**：从 y 轴正方向看进去，y 从 0 到 n-1，x 递减（n-1 到 0），z=i
- ... 以此类推

**直观理解**：对于每张照片的每个像素 (i,j)，沿着视线方向遍历深度 len=0 到 n-1。len=0 是立方体表面，len=n-1 是最深处。get() 函数把 (照片编号, 行, 列, 深度) 映射到立方体里的 (x, y, z) 坐标。

**2. 两步法求解**

**第一步：删除确定空的位置**

如果某张照片上某个像素是 `.`（透明），说明沿着那条视线，从表面一直到最深处，一路都是透明的——对应立方体里的一整排方块都是空的。把这些位置全都标记为 `.`。

**第二步：迭代消除冲突 (类似"推理游戏")**

对于照片上有颜色的像素，沿着视线从外到内找：
- 遇到 `.`（空洞）→ 继续往深处找
- 遇到 `#`（不知道）→ 先把颜色填上（根据照片），这条视线就解决了
- 遇到**相同颜色**→ 匹配成功，视线解决
- 遇到**不同颜色**→ 矛盾！这个位置不可能是这个颜色，删掉它（设为 `.`），标记"有变化"

因为删除一个方块可能导致其他视线的"匹配"变成"冲突"，所以需要反复迭代，直到没有新的变化。

**类比**：这就像玩数独——填写一个格子可能触发其他格子的约束，需要反复检查直到稳定。

**3. 最终统计**

遍历整个立方体，数一数有多少个不是 `.` 的方块。

### 算法方法

**1. 三维坐标变换映射**
- 6 个方向的视图坐标到世界坐标的转换
- 关键技巧：用 `get()` 函数统一处理不同方向的坐标映射

**2. 约束传播 / 迭代消除**
- 类似约束满足问题（CSP）中的 arc consistency 思想
- 不断利用视图信息消除矛盾，直到系统稳定
- 每次冲突消除可能引发新的冲突，需要循环处理

**3. 暴力枚举配合剪枝**
- n ≤ 10，n³ ≤ 1000，可以暴力处理每个位置

### 复杂度分析

- **时间复杂度**：理论最坏O(n⁶)（外层迭代最多 n³ 次，每次 O(6×n³)），但由于每个位置最多被删除一次、实际迭代次数远小于理论最坏情况。n ≤ 10 时最大运算量约 10⁶，完全可接受。
- **空间复杂度**：O(n³)。需要两个三维数组 `pos[n][n][n]` 和 `view[6][n][n]`，n ≤ 10。

```cpp
// 例题6  立方体成像（Image Is Everything, World Finals 2004, UVa1030）
// Rujia Liu
#include<cstdio>
#include<cstring>
#include<cmath>
#include<algorithm>
using namespace std;

#define REP(i,n) for(int i = 0; i < (n); i++)

const int maxn = 10;
int n;
// pos[x][y][z]：立方体的三维状态
//   '.' = 确定透明/不存在
//   '#' = 尚未确定
//   大写字母 = 确定的颜色
char pos[maxn][maxn][maxn];
// view[k][i][j]：第k个视图的第i行第j列看到的颜色
// k=0:前, 1:左, 2:后, 3:右, 4:顶, 5:底
char view[6][maxn][maxn];

// 读取一个有效字符（大写字母或'.'），跳过换行、空格等
char read_char() {
  char ch;
  for(;;) {
    ch = getchar();
    if((ch >= 'A' && ch <= 'Z') || ch == '.') return ch;
  }
}

// 视图坐标 (k, i, j) 和深度 len 转换为三维世界坐标 (x, y, z)
// len=0 是最外层（表面），len=n-1 是最内层
// 6个方向各有不同的坐标变换公式
void get(int k, int i, int j, int len, int &x, int &y, int &z)
{
  if (k == 0) { x = len;    y = j;         z = i;         }  // 前视图：从x轴正方向看
  if (k == 1) { x = n-1-j;  y = len;       z = i;         }  // 左视图：从y轴正方向看
  if (k == 2) { x = n-1-len;y = n-1-j;     z = i;         }  // 后视图：从x轴负方向看
  if (k == 3) { x = j;      y = n-1-len;   z = i;         }  // 右视图：从y轴负方向看
  if (k == 4) { x = n-1-i;  y = j;         z = len;       }  // 顶视图：从z轴正方向看
  if (k == 5) { x = i;      y = j;         z = n-1-len;   }  // 底视图：从z轴负方向看
}

int main() {
  while(scanf("%d", &n) == 1 && n) {
    // ---- 输入阶段：读取 6 个视图 ----
    REP(i,n) REP(k,6) REP(j,n) view[k][i][j] = read_char();
    // ---- 初始化：所有位置初始未知（用'#'标记） ----
    REP(i,n) REP(j,n) REP(k,n) pos[i][j][k] = '#';

    // ---- 第一阶段：删除所有视图中可见的透明位置 ----
    // 如果某个视图的某个位置是'.'，则该视线路径上的所有位置都是透明的
    REP(k,6) REP(i,n) REP(j,n) if (view[k][i][j] == '.')
      REP(p,n) {  // 沿深度方向（从表面到内部）
        int x, y, z;
        get(k, i, j, p, x, y, z);
        pos[x][y][z] = '.';  // 标记为透明/不存在
      }

    // ---- 第二阶段：迭代消除冲突 ----
    for(;;) {
      bool done = true;  // 标记本轮是否有变化，无变化则说明已稳定
      REP(k,6) REP(i,n) REP(j,n) if (view[k][i][j] != '.') {
        // 对视图中每个非空位置，沿深度方向检查
        REP(p,n) {
          int x, y, z;
          get(k, i, j, p, x, y, z);
          if (pos[x][y][z] == '.') continue;        // 透明位置，继续往深处找
          if (pos[x][y][z] == '#') {                 // 未确定的位置
            pos[x][y][z] = view[k][i][j];            // 根据视图信息确定颜色
            break;                                   // 找到第一个非透明位置，停止
          }
          if (pos[x][y][z] == view[k][i][j]) break;  // 颜色匹配，无需再深入
          // 位置颜色与视图颜色冲突 → 该位置不能存在，删除它
          pos[x][y][z] = '.';
          done = false;  // 有变化，需要继续迭代
        }
      }
      if(done) break;  // 没有新冲突，算法收敛
    }

    // ---- 统计阶段：计算所有非透明方块的数量 ----
    int ans = 0;
    REP(i,n) REP(j,n) REP(k,n)
      if (pos[i][j][k] != '.') ans ++;

    printf("Maximum weight: %d gram(s)\n", ans);
  }
  return 0;
}
// 25875758 1030 Image Is Everything Accepted C++ 0.000 2020-12-22 14:26:41
```

## 例题5  蚂蚁（Piotr's Ants, UVa 10881）

### 题目描述

一根长度为 L 厘米的水平木棍上有 n 只蚂蚁，每只蚂蚁以每秒 1 厘米的速度向左或向右爬行。当两只蚂蚁在木棍上相遇时，它们会立即同时掉头，以相同的速度继续向相反方向爬行。当蚂蚁爬到木棍端点（0 或 L）时，会立即掉下木棍。给定每只蚂蚁的初始位置和朝向，求 T 秒后每只蚂蚁的位置和状态。

**输入格式**：第一行一个整数 K（最多 100 组），表示测试用例数。每个测试用例第一行三个整数 L, T, n（1 ≤ L ≤ 10⁹，1 ≤ T ≤ 10⁹，1 ≤ n ≤ 10000）。接下来 n 行，每行一个整数 p（位置，0 ≤ p ≤ L）和一个字符（`L` 或 `R`），表示蚂蚁的初始位置和朝向。

**输出格式**：对于每个测试用例，输出 `Case #X:`，然后按输入顺序输出 n 行，每行表示对应蚂蚁的状态：`p L/R`（位置和朝向）、`Turning`（正在相遇转身）、`Fell off`（已掉落）。

### 解题思路

**1. 等价变换：相遇转身 → 对穿而过**

这是本题最核心的观察。当两只蚂蚁相遇时，它们同时转身。由于**所有蚂蚁没有区别**（从外部观察，蚂蚁之间不可区分），我们可以将"相遇转身"等价替换为"对穿而过"——即两只蚂蚁互不干扰，继续沿原方向前进。在 T 秒后，每只蚂蚁的**最终位置集合**与对穿模型的结果完全一致。

**2. 身份映射：相对顺序不变**

虽然可以等价为对穿，但题目要求输出**每只特定蚂蚁**的状态（按照输入顺序），所以不能简单用对穿计算位置。关键性质：**蚂蚁的相对顺序不会改变**——即如果蚂蚁 A 初始在蚂蚁 B 左边，那么最终 A 仍然在 B 左边（除非它们都掉下去了）。

**3. 三步法处理**

- **计算终态位置**：假设所有蚂蚁对穿而过，每只蚂蚁最终位置 = 初始位置 + T × 方向（向左减 T，向右加 T）。不管身份，将这些位置排序。
- **建立身份映射**：将初始蚂蚁按位置排序，记录每只蚂蚁在排序后的位置序号 `order[i]`。由于相对顺序不变，排序后的第 k 只蚂蚁最终也在终态排序的第 k 个位置。
- **处理相遇状态**：在终态中，如果两只相邻蚂蚁在同一位置，说明它们正在相遇，状态设为 `Turning`。

### 算法方法

**1. 等价变换（对穿模型）**
- 将"相遇转身"转化为"互不干扰对穿"，简化位置计算
- 数学本质：蚂蚁不可区分性 + 速度大小相等

**2. 排序 + 相对位置不变性**
- 利用排序建立初始和终态之间的身份映射
- 蚂蚁的相对顺序是碰撞过程中的不变量

**3. 双指针 / 相邻检测**
- 检测相邻位置相同的蚂蚁，标记为 `Turning` 状态

### 复杂度分析

- **时间复杂度**：O(n log n)。主要开销是两次排序（初始位置排序和终态位置排序），每次 O(n log n)。其他操作都是一重循环 O(n)。
- **空间复杂度**：O(n)。需要两个大小为 n 的 Ant 数组和 order 数组。

```cpp
// 例题5  蚂蚁（Piotr's Ants, UVa 10881）
// Rujia Liu
#include <algorithm>
#include <cstdio>
using namespace std;
const int maxn = 10000 + 5;

// Ant 结构体：描述一只蚂蚁的状态
struct Ant {
  int id;  // 输入顺序，用于建立身份映射
  int p;   // 位置（厘米）
  int d;   // 朝向。-1: 向左; 0:转身中/Turning; 1:向右
  bool operator<(const Ant& a) const { return p < a.p; }  // 按位置排序
} before[maxn], after[maxn];  // before: 初始状态; after: T秒后终态
const char dirName[][10] = {"L", "Turning", "R"};  // d+1 映射到显示文本
int order[maxn];  // order[i]: 输入的第i只蚂蚁在终态排序中的序号（第order[i]个位置）

int main() {
  int K; scanf("%d", &K);
  for (int kase = 1, L, T, n; kase <= K; kase++) {
    scanf("%d%d%d", &L, &T, &n);

    // ---- 输入阶段：读取每只蚂蚁的初始位置和朝向 ----
```

## 例题1  勇者斗恶龙（The Dragon of Loowater, UVa 11292）

### 题目描述

王国里有一条恶龙，它有 n 个头（1 ≤ n ≤ 20000），每个头的直径为 A[i]（1 ≤ A[i] ≤ 20000）。国王雇佣了一批骑士，共 m 个（1 ≤ m ≤ 20000），每个骑士的能力值为 B[j]（1 ≤ B[j] ≤ 20000）。能力值为 x 的骑士可以砍掉任意一个直径不超过 x 的头，但需要支付 x 个金币。每个骑士只能砍一个头，且每个头只能被砍一次。请判断是否能用这些骑士砍掉龙的所有头，如果能，求出最少需要的金币数。

**输入格式**：多组数据。每组第一行两个整数 n 和 m（n=m=0 时结束）。接下来 n 行每行一个整数表示头直径 A[i]，然后 m 行每行一个整数表示骑士能力 B[j]。

**输出格式**：如果无法砍掉所有头，输出 `Loowater is doomed!`；否则输出最小金币数。

### 解题思路

**1. 贪心策略的直觉**

核心贪心原则：**用能力最小的够用骑士，去砍直径最小的头**。理由：
- 如果用一个能力强的骑士砍一个容易的头，那么这个骑士就浪费了——一个较小的头被大材小用了
- 用小的骑士砍小的头，把大的骑士留给大的头，最合理

**2. 排序 + 双指针扫描**

- 将头直径数组 A 和骑士能力数组 B 分别升序排序
- 用指针 cur 指向当前需要砍的头（从小到大），指针 i 遍历骑士（从小到大）
- 如果当前骑士 B[i] 能砍掉头 A[cur]（即 B[i] ≥ A[cur]），就雇佣该骑士，cur 前进，代价加上 B[i]
- 如果 B[i] < A[cur]，说明当前骑士能力不足，跳过
- 当 cur == n 时，所有头已砍完，输出总代价
- 如果遍历完所有骑士后 cur < n，则无法砍完，输出 doomed

**3. 为什么贪心是正确的？**

假设存在一个最优解，其中能力值最小的头没有被能力值最小的够用骑士所砍。我们可以将"砍这个头的骑士"与"最小够用骑士"交换，总代价不会增加（因为最小够用骑士的能力值 ≤ 原来的骑士）。通过不断交换，最优解也能变为贪心解。

### 算法方法

**1. 贪心策略**
- 核心：最小头配最小够用的骑士
- 体现"局部最优 → 全局最优"的经典贪心思想

**2. 排序**
- 对两个数组分别排序是贪心扫描的前提
- 排序后可以用简单的双指针实现

**3. 双指针 / 贪心扫描**
- cur 追踪需要砍的头，i 遍历可用骑士
- 两个指针单调移动，实现 O(n+m) 的线性扫描

### 复杂度分析

- **时间复杂度**：O(n log n + m log m)。两个数组各排序一次分别消耗 O(n log n) 和 O(m log m)，之后的双指针扫描只需 O(n + m)。
- **空间复杂度**：O(n + m)。需要两个数组存储头直径和骑士能力，各 O(20000) = O(1)（常数上限）。

```cpp
// Rujia Liu
#include<cstdio>
#include<algorithm>       // 因为用到了sort
using namespace std;

const int maxn = 20000 + 5;
int A[maxn], B[maxn];  // A: 头的直径; B: 骑士的能力值
int main() {
  int n, m;  // n: 头的数量; m: 骑士的数量
  while(scanf("%d%d", &n, &m) == 2 && n && m) {
    for(int i = 0; i < n; i++) scanf("%d", &A[i]);  // 输入所有头的直径
    for(int i = 0; i < m; i++) scanf("%d", &B[i]);  // 输入所有骑士的能力值

    // ---- 排序阶段：为贪心扫描做准备 ----
    sort(A, A+n);  // 头按直径从小到大排序
    sort(B, B+m);  // 骑士按能力从小到大排序

    // ---- 贪心扫描阶段 ----
    int cur = 0;         // 当前需要砍掉的头的编号（已排序，从小到大处理）
    int cost = 0;        // 当前累计花费的金币数
    for(int i = 0; i < m; i++)  // 遍历所有骑士（从能力最小的开始）
      if(B[i] >= A[cur]) {      // 贪心判断：当前骑士能力 ≥ 当前头的直径
        cost += B[i];           // 雇佣该骑士，花费其能力值金币
        if(++cur == n) break;   // 如果头已经砍完，及时退出循环（优化）
      }
    // 注意：如果 B[i] < A[cur]，说明骑士能力不足，直接跳过，继续找更强的骑士

    // ---- 输出阶段 ----
    if(cur < n) printf("Loowater is doomed!\n");  // 头没砍完，任务失败
    else printf("%d\n", cost);                     // 成功，输出最小总花费
  }
  return 0;
}
```

## 例题3  分金币（Spreading the Wealth, UVa 11300）

### 题目描述

n 个人围坐成一个圆圈（n ≤ 1000000），每人初始有一些金币（总数保证能被 n 整除）。每人可以将一些金币给相邻的人（左手边或右手边，双向均可）。经过若干次转手后，最终每个人拥有的金币数相等。求最少需要转手的金币总数（即所有转移金币的绝对值之和）。

**输入格式**：多组数据。每组第一行为整数 n，接下来 n 行每行一个整数 A[i]（1 ≤ A[i] ≤ 10⁹），表示第 i 个人的初始金币数。文件读到 EOF 结束。

**输出格式**：每组输出一行，一个整数，表示最少转手金币总数。

### 解题思路

**从一个具体例子开始**

假设有 4 个人，金币分别为 A=[8, 4, 1, 7]，总和为 20。最终每人应有 M = 20/4 = 5 个金币。
- 第 1 人多了 3 个（A[1]-M = 8-5 = +3）
- 第 2 人少了 1 个（A[2]-M = 4-5 = -1）
- 第 3 人少了 4 个（A[3]-M = 1-5 = -4）
- 第 4 人多了 2 个（A[4]-M = 7-5 = +2）

目标是让每个人的"多"和"少"通过相邻转移来抵消，且总转移量最小。

**1. 数学建模：用变量表示转移量**

我们用 `x[i]` 表示第 i 个人**给**第 i+1 个人的金币数（x[4] 表示第 4 个人给第 1 个人）。如果 x[i] < 0，表示**反向转移**（第 i+1 个人给第 i 个人）。

关键等式：**第 i 个人最终金币 = 初始金币 - 给出量 + 收到量**
`A[i] - x[i] + x[i-1] = M`（必须等于平均值）

整理得：`x[i] = x[i-1] + A[i] - M`

**2. 递推展开：把一切用 x[0] 表达**

这个公式告诉我们：只要知道了 x[0]（记为 x₀），x[1] 就确定了；知道 x[1]，x[2] 就确定了……所有 x[i] 都能用 x₀ 表示！

```
x[1] = x₀ + (A[1] - M)
x[2] = x₁ + (A[2] - M) = x₀ + (A[1]-M) + (A[2]-M)
x[3] = x₂ + (A[3] - M) = x₀ + (A[1]-M) + (A[2]-M) + (A[3]-M)
...
```

设 `C[i] = (A[1]-M) + (A[2]-M) + ... + (A[i]-M)`，即前 i 个人的"盈余/赤字"累加和（C[0]=0）。则：

**`x[i] = x₀ + C[i]`**

举个例子（A=[8,4,1,7], M=5）：
- C[0] = 0
- C[1] = (8-5) = +3
- C[2] = +3 + (4-5) = +2
- C[3] = +2 + (1-5) = -2
- C[4] = -2 + (7-5) = 0 ← 也必须是 0（总和为 0 的必然结果）

所以 `x[0]=x₀`, `x[1]=x₀+3`, `x[2]=x₀+2`, `x[3]=x₀-2`, `x[4]=x₀+0`。

**3. 目标函数：最小化总转移量**

总转移量 = |x[0]| + |x[1]| + |x[2]| + |x[3]| = **|x₀-0| + |x₀+3| + |x₀+2| + |x₀-2|**

注意 x[4] = x₀ 其实等于第 4 人给第 1 人的量，但公式最终只涉及 C[0..n-1]。
更标准的形式：最小化 `|x₀ - (-C[0])| + |x₀ - (-C[1])| + ... + |x₀ - (-C[n-1])|`

或等价地：最小化 **`sum(|x₀ + C[i]|)`**

这就是"在数轴上找一个点 x₀，使它到 n 个点 (-C[0], -C[1], ..., -C[n-1]) 的距离之和最小"。

在我们的例子中：要使 |x₀-0| + |x₀+3| + |x₀+2| + |x₀-2| 最小。4 个点排序后为 -3, -2, 0, 2，中间位置在 -1 到 0 之间。取 x₀ ≈ -1，总距离 = 1+2+1+3 = 7。

**4. 中位数的引入**

**为什么取中位数？** 在数轴上，如果一个点在 n 个点的中位数位置，那么向左移一格会使左边 n/2 个点距离各减 1，右边 n/2 个点各增 1，总值不变或增大。数学上可以证明：**中位数是距离和的最小值点**。

具体做法：对 C 数组排序，取 `C[n/2]`（中间位置的值）。求 sum of |C[i] - C[n/2]|。

**5. 最后一步：还原答案**

取 `x₀ = -C[n/2]`（或等价地，在代码中直接对 C 排序后，用 `x1 = C[n/2]` 代入），则每个 x[i] = x₀ + C[i] 计算出来，它们的绝对值之和就是答案。

**直观总结**：把每个人的"多出来/少了"的钱依次累加成一条曲线 C[0..n-1]，然后在数轴上找一个点让所有 C[i] 的绝对距离和最小——这个点就是中位数。

### 算法方法

**1. 数学建模与递推**
- 将物理过程的优化问题转化为代数方程
- 利用递推将所有变量用 x[0] 表示

**2. 中位数**
- 将目标函数 "sum |x[0] + C[i]|" 转化为"数轴距离和最小"问题
- 最优解在 C 数组的中位数处达到

**3. 排序**
- 对 C 数组排序以找到中位数，O(n log n)

### 复杂度分析

- **时间复杂度**：O(n log n)。递推计算 C 数组 O(n)，排序 O(n log n)，求和 O(n)。
- **空间复杂度**：O(n)。需要存储 A 和 C 数组，各 O(n)，n 最大 10⁶。

```cpp
// Rujia Liu
#include<cstdio>
#include<algorithm>
using namespace std;

const int maxn = 1000000 + 10;
long long A[maxn], C[maxn], tot, M;
// A[i]: 第i个人的初始金币数 (使用long long防止溢出)
// C[i]: 递推数组，C[i] = C[i-1] + A[i] - M
// tot: 总金币数; M: 平均值(目标每人金币数)

int main() {
  int n;
  while(scanf("%d", &n) == 1) { // 多组数据，输入数据大，scanf比cin快
    // ---- 第一阶段：计算总和与平均值 ----
    tot = 0;
    for(int i = 1; i <= n; i++) {
      scanf("%lld", &A[i]); // 用%lld读取long long
      tot += A[i];          // 累加总金币数
    }
    M = tot / n;  // 平均值：最终每人应拥有的金币数（保证能整除）

    // ---- 第二阶段：递推计算C数组 ----
    C[0] = 0;  // 基准点
    for(int i = 1; i < n; i++)
      C[i] = C[i-1] + A[i] - M; // 递推公式：C[i] = C[i-1] + A[i] - M
    // 此时 x[i] = x[0] + C[i]，C[i] = C[i-1] + A[i] - M（递推结果）

    // ---- 第三阶段：找中位数（C数组的中位数是x1的最优值） ----
    sort(C, C+n);  // 对C数组排序，以便取中位数

    printf("%lld\n", ans);
```

## 例题2  突击战（Commando War, UVa 11729）

### 题目描述

有 n 个士兵需要执行任务（n ≤ 1000）。每个任务 i 包含两个阶段：
1. **交代阶段**：长官向士兵 i 交代任务，需要 B[i] 分钟（长官同一时间只能向一个士兵交代）
2. **执行阶段**：士兵 i 独立执行任务，需要 J[i] 分钟（多个士兵可以同时执行各自的任务）

求从开始交代到所有任务完成的最短总时间。

**输入格式**：多组数据。每组第一行为 n（n=0 时结束）。接下来 n 行，每行两个整数 B[i] 和 J[i]（1 ≤ B[i], J[i] ≤ 10000）。

**输出格式**：每组输出一行 `Case X: Y`，其中 X 为测试用例编号，Y 为最短完成时间。

### 解题思路

**1. 问题的本质：流水线调度**

任务按顺序交代（B[i] 串行），但执行可以并行（J[i] 并行）。这是一个典型的流水线调度（Pipeline Scheduling）问题。任务 i 的完成时间 = 开始交代时间 + B[i] + J[i]。

**2. 贪心策略：按执行时间降序排列**

关键结论：**让执行时间 J[i] 长的任务优先被交代**，即按 J[i] 从大到小排序。

**为什么这样贪心是正确的？** 考虑两个相邻任务 A 和 B，用 B[A], J[A] 和 B[B], J[B] 表示。假设 A 排在 B 前面：
- A 的完成时间：S + B[A] + J[A]
- B 的完成时间：S + B[A] + B[B] + J[B]

两人的完成时间的最大值 = max(S + B[A] + J[A], S + B[A] + B[B] + J[B])

如果交换二者顺序（B 在 A 前）：
- 最大值 = max(S + B[B] + J[B], S + B[B] + B[A] + J[A])

比较两种情况，可以发现：**当 J[A] < J[B] 时，让 B 先执行更优**。因为 B 的执行时间长，早早开始执行可以与其他任务并行，从而缩短总时间。

可以更形式化地用**交换论证法**证明：如果存在两个相邻任务 i 和 i+1 且 J[i] < J[i+1]，交换它们不会使总时间变长。

**3. 模拟计算完成时间**

排序后，按顺序计算每个任务的结束时间：
- 设 s 为当前累积的交代时间（即当前任务开始交代的时刻）
- 对于任务 i：交代结束于 s + B[i]，任务完成于 s + B[i] + J[i]
- 更新 s = s + B[i]，记录 ans = max(ans, s + J[i])

### 算法方法

**1. 贪心调度策略**
- 核心原则：按执行时间 J 降序排序
- 属于"最短处理时间优先（SPT）"的变体——这里是最长处理时间优先

**2. 交换论证 (Exchange Argument)**
- 贪心算法的一种经典证明方法
- 证明任何非贪心的安排都可以通过交换相邻逆序对来改进或保持不变

**3. 线性扫描模拟**
- 排序后只需 O(n) 时间计算完成时间

### 复杂度分析

- **时间复杂度**：O(n log n)。主要是排序开销，n ≤ 1000，非常快。
- **空间复杂度**：O(n)。需要存储 n 个任务的 B 和 J 信息。

```cpp
// 例题2  突击战（Commando War, UVa 11729）
// 陈锋
#include <algorithm>
#include <cstdio>
#include <iostream>
#include <vector>
using namespace std;

// Job 结构体：每个任务包括交代时间 b 和执行时间 j
struct Job {
  int j, b;  // j: 执行时间; b: 交代时间
  bool operator<(const Job& x) const {
    return j > x.j;  // 按执行时间降序排序（关键贪心策略！）
    // 不要忘记const修饰符，这是C++运算符重载的要求
  }
};

int main() {
  ios::sync_with_stdio(false), cin.tie(0);  // 加速输入输出
  for (int n, b, j, kase = 1; cin >> n && n; kase++) {
    vector<Job> v(n);
    for (int i = 0; i < n; i++)
      cin >> v[i].b >> v[i].j;  // 读取每个任务的交代时间和执行时间

    // ---- 贪心排序阶段：按执行时间降序 ----
    sort(v.begin(), v.end());  // 使用Job类的 < 运算符排序
    // 排序后：执行时间长的任务优先被交代，使其尽早进入并行执行阶段

    // ---- 模拟计算阶段 ----
    int ans = 0;  // 记录所有任务完成的最晚时间
    for (int i = 0, s = 0; i < n; i++) {
      s += v[i].b;                 // 累计交代时间：当前任务开始执行前的时刻
      // 任务 i 的完成时间 = 交代结束时刻 + 执行时间 = s + v[i].j
      ans = max(ans, s + v[i].j);  // 更新最晚完成时间
    }
    // 注：s 是当前任务交代结束的时刻，之前的任务执行可能尚未完成
    // 但交代是串行的，s 单调递增
    printf("Case %d: %d\n", kase, ans);
  }
  return 0;
}
```
