# 第5章 图论算法与模型

图论是算法竞赛中题目最多、模型最丰富的专题之一。本章的目标不仅是讲解各个图论算法的实现细节，更是让读者掌握图论问题的建模能力——很多问题表面上与图无关，但一旦你能识别出"节点是什么、边是什么、目标是什么"，就可以调用图论的标准解法。这一建模步骤，往往比算法实现本身更重要。

本章六小节的编排按照"遍历→结构→优化"的递进逻辑。基础题目选讲一节通过若干看似与图无关的问题（如猜序列），展示了图论建模的核心思想——将题目中的约束关系转化为图中的边，然后问题就变成了拓扑排序或差分约束。这种"从约束到图"的翻译能力，是图论入门的第一个关隘。

深度优先遍历一节系统性地讲解了DFS在图论中的结构发现能力。DFS不仅是一种遍历方式，更是一种"解剖"图的工具——通过记录节点的发现时间和完成时间，你可以提取出DFS树、判断边的类型（树边/回边/前向边/横叉边），并在此基础上推导出图中的强连通分量和双连通分量。2-SAT问题则是这些结构的直接应用：将布尔变量和它的否定分别作为两个节点，蕴含关系作为有向边，那么"是否存在满足所有约束的赋值"就等价于"强连通分量中是否同时包含某个变量和它的否定"——这种将逻辑问题转化为图论结构问题的能力，是本节的核心收获。

最短路问题一节从Dijkstra算法的正确性证明出发——为什么每次选取当前距离最小的节点就能保证最短路的全局最优？答案在于"所有边权非负"这一前提保证了最短路的贪心性质。Bellman-Ford算法则从另一个角度解决最短路：通过N-1轮松弛操作发现所有最短路径，并利用第N轮是否还能松弛来判断负环。Floyd算法将最短路问题抽象为"经过哪些中间节点"的决策过程，其三重循环的简洁性背后是动态规划思想。本节通过低价空中旅行等例题展示了当图的状态不是原始节点而是"节点×附加信息"的复合状态时，最短路问题就可能变得非平凡——状态图的构造才是真正的难点。

生成树相关问题一节的核心思想是"最小生成树不是最终答案，而是一个可以调整的起点"。Prim和Kruskal算法是构造MST的基础，但更有趣的是次小生成树的构造——只需枚举不在MST中的每条边，加入后形成的环上去掉最大边即可，这一O(N²)的朴素方法比试图一次性算出次小MST更清晰也更容易理解。最小树形图（朱刘算法）则处理了有向图中的最小生成树问题，其"缩环"操作的设计非常精妙。

二分图匹配一节将匹配问题上升到图论模型的高度。匈牙利算法的增广路思想（交替路、翻转匹配）是理解二分图匹配的标准入口。更重要的是一系列等价关系：在二分图中，最大匹配数等于最小点覆盖数，而最小点覆盖的补集是最大独立集，这些等价关系使得很多看似不相关的问题可以通过二分图匹配解决——关键在于识别出问题中的"二分"结构。KM算法则处理带权匹配问题，其核心在于"势能"（顶标）的设计：为每个节点引入势能使得所有边权转化为非负的"超额值"，并在调整势能的过程中不断逼近最优匹配。

网络流问题一节从最大流问题出发，讲解Ford-Fulkerson方法的核心——增广路——以及Dinic算法如何通过分层图避免DFS反复遍历同一条链。最小割定理揭示了最大流与最小割的等价性，这是网络流建模的核心洞察：很多约束满足问题可以建模为"选择A还是B"的割问题，其中边的容量代表"选择不同侧的代价"，求最小割就是在最小化总代价。费用流则在最大流的基础上引入了"每条边有单位流量费用"的约束，其算法框架（连续最短路增广）展示了如何在寻找增广路时兼顾流平衡和费用最小两个目标。本节通过足球联赛等例题，展示了网络流建模的威力：问题的结构不必天然是图，只要存在"源→分配→消耗→汇"的四段式结构，就可以尝试流建模。
