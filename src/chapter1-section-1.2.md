# 1.2 问题求解常见策略

> **学习目标**：学完本节后，你将掌握算法竞赛最核心的四大策略（分治、贪心、二分答案、搜索），不仅会写代码，更理解每种策略的证明逻辑和适用条件。

## 理论基础

### 为什么需要学这个？

假设你刷了几十道算法题，看到"最大值最小"知道要二分，看到"最优策略"知道要贪心，但真到了考场上——题目换了个包装，你就拿不准了：这个贪心策略到底对不对？二分的时候判定函数该写什么？搜索怎么剪枝才能不超时？

算法的"策略"和"实现"之间有一道关键的鸿沟：**你不仅要会用，还要会证明**。贪心法如果不加证明就用，十个里有五个会错。二分答案的单调性如果不验证，边界条件就写不对。搜索如果不设计剪枝，数据稍微大一点就 TLE。

本节从四个维度系统讲解解题策略：**分治法**给你递归思考的框架，**贪心法**教你交换论证的证明套路，**二分答案**让你学会设计判定函数，**搜索法**帮你掌握剪枝的节奏。这些方法论是你解题工具箱中最核心的部分，后续几乎所有章节都会反复用到。

### 核心概念

**分治法——递归结构模板**：
- 一句话定义：将原问题分解为若干规模较小的子问题，递归求解后再合并结果。
- 最小示例：
  ```
  // 正整数序列问题：每次操作可以将某些数减1，问最少操作次数
  solve(n):
      if n == 1: return 1
      return solve(n/2) + 1   // 每次可以减去一半的数
  ```
- 本质理解：分治的核心不是"分"，而是"治"——找到子问题与原问题之间的递归关系式。汉诺塔问题的递推公式 `f(n) = 2*f(n-1) + 1` 就是这种关系的完美体现。
- **递归树与Master定理的直观理解**：要估算分治法的时间复杂度，画递归树是最直观的方法：
  ```
  f(n)               ← 第0层：1个规模n的子问题
  ├── f(n/2)         ← 第1层：2个规模n/2的子问题
  │   ├── f(n/4)     ← 第2层：4个规模n/4的子问题
  │   └── ...
  └── f(n/2)         ← 每层层数增加，规模指数缩小
  ```
  对于典型的 `T(n) = a*T(n/b) + O(n^d)` 形式，Master定理的直觉其实只有一句话：**哪一层做的工作最多，复杂度就由哪一层主导**。
  - 如果 d > log_b(a)：合并步骤是瓶颈 → O(n^d)，例如归并排序每一层合并O(n)，共log n层→O(n log n)
  - 如果 d = log_b(a)：各层均等贡献 → O(n^d · log n)
  - 如果 d < log_b(a)：叶子（递归底）是瓶颈 → O(n^{log_b(a)})，例如"正整数序列"每次递归规模减半a=1,b=2,d=0→O(log n)
  **不必死记公式**，画出递归树，数一数每层的工作量，复杂度就清楚了。

**贪心法——交换论证法**：
- 一句话定义：用"假设存在逆序对，交换后不会更差"的论证方式，证明贪心策略的正确性。
- 最小示例：突击战问题中，若 J[A] < J[B] 但 A 排在 B 前，交换后总时间不增。
  ```
  假设排列中存在相邻任务 i, i+1 且 J[i] < J[i+1]：
      交换前完成时间 = max(s+B[i]+J[i], s+B[i]+B[i+1]+J[i+1])
      交换后完成时间 = max(s+B[i+1]+J[i+1], s+B[i]+B[i+1]+J[i])
      → 交换后 ≤ 交换前  （通过分类讨论可证）
  ```
- 本质理解：交换论证的套路是"假设最优解中存在一个不如贪心解的地方，证明调整后不更差"。这是竞赛中最常用的贪心证明方法。
- **交换论证的通用证明模板**：在考场上自证贪心策略，或读题解时验证正确性，可以套用这个五步模板：
  ```
  第1步：设贪心解为 G，假设存在最优解 O，若 O = G 已经得证
  第2步：若 O ≠ G，找到 O 和 G 的第一个不同决策位置 i
  第3步：构造新解 O'：按贪心规则调整 O 中第 i 个决策
  第4步：证明 O' 的目标函数值 ≤ O（或 ≥）→ O' 仍然是最优解
  第5步：重复 2-4，最终将任意最优解转变为 G → G 必是最优解
  ```
  "突击战"就是这套模板的完美演绎：假设最优解中存在 J[i] < J[i+1] 但 i 排在 i+1 前——交换两人的位置后，计算新的完成时间，证明它不差于原来的安排。不断交换所有逆序对，最终得到的就是按 J 降序排列的贪心解，从而证明贪心解必然最优。

**二分答案——判定函数设计**：
- 一句话定义：当答案具有单调性时，将"求最优解"转化为"判定某个值是否可行"。
- 最小示例：
  ```
  // 派问题：二分每个人分到的体积V
  while (L + EPS < R) {
      double mid = (L + R) / 2;
      if (canDistribute(mid, pies, F+1)) L = mid;  // 可行，尝试更大
      else R = mid;                                   // 不可行，减小
  }
  // canDistribute: 对于每个派，floor(S[i]/mid) 为能切出的块数，累加比较
  ```
- 本质理解：二分答案的关键不是二分本身，而是判定函数的正确性和效率。判定函数通常用贪心（如"每个派尽量多切"）实现，复杂度 O(n)。
- **判定函数设计五步法**：二分答案的难点不在二分而在判定。设计 `check(mid)` 时按以下流程，就不会遗漏细节：
  ```
  第1步：重新定义问题——给定阈值mid，"原问题"变成"能否在mid限制下达成？"
  第2步：选择判定策略——通常用贪心：在mid约束下，尽量"多分"或"少选"
  第3步：编写精确的计数逻辑——在mid下到底能分几份/选几个？注意整数除法的取整方向
  第4步：处理边界——整数二分时注意上下取整，浮点二分时注意精度（EPS）
  第5步：验证单调性——确认 check(mid)=true ⇒ 对所有 mid' ≤ mid 都有 check(mid')=true
  ```
  以"派"问题为例：第1步→给定体积mid，问题变成"能切出F+1块吗"；第3步→floor(S[i]/mid)精确计数；第5步→体积越小越容易满足，单调性天然成立。以"组装电脑"为例：第2步→每种类型选满足品质要求的最便宜组件就是最优策略。五步走完，check函数就不会出错。

**搜索法——剪枝框架**：
- 一句话定义：在状态空间中系统地枚举所有可能解，并通过剪枝函数提前排除不可能的分支。
- 最小示例：
  ```
  // 通用DFS剪枝框架
  void dfs(int depth) {
      if (找到解) { 记录答案; return; }
      if (不满足约束) return;           // 可行性剪枝
      if (当前代价 + 最优下界 >= ans) return;  // 最优性剪枝
      if (剩余元素不够) return;          // 数量剪枝
      // 枚举分枝，优先选择最有望找到解的分枝（顺序剪枝）
      for each branch:
          dfs(depth + 1);
  }
  ```
- 本质理解：剪枝的本质是"利用已知信息排除不可能的方向"。好的剪枝顺序（先从最有希望的分枝搜索）能大幅加快找到第一个解的速度。
- **剪枝的三大分类**：竞赛中常见的剪枝按优先级分三类：
  ```
  1. 可行性剪枝：当前状态已经不可能达成目标
     例：剩余可选数不够填满 → 直接return；剩余容量为负 → return
     这是最基础也是必须做的，不写可行性剪枝就是在浪费时间
  
  2. 最优性剪枝：当前状态即使"最好情况"也优于当前最优解
     例：cost + 乐观估计下界 ≥ best → return
     这是加速效果最显著的一类。关键在于设计一个"既快又准"的下界估计函数
     ——下界过松则剪不掉，下界计算过重则得不偿失
  
  3. 对称性剪枝：利用问题的对称结构排除等价分枝
     例：排列中固定第一个元素、组合中要求严格递增、立方体中固定第一个立方体
     这类剪枝对枚举问题特别有效，能从根本上减少搜索空间
  ```
  另外，**搜索顺序优化**也不可忽视：优先探索"最有希望"的分枝。把价值高的物品放在循环的前面，能更快找到第一个可行解，从而使最优性剪枝尽早生效——这就是"剪枝之间的协同效应"。

### 快速上手模板

```cpp
// ============ 模板1：二分答案（整数值） ============
int L = 最小值, R = 最大值;
while (L < R) {
    int mid = L + (R - L + 1) / 2;  // 注意：取上中位数，防止死循环
    if (check(mid)) L = mid;        // mid可行 → 尝试更大
    else R = mid - 1;               // mid不可行 → 必须减小
}
return L;  // 最终答案

// check(mid) 判定函数框架：
bool check(int x) {
    int cnt = 0;
    for (int i = 0; i < n; i++) {
        cnt += floor(data[i] / x);  // 贪心计算能分成多少份
        if (cnt >= target) return true;
    }
    return cnt >= target;
}

// ============ 模板2：DFS剪枝搜索 ============
int best = INF;
void dfs(int depth, int cost) {
    if (depth == n) { best = min(best, cost); return; }
    // 最优性剪枝
    if (cost + lower_bound(depth) >= best) return;
    // 可行性剪枝
    if (!feasible(depth)) return;
    // 顺序剪枝：最优分枝优先
    for (auto& branch : sorted_branches(depth)) {
        做出选择;
        dfs(depth + 1, cost + branch.cost);
        回溯;
    }
}

// ============ 模板3：自顶向下分治 ============
int solve(int l, int r) {
    if (l == r) return base_case(l);
    int mid = (l + r) / 2;
    int left = solve(l, mid);
    int right = solve(mid + 1, r);
    return merge(left, right);  // 合并子问题结果
}
```

## 例题13  派（Pie, NWERC 2006, Codeforces Gym100722C）

### 题目描述

我有 N 个派（N ≤ 10000），每个派是一个高度为 1 的圆柱体，底面半径为 r[i]。现在有 F 个朋友（F ≤ 10000），加上我自己共 F+1 个人。每个人需要分到一块体积相同的派（形状可以是任意扇形的圆柱体，但只能来自同一个派）。求每个人能分到的**最大派体积**，保留 4 位小数。

**输入格式**：第一行一个整数 T，表示测试用例数。每个用例第一行两个整数 N 和 F。接下来 N 行，每行一个整数 r[i]（1 ≤ r[i] ≤ 10000），表示第 i 个派的半径。

**输出格式**：每组数据输出一行，一个保留 4 位小数的实数，表示每个人能分到的最大体积。

### 解题思路

**1. 二分答案的适用条件**

问题要求最大化"每个人分到的体积"，具有单调性：如果每人能分到体积 V，那么对于任意 V' < V，每人也能分到 V'。因此可以对答案进行二分。

**2. 判定函数的实现**

对于给定的体积 x，检验是否能分给 F+1 个人：
- 对于每个派，能切出 `floor(S[i] / x)` 块
- 累加所有派能切出的块数，如果 ≥ F+1 则可行

**3. 二分的边界和精度**

- 下界 L = 0（体积为 0 一定可行）
- 上界 R = max(S[i]) = max(r[i]² × π)（所有派中最大的那个）
- 精度要求 4 位小数，使用 EPS = 10⁻⁶
- 使用浮点二分：while (L + EPS < R)，如果判定可行则 L = M，否则 R = M

### 算法方法

**1. 二分答案（Binary Search on Answer）**
- 在实数域上二分查找最大可行体积
- 判定函数具有单调性，保证了二分的正确性

**2. 贪心计数**
- 对于每个派，计算它能贡献的最大块数（向下取整）
- 贪心切割：每个派尽可能多地切出大小相等的块

### 复杂度分析

- **时间复杂度**：O(N × log(R/EPS))。每次判定需要 O(N) 遍历所有派，二分迭代次数约为 log₂(R/EPS) ≈ log₂(10⁹) ≈ 30 次。
- **空间复杂度**：O(N)。需要存储 N 个派的面积。

```cpp
// 例题13  派（Pie, NWERC 2006, Codeforces Gym100722C）
// 陈锋
#include <cmath>
#include <cstdio>
#include <algorithm>

using namespace std;
#define _for(i, a, b) for (int i = (a); i < (b); ++i)
typedef long long LL;
const int NN = 10000 + 4;
double S[NN];  // S[i]：第i个派的面积（π × r²）
const double PI = acos(-1), EPS = 1e-6;  // π值和二分精度

int main() {
  int T; scanf("%d", &T);
  for (int t = 0, N, F; t < T; t++) {
    scanf("%d%d", &N, &F), ++F;  // F++：加上自己，共F+1个人
    double L = 0, R = 0, r;
    // ---- 读取派半径，计算面积并确定二分上限 ----
    _for(i, 0, N) {
      scanf("%lf", &r);
      R = max(R, S[i] = r * r * PI);  // 上限为最大派的面积
    }

    // ---- 二分答案阶段 ----
    while (L + EPS < R) {  // 精度到EPS即可，L始终保持可行解
      double M = (L + R) / 2;  // 二分中点：尝试每人分M体积
      int f = 0;  // 能切出的总块数
      _for(i, 0, N) {
        f += (int)floor(S[i] / M);  // 第i个派能切出多少块M体积的块
        if (f >= F) break;  // 已经够了，提前退出
      }
      // 如果够分，说明M可行→增大下界；否则减小上界
      if (f >= F) L = M; else R = M;
    }
    printf("%.4lf\n", L);  // 输出下界（保证是可行解）
  }
}
```

## 例题12  组装电脑（Assemble, NWERC 2007,  LA3971/POJ3497）

### 题目描述

要组装一台电脑，共有 n 个可选组件（n ≤ 1000），分为多个类型（如 CPU、内存、硬盘等）。每个组件有一个价格 price 和品质因子 quality。对于每种类型，必须恰好选择一个组件。总预算为 b。要求在所有不超过预算的方案中，找到使品质最低组件品质最高的方案，输出该最低品质的最大值。

**输入格式**：第一行 T，测试用例数。每组第一行两个整数 n 和 b（1 ≤ b ≤ 10⁹）。接下来 n 行，每行：类型名（字符串）、组件名（字符串）、价格 p（1 ≤ p ≤ 10⁶）、品质 q（1 ≤ q ≤ 10⁹）。

**输出格式**：每组输出一行，一个整数，表示所选组件中最低品质的最大可能值。

### 解题思路

**1. 二分答案的单调性**

问题要求最大化"最低品质"，具有单调性：如果品质 q 可达（存在一种选择方案使得所有组件品质 ≥ q 且总价 ≤ b），那么任意 q' < q 也可达。因为放宽品质要求不会使问题更难。

**2. 判定函数的实现**

对于给定的品质阈值 q，检验能否组装出总价 ≤ b 的电脑：
- 对于每种组件类型，筛选出品质 ≥ q 的所有组件
- 从每种类型中选一个最便宜的组件（因为品质满足要求即可，选更贵的没有好处）
- 如果某种类型没有品质 ≥ q 的组件，则不可能
- 累加各类型最低价，检查是否 ≤ b

**3. 二分搜索技巧（整数二分的重要模式）**

由于答案 q 是整数，使用整数二分。初学者容易犯的错误是：中点向下取整导致死循环。

**问题场景**：当 L=3, R=4 时，如果 M = (L+R)/2 = 3（向下取整），且 ok(3)=true，则 L=M=3 —— 范围卡在 [3,4] 不动了！

**解决方案**：使用**上取整**的中点 `M = L + (R-L+1)/2`（等价于 ceil((L+R)/2)）。

配合模式：如果 M 可行，`L = M`（尝试更大）；如果不可行，`R = M-1`。

这个模式保证每次迭代范围严格缩小。具体推导请参考本章后续的二分专题。

**给初学者的记忆口诀**：
- 当 `L = M`（答案可能在 M 或更大）时 → M 用上取整
- 当 `R = M`（答案可能在 M 或更小）时 → M 用下取整

### 算法方法

**1. 二分答案（Binary Search on Answer）**
- 在品质范围 [0, maxq] 上二分查找最大值
- 答案空间离散（整数），处理方式与实数二分略有不同

**2. 贪心选择（判定函数内）**
- 每种类型选满足品质要求的最低价组件
- 这是最优策略：在品质约束下，价格应尽量低

**3. 字符串哈希/映射**
- 使用 map 将类型名映射到整数 ID

### 复杂度分析

- **时间复杂度**：O(n × log(maxq))。每次判定 O(n) 遍历所有组件，二分迭代 log₂(maxq) ≈ 30 次。
- **空间复杂度**：O(n)。需要按类型存储所有组件信息。

```cpp
// 例题12  组装电脑（Assemble, NWERC 2007,  LA3971/POJ3497）
// Rujia Liu
#include<cstdio>
#include<string>
#include<vector>
#include<map>
using namespace std;

int cnt; // 组件的类型数（不同类型的数量）
map<string,int> id;
// 字符串哈希：将类型名映射为整数ID
int ID(string s) {
  if(!id.count(s)) id[s] = cnt++;  // 新类型，分配新ID
  return id[s];
}

const int maxn = 1000 + 5;

struct Component {
  int price;   // 价格
  int quality; // 品质因子
};
int n, b; // n: 组件的数目, b: 总预算
vector<Component> comp[maxn];  // comp[i]: 第i种类型的所有可选组件

// 判定函数：品质因子不低于q的组件能否组装成一个不超过b元的电脑
bool ok(int q) {
  int sum = 0;  // 累计总价格
  for(int i = 0; i < cnt; i++) {  // 遍历每种组件类型
    int cheapest = b+1, m = comp[i].size();  // cheapest初始化为超过预算的值
    for(int j = 0; j < m; j++)  // 在该类型中遍历所有组件
      // 贪心：在品质≥q的组件中选最便宜的
      if(comp[i][j].quality >= q) cheapest = min(cheapest, comp[i][j].price);
    if(cheapest == b+1) return false;  // 无满足品质要求的组件，不可行
    sum += cheapest;
    if(sum > b) return false;  // 预算溢出，不可行
  }
  return true;  // 所有类型都满足，总价在预算内
}

int main() {
  int T;
  scanf("%d", &T);
  while(T--) {
    scanf("%d%d", &n, &b);

    // ---- 初始化：清空之前的数据 ----
    cnt = 0;
    for(int i = 0; i < n; i++) comp[i].clear();
    id.clear();

    // ---- 输入阶段 ----
    int maxq = 0;  // 记录最大品质，作为二分上限
    for(int i = 0; i < n; i++) {
      char type[30], name[30];
      int p, q;
      scanf("%s%s%d%d", type, name, &p, &q);
      maxq = max(maxq, q);  // 更新最大品质
      comp[ID(type)].push_back((Component){p, q});  // 按类型分组存储
    }

    // ---- 整数二分答案阶段（查找最大可达品质） ----
    int L = 0, R = maxq;
    while(L < R) {
      int M = L + (R-L+1)/2;  // 上取整：避免L=M时死循环
      if(ok(M)) L = M;        // M可行→品质可以更高，升下界
      else R = M-1;           // M不可行→降低上界
    }
    printf("%d\n", L);  // L即为最大可达的最低品质
  }
  return 0;
}
```

## 例题11  新汉诺塔问题（A Different Task, UVa 10795）

### 题目描述

经典的汉诺塔问题有 3 根柱子（编号 1, 2, 3）和 n 个大小不同的盘子（n ≤ 60），初始时所有盘子从小到大堆叠在某些柱子上。给定一个目标配置（每个盘子在哪个柱子上），每次只能移动一个盘子，且大盘不能放在小盘上面。求从初始配置到目标配置所需的**最少步数**。

**输入格式**：多组数据。每组第一行 n，n=0 结束。第二行 n 个整数（1-3），表示初始每个盘子的柱子（按从小到大顺序）。第三行 n 个整数，表示目标每个盘子的柱子。

**输出格式**：每组输出 `Case X: Y`，X 为编号，Y 为最少步数。

### 解题思路

**回顾经典汉诺塔**

经典问题：把 n 个盘子从柱子 A 移到柱子 C，借助 B。最少步数 = 2^n - 1。
关键性质：移动 n 个盘子到目标柱 = 移动 n-1 个到辅助柱 + 移动第 n 个 + 移动 n-1 个到目标柱。即 **`T(n) = 2 × T(n-1) + 1 = 2^n - 1`**。

**本题不同之处**：初始和目标配置各不相同，但核心思路相同——从最大的盘子开始考虑。

**1. 子问题定义：把前 i 个盘子都移到柱子 final**

定义 `f(P, i, final)` = 把编号 1..i 的盘子（P 是当前它们的柱子分布）全部移到柱子 final 的最少步数。

递归计算：
- 如果盘子 i 已经在 final 上了 → `f(P, i-1, final)`（不需要动它）
- 如果盘子 i 不在 final → 需要先把 1..i-1 移到"辅助柱" other（留出位置），然后把 i 移到 final
  - 移走 1..i-1 到 other：**2^(i-1) 步**（经典汉诺塔性质）
  - 但此时 1..i-1 中有些可能已经在 final 上了！需要额外递归 `f(P, i-1, other)`
  - 最终 i 移动 1 步到位

**代码中的 `f(P, i, final)`**：

`if P[i] == final` → 已在目标柱，递归处理 i-1
`else` → 需要 2^(i-1) 步 + 递归把 i-1 个盘子移到辅助柱

**2. 从大到小处理：找到第一个位置不同的盘子**

从最大盘子 k=n 开始往下找，找到 `start[k] != finish[k]` 的最大的 k。
如果 k=0，说明初始就是目标，ans=0。

**3. 分三步完成转换**

设 other = 6 - start[k] - finish[k]（三根柱子编号 1,2,3，和为 6）

```
步骤 1: 把前 k-1 个盘子移到 other → f(start, k-1, other) 步
步骤 2: 把盘子 k 移到目标柱 → 1 步
步骤 3: 把 other 上的 k-1 个盘子移到目标 → f(finish, k-1, other) 步
```

**关键直觉**：为什么是 `f(start, k-1, other) + 1 + f(finish, k-1, other)`？

想想我们只有一个大箱子（盘子 k）和一堆小箱子（盘子 1..k-1）。要把大箱子从房间 A 搬到房间 B：
1. 先把地上的小箱子搬到走廊 (other) —— `f(start, k-1, other)`
2. 搬走大箱子 (1 步)
3. 把小箱子从走廊搬回 B —— `f(finish, k-1, other)`

这就是分治思想：**最大盘子必须最先确认位置，它定了之后，所有小盘子只需要考虑"移到哪里为它让路"。**

### 算法方法

**1. 递归/分治**
- 利用经典汉诺塔的递推性质，自顶向下分解问题
- 从最大的盘子开始考虑，逐步缩小问题规模

**2. 状态转换与等价**
- 将"任意初始→任意目标"转化为"初始→统一柱子"和"统一柱子→目标"两个子问题
- 利用汉诺塔问题的对称性

**3. 位运算快速计算 2^n**
- `1LL << (i-1)` 直接计算 2^(i-1)，O(1)

### 复杂度分析

- **时间复杂度**：O(n)。递归深度为 n（n ≤ 60），每次递归 O(1)。
- **空间复杂度**：O(n)。需要存储 start 和 finish 数组。

```cpp
// 例题11  新汉诺塔问题（A Different Task, UVa 10795）
// Rujia Liu
#include<cstdio>

// 递归函数：将编号1~i的盘子全部移到柱子final所需的最少步数
// P[]: 当前每个盘子的位置
// i: 考虑的最大盘子编号（1~i）
// final: 目标柱子编号(1/2/3)
long long f(int* P, int i, int final) {
  if(i == 0) return 0;  // 没有盘子需要移动
  if(P[i] == final) return f(P, i-1, final);  // 盘子i已在目标位置，处理更小的盘子
  // 盘子i不在目标位置：需要先将1~i-1移到辅助柱(other)，再将i移到final
  // 3根柱子编号1+2+3=6，other = 6 - P[i] - final
  // 将i-1个盘子移到other需要(1LL << (i-1))步，加上移动i的1步
  return f(P, i-1, 6-P[i]-final) + (1LL << (i-1));
}

const int maxn = 60 + 10;
int n, start[maxn], finish[maxn];  // start[i]:第i个盘子的初始柱; finish[i]:目标柱

int main() {
  int kase = 0;
  while(scanf("%d", &n) == 1 && n) {
    for(int i = 1; i <= n; i++) scanf("%d", &start[i]);   // 读入初始配置
    for(int i = 1; i <= n; i++) scanf("%d", &finish[i]);  // 读入目标配置

    // ---- 第一阶段：找到需要移动的最大盘子 ----
    int k = n;
    while(k >= 1 && start[k] == finish[k]) k--;  // 从大到小，找到第一个位置不同的盘子
    // 如果k==0，说明初始和目标相同，不需要移动

    // ---- 第二阶段：计算答案 ----
    long long ans = 0;
    if(k >= 1) {
      int other = 6-start[k]-finish[k];  // 辅助柱：既不是k的初始柱也不是目标柱
      // 分三步：1)前k-1个盘子移到other → f(start,k-1,other)
      //         2)盘子k移到目标 → 1步
      //         3)前k-1个盘子从other移到目标 → f(finish,k-1,other)
      ans = f(start, k-1, other) + f(finish, k-1, other) + 1;
    }
    printf("Case %d: %lld\n", ++kase, ans);
  }
  return 0;
}
```

## 例题9  中国麻将（Chinese Mahjong, UVa 11210）

### 题目描述

麻将共有 34 种牌，包括：万子（1W-9W）、筒子（1T-9T）、条子（1S-9S）、风牌（东DONG、南NAN、西XI、北BEI）、箭牌（中ZHONG、发FA、白BAI）。每种牌最多 4 张。

给定一个 13 张牌的手牌，问**听**哪些牌（即再加一张什么牌后可以和牌）。和牌需要满足：14 张牌由一个**将牌**（2张相同的牌）和 4 组**面子**（每组是3张相同的刻子 或 3张同花色连续的顺子）组成。

**输入格式**：多组数据。每组一行 13 个字符串表示手牌。以单独一行 "0" 结束。

**输出格式**：每组输出 `Case X:`，空格后按顺序列出能听的牌。如果无法听任何牌，输出 `Not ready`。

### 解题思路

**1. 暴力枚举能听的牌**

枚举 34 种牌，假设加上这张牌后手牌为 14 张。但每种牌最多 4 张，加之前需要检查是否超过上限。

**2. 回溯搜索和牌**

对于每种可能的听牌，检查加了之后能否和牌：
- 枚举将牌（遍历 34 种牌，如果某种牌数量 ≥ 2，减 2 后尝试）
- 剩余 12 张牌，需要组成 4 组面子，递归搜索

**3. 面子搜索策略**

递归函数 `search(dep)` 表示还需要组 dep 个面子：
- 先尝试**刻子**：遍历所有牌，如果有 3 张相同的就尝试减去
- 再尝试**顺子**：遍历所有牌，如果有连续三张就尝试减去（顺子只在万、筒、条中寻找，即 i % 9 ≤ 6，且 i ≤ 24）
- 注意回溯：尝试后要恢复牌的数量

**4. 递归深度**

search(0) → search(1) → search(2) → search(3)，深度固定为 4，所以不会超时。

### 算法方法

**1. 枚举 + 回溯搜索（DFS）**
- 枚举候选听牌（34 种）
- 对每种候选，回溯验证是否能和牌

**2. 组合搜索与剪枝**
- 枚举将牌 + 递归组面子
- 搜索空间小（深度 4），无需复杂剪枝

### 复杂度分析

- **时间复杂度**：O(34 × 34 × 4³⁴)，实际上由于每种牌最多 4 张、实际搜索状态远小于理论上限，且 n 固定为 13，完全可以接受。
- **空间复杂度**：O(1)。只需 34 个计数器的数组。

```cpp
#include<string.h>

// 0-8: 万子(1T-9T); 9-17: 条子(1S-9S); 18-26: 筒子(1W-9W)

    if(strcmp(mahjong[i], s) == 0) return i;

// dep: 已经组好的面子数量，目标为4（因为14-2=12，需要4组面子）
bool search(int dep){
  int i;
  // ---- 尝试刻子（3张相同） ----
  for(i = 0; i < 34; i++) if (c[i] >= 3){ // 该牌≥3张，可组成刻子
    if(dep == 3) return true;  // 已组成3组，加上本组共4组，成功！
    c[i] -= 3;                 // 选择：拿走3张组成刻子
    if(search(dep+1)) return true;  // 递归搜索剩余
    c[i] += 3;                 // 回溯：恢复3张
  }
  // ---- 尝试顺子（3张连续同花色） ----
  // 顺子只在0-26范围内（万、条、筒），且花色内不能跨边界（i%9≤6保证起始牌后还有2张）

    if(dep == 3) return true;
    c[i]--; c[i+1]--; c[i+2]--;  // 拿走3张顺子
    if(search(dep+1)) return true;
    c[i]++; c[i+1]++; c[i+2]++;  // 回溯
  }
  return false;  // 无法组成面子
}

// 策略：枚举将牌（2张相同的牌），然后尝试组成4组面子
bool check(){
  int i;
  for(i = 0; i < 34; i++)
    if(c[i] >= 2){  // 该牌≥2张，可作为将牌候选
      c[i] -= 2;    // 选择：拿走2张作为将牌
      if(search(0)) return true;  // 从0组面子开始搜索，全部可组成则成功
      c[i] += 2;    // 回溯：恢复将牌
  }
  return false;  // 无将牌可达，或所有将牌尝试均失败
}

    if(s[0] == '0') break;  // 输入结束标志
    printf("Case %d:", ++caseno);
    mj[0] = convert(s);  // 第一张牌
    for(i = 1; i < 13; i++){  // 读入剩余12张
      scanf("%s", &s);
      mj[i] = convert(s);
    }
    ok = false;
    // ---- 枚举所有可能的听牌（34种） ----
    for(i = 0; i < 34; i++){
      memset(c, 0, sizeof(c));  // 重置计数
      for(j = 0; j < 13; j++) c[mj[j]]++;  // 统计手牌
      if(c[i] >= 4) continue;  // 每种牌最多4张，如果已有4张则不能"听"此牌
      c[i]++;  // 假设拥有这张牌（加入后共14张）
      if(check()){  // 检验14张能否和牌
        ok = true;
        printf(" %s", mahjong[i]);  // 输出听的牌名
      }
      c[i]--;  // 恢复
    }
    if(!ok) printf(" Not ready");
    printf("\n");
  }
  return 0;
}
```

## 例题10  正整数序列（Help is needed for Dexter, UVa 11384）

### 题目描述

给定正整数 n（1 ≤ n ≤ 10⁹），要求通过若干次操作，将序列 1, 2, 3, ..., n 全部变为 0。每次操作可以**选择任意一个子序列**，将子序列中的每个数都减去同一个正整数（每个数减去相同的值，但不同操作可以减不同的值）。求最少操作次数。

**输入格式**：多组数据，每行一个整数 n。文件读到 EOF 结束。

**输出格式**：每组输出一行，一个整数，表示最少操作次数。

### 解题思路

**1. 问题的分治视角**

每次操作可以减去序列中某些数的某个值。目标是让所有数都变成 0。考虑如何用最少的操作。

**2. 最优策略：每次让半数变为 0**

考虑一次操作的最优效果：如果我们减去某个值 x，所有 ≥ x 的数中部分减去 x 后可能变成 0，其他数减小。最优策略是：
- 每次操作让恰好 **一半** 的数变为 0（选取序列的后半部分，减去 n/2）
- 剩余的数形成新的序列 1, 2, ..., n/2（或 floor(n/2)）

**3. 递推公式**

设 f(n) 为将 1~n 变为 0 的最少次数：
- n=1：一次操作减 1，f(1)=1
- n≥2：减去 n/2（或 floor(n/2)+1），问题规模变为 n/2
- f(n) = f(n/2) + 1

这等价于求 n 的二进制长度（即 ceil(log₂(n))），或者说求 n 不断整除 2 直到为 1 的次数。

**4. 证明最优性**

每次操作最多能将序列中一半的数变为 0（因为如果减去的值使得超过一半的数变为 0，那至少有两个数差值和变为0矛盾）。因此至少需要 ⌈log₂ n⌉ 次操作。

### 算法方法

**1. 递归 / 分治**
- 将规模为 n 的问题分解为规模 n/2 的子问题 + 1 次操作
- 体现了"折半"思想

**2. 贪心构造**
- 每次操作选择序列的一半（后半部分），减去 appropriate 的值使其变为 0
- 局部最优导致全局最优

### 复杂度分析

- **时间复杂度**：O(log n)。递归深度为 log₂ n，每次 O(1)。
- **空间复杂度**：O(log n)（递归栈空间），或 O(1)（迭代实现）。

```cpp
// 例题10  正整数序列（Help is needed for Dexter, UVa 11384）
// Rujia Liu
#include<cstdio>
// 递归函数：将序列1~n变为0的最少操作次数
// 递推关系：f(n) = f(n/2) + 1（每次操作将后半部分变为0，问题规模缩小一半）
int f(int n) {
  return n == 1 ? 1 : f(n/2) + 1;  // 递归边界 + 折半分治
  // 例如：n=8时，操作1：减去4→(1,2,3,4,0,0,0,0)，剩余1,2,3,4→等价于n=4
}

int main() {
  int n;
  while(scanf("%d", &n) == 1)
    printf("%d\n", f(n));  // 最少操作次数 = floor(log2(n)) + 1
  return 0;
}
```

## 例题7  偶数矩阵（Even Parity, UVa 11464）

### 题目描述

给定一个 n×n 的 01 矩阵（n ≤ 15），其中 0 和 1 已经部分确定。你可以将一些 0 变为 1（但不能将 1 变为 0）。要求变换后矩阵满足：每个元素的上下左右四个邻居之和为偶数。求最少需要将多少个 0 变为 1。如果不可能，输出 -1。

**输入格式**：第一行 T，测试用例数。每组第一行 n，接下来 n 行每行 n 个整数（0 或 1）。

**输出格式**：每组输出 `Case X: Y`，X 编号，Y 为最少变换次数或 -1。

### 解题思路

**核心发现：已知第一行，整个矩阵就确定了！**

考虑一个 n×n 的 01 矩阵 B，要求每个位置的上下左右邻居之和为偶数（模 2 意义下）。

**1. 递推关系的推导**

以 3×3 为例，先看位置 (0,0)（第一行第一列）：

```
B[0][0] 的邻居 = 上(无) + 下(B[1][0]) + 左(无) + 右(B[0][1])
= B[1][0] + B[0][1] ≡ 0 (mod 2)
```

因此 **`B[1][0] = B[0][1]`**（模 2 下，x + y = 0 等价于 x = y）

再看位置 (0,1)（第一行中间）：
```
B[0][1] 的邻居 = B[1][1] + B[0][0] + B[0][2] ≡ 0 (mod 2)
```
因此 **`B[1][1] = B[0][0] + B[0][2]`**（模 2，即 B[0][0] XOR B[0][2]）

**规律**：已知 B[r-1] 和 B[r-2] 行后，B[r][c] 必须满足 B[r-1][c] 的邻居和为偶数。而已知 B[r-1][c] 的上下左右，其中"上"是 B[r-2][c]，"左"是 B[r-1][c-1]，"右"是 B[r-1][c+1]，三者已知后，"下"（即 B[r][c]）就被唯一确定了：

**`B[r][c] = (B[r-2][c] + B[r-1][c-1] + B[r-1][c+1]) % 2`**

这是一个**递推公式**：如果你知道了前两行（第 r-2 行和第 r-1 行），第 r 行就被唯一确定。

**因此**：只需知道第 0 行（第一行），第 1 行就被确定（用 B[-1][c]=0 处理），然后第 2 行被确定，以此类推，整个矩阵就确定了。

**2. 枚举第一行**

n ≤ 15，第一行有 2^n ≤ 32768 种可能。用二进制数 s（0 到 2^n-1）表示第一行：s 的第 c 位是 1 表示 B[0][c]=1。

对每种第一行：
1. 根据递推公式生成整个矩阵 B
2. 检查 B 是否与输入 A 兼容（A 中的 1 在 B 中也必须是 1，不能把 1 变成 0）
3. 如果不兼容，跳过（标记为不可行 INF）
4. 如果兼容，统计有多少个位置需要 0→1（A=0 但 B=1 的位置）
5. 在所有可行的第一行中取最小值

**3. 为什么这是正确的？**

递推公式保证：只要第一行的值符合"偶数约束"的前提（没有冲突），递推出来的每一行也都自动满足偶数约束。输入固定的 1 在递推结果中必须存在，否则说明这种第一行选择是不可行的。

### 算法方法

**1. 状态压缩枚举**
- 用二进制位掩码 s（0 到 2^n-1）枚举第一行的所有可能
- s 的第 c 位表示 B[0][c] 的值

**2. 递推 / 动态规划**
- 根据第一行递推出整个矩阵
- 每行由前两行唯一确定

**3. 可行性检查**
- 检查 B 是否与 A 兼容（A[i][j]=1 时 B[i][j] 必须为 1）
- A[i][j]=0 时 B[i][j] 可以是 0 或 1（变换就是 0→1）

### 复杂度分析

- **时间复杂度**：O(2^n × n²)。枚举 2^n 种第一行，每种 O(n²) 递推并统计。n ≤ 15，2^15 = 32768，n² = 225，总计约 7.4×10⁶，可接受。
- **空间复杂度**：O(n²)。需要两个 n×n 的矩阵。

```cpp
// 例题7  偶数矩阵（Even Parity, UVa 11464）
// Rujia Liu
#include<cstdio>
#include<cstring>
#include<algorithm>
using namespace std;

const int maxn = 20;
const int INF = 1000000000;  // 表示不可行的无穷大值
int n, A[maxn][maxn], B[maxn][maxn];  // A: 输入矩阵; B: 构造的矩阵

// 校验函数：给定第一行状态s（二进制位掩码），计算最少变换次数
// 返回值：INF表示不可行，否则为需要将0变为1的次数
int check(int s) {
  memset(B, 0, sizeof(B));
  // ---- 初始化第一行 ----
  for(int c = 0; c < n; c++) {
    if(s & (1<<c)) B[0][c] = 1;  // s的第c位为1→B[0][c]=1
    else if(A[0][c] == 1) return INF; // A要求1但B要设为0→不可行（1不能变成0）
  }

  // ---- 递推阶段：从第1行开始逐行确定 ----
  for(int r = 1; r < n; r++)
    for(int c = 0; c < n; c++) {
      // 计算 B[r-1][c] 的邻居（上、左、右）之和
      // B[r][c] 需要满足：B[r-1][c]的上下左右和为偶数
      int sum = 0; // B[r-1][c]的上(B[r-2][c])、左(B[r-1][c-1])、右(B[r-1][c+1])之和
      if(r > 1) sum += B[r-2][c];      // 上方邻居
      if(c > 0) sum += B[r-1][c-1];    // 左邻居
      if(c < n-1) sum += B[r-1][c+1];  // 右邻居
      // 要满足偶数性：sum + B[r][c] ≡ 0 (mod 2) → B[r][c] = sum % 2
      B[r][c] = sum % 2;
      // 兼容性检查：A[r][c]=1时B[r][c]必须为1
      if(A[r][c] == 1 && B[r][c] == 0) return INF; // 冲突，不可行
    }

  // ---- 统计阶段：计算需要将多少0变为1 ----
  int cnt = 0;
  for(int r = 0; r < n; r++)
    for(int c = 0; c < n; c++)
      if(A[r][c] != B[r][c]) cnt++;  // B与A不同→必定是A=0,B=1→0变1
  return cnt;
}

int main() {
  int T;
  scanf("%d", &T);
  for(int kase = 1; kase <= T; kase++) {
    scanf("%d", &n);
    for(int r = 0; r < n; r++)
      for(int c = 0; c < n; c++) scanf("%d", &A[r][c]);

    // ---- 枚举第一行的所有可能 ----
    int ans = INF;
    for(int s = 0; s < (1<<n); s++)  // s从0到2^n-1，共2^n种第一行
      ans = min(ans, check(s));      // 取最小变换次数

    if(ans == INF) ans = -1;  // 所有可能都不可行
    printf("Case %d: %d\n", kase, ans);
  }
  return 0;
}

```

## 例题14  填充正方形（Fill the Square, UVa11520）

### 题目描述

给定一个 n×n（n ≤ 10）的网格，部分格子已填有大写字母，其余格子为 `.`（待填充）。需要将所有 `.` 填上大写字母（A-Z），使得每个格子的上下左右相邻格子（如果存在）的字母都与自己不同，且整个网格的字典序最小（按行优先比较）。

**输入格式**：第一行 T，测试用例数。每组第一行 n，接下来 n 行每行 n 个字符。

**输出格式**：每组输出 `Case X:`，然后 n 行填充后的网格。

### 解题思路

**1. 字典序最小的含义**

要求字典序最小，意味着在处理每个格子时，应尽量填入尽可能小的字母。由于是按行优先比较，从左到右、从上到下依次贪心填充即可。

**2. 贪心策略：逐格处理**

对于每个格子 (i, j)：
- 如果已是字母，跳过（不能修改已有内容）
- 如果是 `.`，从 `'A'` 到 `'Z'` 依次尝试
- 检查填入的字母是否与上下左右的邻居冲突
- 选择第一个不冲突的字母填入

**3. 为什么贪心是正确的？**

由于填充一个格子只影响其四个邻居，而且处理顺序是按行优先递增的，后面处理的格子不会影响前面已处理格子的合法性。因此，在每个位置选取最小可行字母可以保证全局字典序最小。

**4. 边界处理**

注意检查邻居时不要越界（检查 i>0, i<n-1, j>0, j<n-1）。

### 算法方法

**1. 贪心策略**
- 每个位置选最小可行字母
- 按行优先顺序保证全局字典序最小

**2. 邻域约束检查**
- 每次填入字母时检查上下左右四方向
- 约束简单：不与邻居相同即可

### 复杂度分析

- **时间复杂度**：O(n²)。每个格子最多尝试 26 次（字母 A-Z），每次检查 4 个方向。n ≤ 10，总量极小。
- **空间复杂度**：O(n²)。需要存储网格。

```cpp
// 例题14  填充正方形（Fill the Square, UVa11520）
// Rujia Liu
#include<cstdio>
#include<cstring>
const int maxn = 10 + 5;
char grid[maxn][maxn];  // 网格
int n;
int main() {
  int T;
  scanf("%d", &T);
  for(int kase = 1; kase <= T; kase++) {
    scanf("%d", &n);
    for(int i = 0; i < n; i++) scanf("%s", grid[i]);  // 读入行

    // ---- 贪心填充阶段：按行优先顺序处理每个格子 ----
    for(int i = 0; i < n; i++)
      for(int j = 0; j < n; j++) if(grid[i][j] == '.') {  // 只处理待填充的格子
        for(char ch = 'A'; ch <= 'Z'; ch++) {  // 从A到Z依次尝试（保证字典序最小）
          bool ok = true;  // 标记ch是否可行
          // 检查四个方向的邻居（注意边界判断）：
          if(i>0 && grid[i-1][j] == ch) ok = false;      // 上方邻居
          if(i<n-1 && grid[i+1][j] == ch) ok = false;    // 下方邻居
          if(j>0 && grid[i][j-1] == ch) ok = false;      // 左方邻居
          if(j<n-1 && grid[i][j+1] == ch) ok = false;    // 右方邻居
          if(ok) { grid[i][j] = ch; break; }  // 找到第一个不冲突的字母，填入并停止尝试
          // 由于字母表有26个而邻居最多4个，一定能找到合适的字母
        }
      }

    // ---- 输出阶段 ----
    printf("Case %d:\n", kase);
    for(int i = 0; i < n; i++) printf("%s\n", grid[i]);
  }
  return 0;
}
```

## 例题15  网络（Network, Seoul 2007,UVa1267）

### 题目描述

有一棵树表示网络拓扑，共 N 个节点（N ≤ 1000）。节点 S 是服务器（初始已有一个服务器）。现在需要放置尽可能少的额外服务器，使得每个叶子节点到最近的服务器距离不超过 K（K ≤ 1000）。求最少需要放置几个额外服务器。

**输入格式**：第一行 T，测试用例数。每组第一行三个整数 N, S, K。接下来 N-1 行每行两个整数 u, v，表示一条边。

**输出格式**：每组输出一行，一个整数表示最少额外服务器数量。

### 解题思路

**1. 问题的直观理解**

把树想象成公司的组织架构：S 是总部（已有服务器），叶子节点是偏远办公室（需要网络覆盖）。服务器可以覆盖距离不超过 K 的所有节点。目标是用最少的额外服务器覆盖所有叶子。

**2. 贪心策略：从最深的叶子开始，放在尽量高处**

为什么从最深开始？因为深层的叶子最难覆盖——如果先处理浅层的叶子，可能会忽略深层，导致需要更多服务器。

为什么放在 K 级祖先？因为这样：
- 向下能覆盖目标叶子（距离恰好 K，最大化向下覆盖范围）
- 向上还能再覆盖 K 层（帮助覆盖更深层的其他叶子）
- 放在更高处不会增大覆盖范围（只会让下面覆盖不到）

**3. 算法步骤**

以 S 为根 DFS，把树转成有根树，按深度分组叶子节点。

从最深层往浅层处理（贪心）：
- 取一个未被覆盖的叶子 u
- 找到 u 的 K 级祖先 v（向上走 K 步）
- 在 v 处放一个服务器
- 从 v 开始 DFS，标记距离 ≤ K 的所有节点为"已覆盖"

**4. 为什么 K 级祖先是最优的？**

如果在低于 K 级祖先处（比如距离叶子更近）放服务器，虽然也能覆盖这个叶子，但向上覆盖范围就减少了，可能无法覆盖其他叶子。只有在"尽量接近根"的位置放置，才能覆盖最多的节点。而 K 级祖先恰好就是能覆盖这个叶子的最高位置。

**5. 树的预处理**

DFS 建树时：
- 计算每个节点的父亲（fa[u]），方便向上找 K 级祖先
- 记录节点深度
- 把叶子按深度分组（nodes[depth]），方便从深到浅处理

### 算法方法

**1. 树转有根树（DFS）**
- 以初始服务器 S 为根建立父子关系
- 计算每个节点的深度

**2. 树的贪心覆盖**
- 按深度从深到浅处理叶子
- 在 K 级祖先处放置服务器
- DFS 标记覆盖范围

**3. 深度分组优化**
- 将叶子按深度分组，从深层往浅层处理
- 避免重复处理已覆盖的叶子

### 复杂度分析

- **时间复杂度**：O(N)。建树 O(N)，贪心覆盖过程每个节点被标记覆盖一次，总体 O(N)。
- **空间复杂度**：O(N)。需要存储邻接表、父节点数组、深度分组等。

```cpp
// 例题15  网络（Network, Seoul 2007,UVa1267）
// Rujia Liu
#include <bits/stdc++.h>
using namespace std;
const int NN = 1000 + 4;
vector<int> G[NN], nodes[NN];  // G: 邻接表; nodes[d]: 深度为d的叶子节点列表
int N, S, K, fa[NN];  // fa[u]: 节点u的父亲
bool covered[NN];     // covered[u]: 节点u是否已被服务器覆盖

// 第一次DFS：无根树转化为以S为根的有根树
// 同时按深度将叶子结点分组插入nodes表里
void dfs(int u, int f, int d) {
  fa[u] = f;  // 记录父节点
  // 叶子节点判定：度为1（只有一个邻居）且深度 > K
  // 深度≤K的叶子距离S不足K，不需要额外服务器
  if (G[u].size() == 1 && d > K)
    nodes[d].push_back(u);  // 按深度分组存储叶子
  for (int v : G[u])
    if (v != f)  // 不往回走
      dfs(v, u, d + 1);
}

// 第二次DFS：从节点u开始，覆盖距离不超过K的所有节点
void dfs2(int u, int f, int d) {
  covered[u] = true;  // 标记为已覆盖
  for (int v : G[u])
    if (v != f && d < K)  // 只覆盖距离不超过K的节点（越远越弱）
      dfs2(v, u, d + 1); 
}

int solve() {
  int ans = 0;  // 额外服务器计数
  fill_n(covered, N+1, 0);  // 初始化所有节点为未覆盖

  // ---- 贪心阶段：从深层到浅层处理叶子 ----
  for (int d = N - 1; d > K; d--)  // 从最大深度向下到K+1
    for(int u : nodes[d]){  // 处理深度为d的所有叶子
      if (covered[u]) continue;  // 已覆盖，跳过
      // 找到u的第K级祖先：从u向上走K步
      int v = u;
      for (int j = 0; j < K; j++) v = fa[v];  // v是u的K级祖先
      // 在v处放置服务器，并标记覆盖范围
      dfs2(v, -1, 0), ans++;  // 从v开始覆盖距离K内的所有节点
    }
  return ans;
}

int main() {
  ios::sync_with_stdio(false), cin.tie(0);
  int T; cin >> T;
  while (T--) {
    cin >> N >> S >> K;
    // ---- 初始化 ----
    for (int i = 1; i <= N; i++)
      G[i].clear(), nodes[i].clear();

    // ---- 读入树的边 ----
    for (int i = 0, a, b; i < N - 1; i++)
      cin>>a>>b, G[a].push_back(b), G[b].push_back(a);

    // ---- 以S为根建树并分组叶子 ----
    dfs(S, -1, 0);
    printf("%d\n", solve());
  }
  return 0;
}
// 25875799	1267	Network	Accepted	C++	0.000	2020-12-22 14:36:36
```

## 例题16  长城守卫（Beijing Guards, CERC 2004, UVa1335）

### 题目描述

n 个人（n ≤ 100000）围成一个圈，第 i 个人需要 A[i] 个礼物（1 ≤ A[i] ≤ 100000）。现在有若干种礼物（每种无限量），需要给每个人发恰好 A[i] 个礼物，要求相邻的两个人（环上的邻居，即第 n 个人和第 1 个人也是邻居）不能有相同的礼物。求最少需要多少种礼物。

**输入格式**：多组数据。每组第一行 n，n=0 结束。接下来 n 行每行一个整数 A[i]。

**输出格式**：每组输出一个整数，表示最少需要的礼物种类数。

### 解题思路

**核心困难：环形约束**

n 个人围成一个圈，相邻的人不能有相同的礼物。如果所有人都排成一条直线，那很简单——相邻两人的需求之和就是一个下界。但因为是环，第 n 个人和第 1 个人也是相邻的，这一对可能产生额外的约束。

**关键观察：n 为偶数 vs. 奇数的区别**

- **n 为偶数**：可以交替分配"左区间礼物"和"右区间礼物"，使得第 1 人和第 n 人自然不冲突。下界 L = max(A[i]+A[i+1]) 就是答案。
- **n 为奇数**：奇偶交替无法完美闭合——第 1 人是奇数位（取左边礼物），第 n 人也是奇数位（也取左边礼物），它们在环上相邻，会冲突！所以需要更多礼物种类。

**1. 二分答案框架**

答案具有单调性（礼物种类越多越容易），所以可以在 [下界, 上界] 二分。

下界 = max(A[i] + A[i+1])（考虑环上所有相邻对）
当 n 为偶数时，下界就是答案。

**2. 判定函数 `test(p)`：给定 p 种礼物，能否满足？**

**直观思路**：把 p 种礼物分成两半：
- **左边**（编号 1..A[1]）：第 1 个人全拿这部分
- **右边**（编号 A[1]+1 .. p）：剩下的部分

然后从第 2 个人开始，贪心分配：
- **奇数编号的人**（第 3, 5, 7...）：尽量拿**右边**的礼物（避免和第 1 个人冲突）
- **偶数编号的人**（第 2, 4, 6...）：尽量拿**左边**的礼物

每次分完之后，记录这个人拿了多少"左边礼物"（L[i]）和多少"右边礼物"（R[i]），它们加起来等于 A[i]。

**贪心公式**（已知前一个人的 L 和 R）：

奇数 i：`R[i] = min(rc - R[i-1], A[i])`（尽量用右边，但不能超过右边还剩的量）
偶数 i：`L[i] = min(lc - L[i-1], A[i])`（尽量用左边，但不能超过左边还剩的量）

其中 lc=A[1]（左边总量），rc=p-A[1]（右边总量）。剩下的不足部分用另一边补充。

**3. 判定条件**

第 n 个人（也是奇数，在环上与第 1 人相邻）的 **L[n] 必须为 0**——因为他和第 1 个人不能共用左边的礼物。如果 L[n] > 0，说明有冲突，需要更多礼物种类（调高 p）。

**为什么这样贪心是合理的？** 因为约束只需要检查第 n 个人与第 1 个人的冲突，而奇数尽量取右边、偶数尽量取左边，是让冲突最小化的最优策略。

### 算法方法

**1. 二分答案**
- 在 [下界, 上界] 范围内二分查找最小可行种类数
- n 为偶数时下界即为答案（不需二分）

**2. 贪心分配策略**
- 按奇偶性决定取左边还是取右边
- 算是一种"双向贪心"来避免环形冲突

**3. 环形约束处理**
- 特殊处理第 1 个人和第 n 个人的关系
- 用 L[n]==0 检验可行性

### 复杂度分析

- **时间复杂度**：O(n × log(maxA))。每次判定 O(n)，二分迭代 log(maxA) ≈ 17 次。
- **空间复杂度**：O(n)。需要存储 A, L, R 数组。

```cpp
// 例题16  长城守卫（Beijing Guards, CERC 2004, UVa1335）
// Rujia Liu
#include <bits/stdc++.h>
#define _all(i, a, b) for (int i = (a); i <= (int)(b); ++i)
using namespace std;
const int NN = 1e5 + 4;
// N: 人数; A[i]: 第i个人需要的礼物数
// L[i], R[i]: 第i个人拿到的左/右区间的礼物数
int N, A[NN], L[NN], R[NN];

// 判定函数：测试p个礼物是否足够
// 分界点：前A[1]个礼物（编号1..A[1]）分配给第1个人
// L[i]：第i个人拿了多少"左边"(≤A[1])的礼物
// R[i]：第i个人拿了多少"右边"(>A[1])的礼物
bool test(int p) {
  int lc = A[1], rc = p - A[1];  // lc: 左区间大小; rc: 右区间大小
  L[1] = lc, R[1] = 0;  // 第1个人全拿左边的礼物
  _all(i, 2, N) {
    if (i % 2)  // 奇数编号：尽量拿右边的礼物（减少与1的冲突）
      R[i] = min(rc - R[i - 1], A[i]), L[i] = A[i] - R[i];
    else        // 偶数编号：尽量拿左边的礼物（减少与1的冲突）
      L[i] = min(lc - L[i - 1], A[i]), R[i] = A[i] - L[i];
  }
  // 关键是：第N个人必须不与第1个人共享左边的礼物
  return L[N] == 0;  // N与1没有在左区间冲突
}

int solve() {
  if (N == 1) return A[1];  // 特判：只有1个人，只需A[1]种
  A[N + 1] = A[1];  // 环的闭合

  // ---- 计算二分下界和上界 ----
  int l = 0, r = 0;
  _all(i, 1, N) {
    l = max(l, A[i] + A[i + 1]);  // 下界：相邻两人需求之和（必须各不相同）
    r = max(r, A[i] * 3);         // 上界：每个人A[i]个礼物，相邻限制最多3倍
  }

  // ---- 二分答案（仅当N为奇数时需二分；N为偶数时l即为答案） ----
  while (N % 2 && l < r) {
    int m = l + (r - l) / 2;
    if (test(m)) r = m;    // m可行，尝试更小
    else l = m + 1;        // m不可行，需要更大
  }
  return l;  // l即为最少需要的礼物种类数
}

int main() {
  ios::sync_with_stdio(false), cin.tie(0);
  while (cin >> N && N) {
    _all(i, 1, N) cin >> A[i];  // 读入每个人的需求
    cout << solve() << endl;
  }
  return 0;
}

```

## 例题8  彩色立方体（Colored Cubes, Tokyo 2005, UVa1352）

### 题目描述

有 n 个立方体（n ≤ 4），每个立方体的 6 个面分别涂有某种颜色（颜色用字符串表示）。你可以任意旋转每个立方体（24 种不同的旋转姿态），然后重新给某些面涂色。目标是让所有立方体对应面（每个立方体的第 1 面、第 2 面……第 6 面）的颜色完全相同。求最少需要重新涂色的面数。

**输入格式**：多组数据。每组第一行为 n，n=0 结束。接下来 n 组，每组 6 个字符串表示该立方体 6 个面的颜色（按顺序：前后左右上下）。

**输出格式**：每组输出一行，一个整数表示最少重新涂色面数。

### 解题思路

**1. 立方体的旋转**

每个立方体有 6 个面（编号 0~5），它在空间中有 24 种旋转姿态（6 个面任选一个朝前 × 4 个旋转方向 = 24）。

预计算一个表 `dice24[24][6]`：第 r 行是第 r 种旋转方式。`dice24[r][j]` 的含义是：**原立方体的第 j 个面，旋转后到了新立方体的第几个位置**。

例如 `dice24[0] = {2,1,5,0,4,3}`：
- 原第 0 面 → 新第 2 面
- 原第 1 面 → 新第 1 面（不变）
- 原第 2 面 → 新第 5 面
- 原第 3 面 → 新第 0 面
- ...

**给初学者的简化理解**：不必深究 dice24 表的推导细节，把它当"查字典"——给定旋转编号 r 和原始面编号 j，就能查出旋转后这个面去了哪个位置。

**2. 搜索框架**

由于 n ≤ 4，可以暴力枚举：
- 固定第 1 个立方体不旋转（24 种旋转彼此等价，固定一种减少计算量）
- 枚举第 2、3、4 个立方体各 24 种旋转 → 最多 24³ = 13824 种组合

对每种旋转组合，计算最少需要重新涂色的面数。

**3. 计算重涂色数量**

对于 6 个面中的每个面（第 j 面）：
- 统计 n 个立方体旋转后，该面出现哪些颜色
- 找出现次数最多的颜色（比如红色出现了 3 次）
- 那么把其他 n-3=1 个面都涂成红色即可
- 该面的重涂代价 = n - 出现次数最多的颜色次数

6 个面的代价之和就是当前旋转组合的总代价。取所有组合的最小值。

**4. 为什么固定第一个立方体？**

24 种旋转只是改变了立方体的朝向。如果第 1 个立方体也参与枚举，相当于整体旋转——对每面的颜色分布不影响（只是重命名了面编号）。所以固定第 1 个 = 减少 24 倍搜索量。**初学者可以把它理解为"以第 1 个立方体为参照系"**。

### 算法方法

**1. 状态空间搜索 / 枚举**
- 枚举所有可能的旋转组合
- n 很小时（≤4）暴搜可行

**2. 旋转表**
- 预计算 24 种旋转的姿态，避免重复计算
- 将 3D 几何问题转化为查表

**3. 贪心统计**
- 对于每个面，找出现最多的颜色，其余需要重涂
- 这是局部最优，全局最优通过搜索保证

### 复杂度分析

- **时间复杂度**：O(24^(n-1) × n × 6)。n ≤ 4 时最大约 13824 × 24 ≈ 3.3×10⁵。
- **空间复杂度**：O(1)。只使用常数大小的旋转表和少量数组。

```cpp
// 例题8  彩色立方体（Colored Cubes, Tokyo 2005, UVa1352）
// Rujia Liu

// 预计算的24种旋转姿态表
// dice24[r][j] = 原第j个面在旋转r后的新位置
// 例如：dice24[0] = {2,1,5,0,4,3} 表示原0面到了2，原1面到了1...
// generated by la3401_make.cpp, then removed extra spaces
int dice24[24][6] = {
{2, 1, 5, 0, 4, 3},{2, 0, 1, 4, 5, 3},{2, 4, 0, 5, 1, 3},{2, 5, 4, 1, 0, 3},{4, 2, 5, 0, 3, 1},
{5, 2, 1, 4, 3, 0},{1, 2, 0, 5, 3, 4},{0, 2, 4, 1, 3, 5},{0, 1, 2, 3, 4, 5},{4, 0, 2, 3, 5, 1},
{5, 4, 2, 3, 1, 0},{1, 5, 2, 3, 0, 4},{5, 1, 3, 2, 4, 0},{1, 0, 3, 2, 5, 4},{0, 4, 3, 2, 1, 5},
{4, 5, 3, 2, 0, 1},{1, 3, 5, 0, 2, 4},{0, 3, 1, 4, 2, 5},{4, 3, 0, 5, 2, 1},{5, 3, 4, 1, 2, 0},
{3, 4, 5, 0, 1, 2},{3, 5, 1, 4, 0, 2},{3, 1, 0, 5, 4, 2},{3, 0, 4, 1, 5, 2},
};

#include<cstdio>
#include<cstring>
#include<string>
#include<vector>
#include<algorithm>
using namespace std;

const int maxn = 4;  // 最多4个立方体
int n, dice[maxn][6], ans;  // dice[i][j]: 第i个立方体第j面的颜色编号

vector<string> names;  // 颜色名→编号的映射
int ID(const char* name) {
  string s(name);
  int n = names.size();
  for(int i = 0; i < n; i++)
    if(names[i] == s) return i;
  names.push_back(s);
  return n;  // 返回新编号
}

int r[maxn];           // r[i]: 第i个立方体的旋转方式编号(0-23)
int color[maxn][6];    // 旋转后，第i个立方体第j面的颜色编号

// 计算当前旋转组合下最少需要重新涂色的面数
void check() {
  // ---- 应用旋转：计算每个立方体旋转后每个面的颜色 ----
  for(int i = 0; i < n; i++)
    for(int j = 0; j < 6; j++)
      // dice24[r[i]][j]：旋转r[i]后，新位置j对应的是原哪个面
      // dice[i][j]：该面原始颜色编号
      color[i][dice24[r[i]][j]] = dice[i][j];

  // ---- 统计每个面的重涂数 ----
  int tot = 0;  // 需要重新涂色的总面数
  for(int j = 0; j < 6; j++) {  // 对6个面分别处理
    int cnt[maxn*6];  // 统计该面上每种颜色出现的次数
    memset(cnt, 0, sizeof(cnt));
    int maxface = 0;  // 出现最多的颜色次数
    for(int i = 0; i < n; i++)
      maxface = max(maxface, ++cnt[color[i][j]]);  // 统计并更新最大值
    // 该面需要重涂的数量 = 总立方体数 - 出现最多的颜色次数
    tot += n - maxface;
  }
  ans = min(ans, tot);  // 更新全局最优
}

// DFS搜索：枚举第d个立方体的旋转方式
void dfs(int d) {
  if(d == n) check();  // 所有立方体都确定了旋转，计算答案
  else for(int i = 0; i < 24; i++) {
    r[d] = i;    // 第d个立方体选择旋转方式i
    dfs(d+1);    // 递归处理下一个
  }
}

int main() {
  while(scanf("%d", &n) == 1 && n) {
    // ---- 输入阶段：读入颜色并转换为编号 ----
    names.clear();
    for(int i = 0; i < n; i++)
      for(int j = 0; j < 6; j++) {
        char name[30];
        scanf("%s", name);
        dice[i][j] = ID(name);  // 将颜色名转为整数编号
      }
    ans = n*6;  // 最坏情况：所有面都需要重新涂色
    r[0] = 0;   // 固定第一个立方体的旋转方式（24种旋转等价，固定一种不重复搜索）
    dfs(1);     // 从第2个立方体开始枚举
    printf("%d\n", ans);
  }
  return 0;
}
```
