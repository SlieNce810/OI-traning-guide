# 2.6 置换及其应用

> **学习目标**：理解置换的循环表示、循环分解算法，以及用Burnside引理/Polya计数处理对称性下的计数问题。

## 理论基础

### 为什么需要学这个？

假如你有n个珠子要用t种颜色染色，但"旋转后相同"的染色方案视为同一种——你会怎么算？直觉告诉你不就是 t^n / n 嘛，但这样算出来往往不是整数，说明有重复计算。这就是置换群要解决的问题：**当对称性存在时，如何正确计数**。置换不仅是群论的入门概念，在算法竞赛中它更是一个实用的工具——循环分解帮你理解排列的结构，Burnside引理让你在对称变换下精确计数，而这些都是有固定模板的套路题。

### 核心概念

**置换的循环表示法**：一个置换可以拆成若干不相交的循环。比如排列 (2,3,1,5,4) 表示：1→2, 2→3, 3→1（一个3-循环）；4→5, 5→4（一个2-循环）。写作 (1 2 3)(4 5)。循环分解是理解置换结构最基本的方式——它告诉你在排列中有多少个"独立运转的小齿轮"。

**循环分解算法**：用一个visited数组标记每个位置。从未访问的位置出发，沿置换函数 P(x) 一直走直到回到起点，得到的环就是一个循环。复杂度O(n)。这个算法极其简单但功能强大：循环的长度分布决定了置换的阶（多少次回到恒等置换）、幂次后的效果等。

**置换群的概念**：一个置换群是某些置换组成的集合，满足封闭性、结合律、存在单位元和逆元。在竞赛中我们主要关心两类群：循环群 C_n（n个旋转）和二面体群 D_n（n个旋转 + n个翻转）。它们是处理环形对象对称性的标准工具。

**置换群与对称群的关系**：n个元素的所有 n! 个置换构成的群称为**对称群** S_n——它是最大的置换群。任何n个元素上的置换群都是 S_n 的子群（包括C_n和D_n）。理解这个层级关系很重要：S_n 的子群种类繁多（由凯莱定理，每个有限群都同构于某个置换群），但竞赛中只关注特定的子集。C_n是S_n的子群，|C_n|=n；D_n包含C_n和n个翻转，|D_n|=2n。两者的共同点是它们都描述了在平面上绕一个正n边形顶点的对称操作。理解了S_n是"最大的置换宇宙"，C_n和D_n是其中两个特定的"小宇宙"，你就不会对题目中出现的各种对称群感到陌生。

**Burnside引理与Polya计数**：Burnside引理说：在群G作用下，不同轨道的个数 = (1/|G|) * Σ_{g∈G} Fix(g)，其中Fix(g)是在置换g下保持不变的着色方案数。Polya计数定理是对Burnside的推广——当每个循环内的元素必须同色时，Fix(g) = t^{c(g)}，其中c(g)是g的循环个数。这样着色计数就变成了简单的指数运算。

**用涂色问题理解Burnside与Pólya**：以"用t种颜色给n颗珠子涂色，旋转相同视为同一方案"为例。第一步，群G是旋转群C_n，|G|=n。第二步，旋转k步的置换将珠子i映射到(i+k)mod n，它的循环结构如何？当gcd(k,n)=d时，置换分为d个长度为n/d的循环。第三步，Polya公式中：旋转k步对应的不动点数目 = t^{gcd(k,n)}（因为每个循环内珠子必须同色）。第四步，总不同方案数 = 1/n · Σ_{k=0}^{n-1} t^{gcd(k,n)}。以n=4,t=2为例：k=0时gcd(0,4)=4贡献2^4=16；k=1时gcd=1贡献2；k=2时gcd=2贡献4；k=3时gcd=1贡献2；总和=(16+2+4+2)/4=6种。你可以画出这6种：全红、全蓝、3红1蓝、1红3蓝、红蓝红蓝交替、红红蓝蓝相邻。Burnside告诉你答案一定是整数，Polya让你不需要枚举就能算出。如果在涂色问题中加入翻转（变成二面体群D_n），公式中新加n个翻转置换的贡献——n为奇数时每个翻转有(n+1)/2个循环，偶数时一半有n/2+1个循环、一半有n/2个循环——这就是Polya模板中分支判断的来源。

**置换的轮换指标（Cycle Index）**：轮换指标 Z(G) 是Burnside/Polya的代数化表述。对于群G，定义 Z(G; x_1, x_2, ..., x_n) = 1/|G| · Σ_{g∈G} Π_{c∈cycles(g)} x_{length(c)}。其中 Π 是对g的每个循环取x_{循环长度}的乘积。这个多项式看起来复杂，但它的妙处在于：每种涂色方案数就是 Z(G; t, t, ..., t)（所有x_i用t替换），这与Polya公式 t^{c(g)} 完全一致。轮换指标的价值在于：当你遇到"每种颜色有使用限制"的变种问题（如红色最多用r颗），就不能简单指数，而需要展开轮换指标中的 x_i → (颜色的某种组合计数)，本质上是用生成函数替换每个变量。在竞赛中，轮换指标对于理解"带限制的Polya计数"（如Cayley定理在计数有根树时的应用）是不可或缺的基础概念。

### 知识脉络

```
循环分解  ──→  循环结构分析  ──→  平方根/幂次性质
     │
     └── 置换群  ──→  Burnside引理  ──→  轨道计数
                         │
                         ├── 循环群C_n (旋转)
                         │
                         └── 二面体群D_n (旋转+翻转)
                                   │
                                   └── Polya计数: 方案数 = 1/|G| Σ t^{c(g)}
```

循环分解是操作基础，用于分析单个置换的结构。Burnside引理是理论核心，回答了"对称变换下有多少种本质不同的方案"。Polya计数是Burnside的特化版本，专门处理着色计数，公式极其简洁。本节中的**Polya计数的指数公式**与第2.1节的排列组合基础直接相连——每个循环内的颜色分配本质上就是一个乘法原理的应用；**置换的循环分解算法**则与第1.3节的"Floyd判圈"共享相同的"追踪序列直到回到起点"的图遍历思想；而**Burnside引理中的容斥求和**则是第2.1节容斥原理在群作用上的推广。

### 快速上手模板

```cpp
// 置换的循环分解
int vis[MAXN], cnt_len[MAXN];  // cnt_len[l]:长度为l的循环个数
void decompose(int* P, int n) {
    memset(vis, 0, sizeof(vis));
    memset(cnt_len, 0, sizeof(cnt_len));
    for (int i = 0; i < n; i++) {
        if (vis[i]) continue;
        int j = i, len = 0;
        do {
            vis[j] = 1;
            j = P[j];   // 沿置换走一步
            len++;
        } while (j != i);
        cnt_len[len]++;
    }
}

// Polya计数：n个珠子用t种颜色染色，考虑旋转和翻转
// 返回值: (项链数, 手镯数)
pair<LL, LL> polya(int n, int t) {
    LL neck = 0, brac = 0;
    // 预计算 t^0, t^1, ..., t^n
    vector<LL> pow_t(n + 1, 1);
    for (int i = 1; i <= n; i++) pow_t[i] = pow_t[i-1] * t;

    // 旋转部分
    for (int k = 0; k < n; k++)
        neck += pow_t[gcd(k, n)];

    // 翻转部分（加入二面体群）
    if (n % 2 == 1)
        brac = n * pow_t[(n + 1) / 2];
    else
        brac = n/2 * (pow_t[n/2 + 1] + pow_t[n/2]);

    return {neck / n, (neck + brac) / (2 * n)};
}
```

## 例题23  Leonardo的笔记本（Leonardo's Notebook , NWERC 2006, Codeforces Gym100722I）

### 题目描述
Leonardo da Vinci以镜像书写（反向书写）而闻名。他的笔记本上有一个字符串B（长度为26的大写字母排列，即A-Z的置换）。问是否存在一个排列A，使得A<U+00B2> = B（即B是A的平方置换）。这里的平方指A应用两次：B[i] = A[A[i]]。

**输入**：第一行T（T ≤ 500）。接下来T行，每行一个长度为26的大写字母排列B。

**输出**：如果存在A满足A<U+00B2>=B，输出"Yes"；否则输出"No"。

### 解题思路

**置换的循环分解**：
将排列B分解为若干个不相交的循环。B的每个循环长度为L。

关键结论：一个排列有平方根当且仅当对于每个偶数长度L的循环，B中该长度的循环个数为偶数。

**证明**：
- 设A有一个长度为L的循环。在A<U+00B2>（即B）中，该循环会发生什么？
  - 如果L为奇数，在A<U+00B2>中该循环仍然是一个长度为L的循环
  - 如果L为偶数，在A<U+00B2>中该循环会分裂为两个长度为L/2的循环

- 反过来，B中的偶数长度循环必须成对出现才能配对合并为A中的循环。例如B中有两个长度为L的循环，它们在A中可以合并为一个长度为2L的循环。

因此，只需要统计B中每种循环长度的出现次数，检查所有偶数长度的循环出现次数是否为偶数。

**实现**：
1. 遍历26个字母，对于每个未访问的字母，沿着B追踪形成循环
2. 记录每个循环的长度，cnt[len]++
3. 检查cnt[len]对所有偶数len是否为偶数

### 算法方法
- **置换群**：循环分解 + 平方根存在条件
- 利用置换的循环结构性质

### 复杂度分析
- **时间复杂度**：O(T·26)，每组O(26)遍历字母
- **空间复杂度**：O(26)，存储标记数组

```cpp
// 例题23  Leonardo的笔记本（Leonardo's Notebook , NWERC 2006, Codeforces Gym100722I）
// Rujia Liu
#include<cstdio>
#include<cstring>
int main() {
  char B[30];       // 输入的排列
  int vis[30], cnt[30], T;  // vis:访问标记, cnt[len]:长度为len的循环个数

  scanf("%d", &T);
  while(T--) {
    scanf("%s", B);
    memset(vis, 0, sizeof(vis));
    memset(cnt, 0, sizeof(cnt));

    // 循环分解：对每个未访问的字母追踪其循环
    for(int i = 0; i < 26; i++)
      if(!vis[i]) { // 找一个从i开始的循环
        int j = i, n = 0;  // n: 循环长度
        do {
          vis[j] = 1; // 标记j为已访问
          j = B[j] - 'A';       // B[j]是j的下一个位置（注意B[j]是大写字母）
          n++;                  // 循环长度+1
        } while(j != i);        // 回到起点，循环结束
        cnt[n]++;  // 记录该长度循环的出现次数
      }

    // 检查条件：所有偶数长度循环的个数必须为偶数
    int ok = 1;
    for(int i = 2; i <= 26; i++)
      if(i%2 == 0 && cnt[i]%2 == 1) ok = 0;  // 偶数长度奇数个则不存在平方根

    if(ok) printf("Yes\n"); else printf("No\n");
  }
  return 0;
}
```

## 例题22  项链和手镯（Arif in Dhaka(First Love Part 2), UVa 10294）

### 题目描述
有n颗珠子（n ≤ 50），每颗珠子可以用t种颜色中的一种染色。考虑两种对称情况：
1. **项链（Necklace）**：可以旋转。两个项链视为相同当且仅当可以通过旋转互相转化。
2. **手镯（Bracelet）**：可以旋转和翻转。两个手镯视为相同当且仅当可以通过旋转或翻转变换互相转化。

分别求n颗珠子t种颜色的项链数和手镯数。

**输入**：多组数据，每组一行n和t（0 < n ≤ 50，0 < t ≤ 10）。n=t=0结束。

**输出**：对于每组数据，输出两个整数：项链数和手镯数。

### 解题思路

**Polya计数定理（Burnside引理）**：

**项链（仅旋转）**：
旋转群C_n = {旋转k个位置 | k=0,1,…,n-1}。

对于旋转k个位置的置换，珠子被划分成gcd(k,n)个循环，每个循环长度n/gcd(k,n)。每个循环内的珠子必须同色。

根据Burnside引理：
项链数 = (1/n)·Σ_{k=0}^{n-1} t^{gcd(k,n)}

**手镯（旋转+翻转）**：
在二面体群D_n下计数。除了旋转外还有翻转。

翻转（反射）：
- 如果n为奇数：有n条通过对角的对称轴，每条将珠子分为(n+1)/2个循环
- 如果n为偶数：有n/2条通过相对珠子的轴（每个轴分成n/2+1个循环）和n/2条通过相对空隙的轴（每个轴分成n/2个循环）

根据Burnside引理：
手镯数 = (1/(2n))·(Σ_{k=0}^{n-1} t^{gcd(k,n)} + n·t^{(n+1)/2})  // n奇数
手镯数 = (1/(2n))·(Σ_{k=0}^{n-1} t^{gcd(k,n)} + (n/2)·t^{n/2+1} + (n/2)·t^{n/2})  // n偶数

### 算法方法
- **置换群/Polya计数**：Burnside引理
- **数论**：gcd函数用于计算旋转的循环数

### 复杂度分析
- **时间复杂度**：O(n·log n)，n≤50，预计算t^i
- **空间复杂度**：O(n)，存储t的幂

```cpp
// 例题22  项链和手镯（Arif in Dhaka(First Love Part 2), UVa 10294）
// 陈锋
#include <cstdio>
typedef long long LL;
const int NN = 100;

// 欧几里得算法求最大公约数
int gcd(int a, int b) { return b == 0 ? a : gcd(b, a % b); }

int main() {
  for (int n, t; scanf("%d%d", &n, &t) == 2 && n;) {
    LL pow[NN], a = 0, b = 0;  // a:旋转群Burnside和, b:翻转群Burnside和
    pow[0] = 1;  // t^0 = 1
    for (int i = 1; i <= n; i++) pow[i] = pow[i - 1] * t;  // 预计算t^i

    // 旋转部分：Σ_{i=0}^{n-1} t^{gcd(i,n)}
    for (int i = 0; i < n; i++) a += pow[gcd(i, n)];

    // 翻转部分
    if (n % 2 == 1)
      b = n * pow[(n + 1) / 2];  // n奇数：n条轴，每条(n+1)/2个循环
    else
      b = n / 2 * (pow[n / 2 + 1] + pow[n / 2]);  // n偶数：n/2条(轴) * (t^{n/2+1}+t^{n/2})

    // 项链数 = a/n，手镯数 = (a+b)/(2n)
    printf("%lld %lld\n", a / n, (a + b) / 2 / n);
  }
  return 0;
}
```

## 例题24  排列统计（Find the Permutations, UVa 11077）

### 题目描述
给定正整数n和k（1 ≤ n ≤ 21，0 ≤ k < n），求1到n的排列中有多少个排列恰好需要进行k次交换才能变为有序排列（交换任意两个元素，每次交换算一次操作）。即求有多少个置换的"最短排序交换次数"等于k。

注：将一个排列通过交换相邻元素变成有序的最小次数是逆序数；但本题允许交换任意两个元素。

**输入**：多组数据，每组一行n和k。n=k=0结束。

**输出**：对于每组数据，输出排列数。

### 解题思路

**第一类斯特林数**：
排列的"排序所需最小交换次数"实际上等于n减去排列的循环数。这是因为长度为L的循环可以通过L-1次交换变成有序（每次交换可以把一个元素放对位置）。

因此，求恰好需要k次交换的排列数 = 求循环数为n-k的排列数。

这恰好是**第一类无符号斯特林数** s(n, n-k) = c(n, k)，即n个元素分成n-k个循环的排列方式数。

**递推公式**：
设f[n][k] = 长度为n的排列中，需要恰好k次交换变为有序的排列数。
（等价于第一类斯特林数s(n, n-k)）

递推：
f[1][0] = 1（单位排列，0次交换）
f[n][k] = f[n-1][k] + f[n-1][k-1]·(n-1)

解释：考虑第n个元素的插入位置。
- 情况1：n单独成一个循环（放在末尾，不需交换），从f[n-1][k]转移
- 情况2：n插入到已有的某个循环中。有n-1个位置可以插入，每次插入使需要的交换次数+1（增加1个交换但循环数不变），贡献n-1倍的f[n-1][k-1]

### 算法方法
- **置换群/第一类斯特林数**：循环数与交换次数的关系
- **递推**：第一类斯特林数的递推公式

### 复杂度分析
- **时间复杂度**：O(n²)，预处理所有n≤21，k<n
- **空间复杂度**：O(n²)，DP表

```cpp
// 例题24  排列统计（Find the Permutations, UVa 11077）
// 刘汝佳
#include <cstdio>
#include <cstring>
const int maxn = 30;
// f[i][j]: 长度为i的排列需要j次交换才能变为有序的排列数
// (等价于第一类无符号斯特林数s(i, i-j))
unsigned long long f[maxn][maxn];

int main() {
  memset(f, 0, sizeof(f));
  f[1][0] = 1;  // 边界：单元素排列不需要交换

  // 递推计算
  for (int i = 2; i <= 21; i++)
    for (int j = 0; j < i; j++) {
      f[i][j] = f[i - 1][j];  // 第i个元素自成循环，不增加交换次数
      if (j > 0)
        f[i][j] += f[i - 1][j - 1] * (i - 1);  // 第i个元素插入已有循环，交换+1
        // (i-1)表示有i-1个位置可以插入
    }

  int n, k;
  while (scanf("%d%d", &n, &k) == 2 && n)
    printf("%llu\n", f[n][k]);
  return 0;
}
```
